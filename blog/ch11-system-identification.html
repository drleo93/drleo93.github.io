Chapter 11 — ARX, ARMAX, N4SID, NARX, and Neural State-Space Models

TL;DR

  • System identification is the science of building dynamic models from input-output data. Choose the model structure before looking at data — this is not optional.
  • Input signal design is as important as model selection. Use PRBS for linear systems, multisine for frequency-specific excitation, GBN for systems with bandwidth constraints.
  • ARX: closed-form LS solution, but noise enters through $1/A(q)$ — biased if noise is correlated. Use for one-step prediction. Use OE for simulation.
  • BIC, not AIC, for model order selection in engineering (penalises overparameterisation more strongly).
  • N4SID provides a consistent, non-iterative subspace identification method for state-space models — the right starting point for MIMO systems.
  • NARX and Hammerstein-Wiener handle nonlinear dynamics; neural state-space models are the data-driven limit.

1. The SysID Framework

System identification follows a four-step loop that must be executed in order:

  1. Experiment design: select input signals that maximally excite the system dynamics of interest. A poorly designed experiment cannot be recovered by better modelling.
  2. Model structure selection: choose between ARX, ARMAX, OE, BJ, state-space, or nonlinear structures based on the physical system and intended use.
  3. Parameter estimation: minimise the prediction error (PEM) or use subspace methods (N4SID).
  4. Validation: distinguish one-step prediction error (easy) from simulation error (hard, but what matters for controllers and virtual sensors).
Pitfall — Prediction vs. Simulation Error
A model validated only on one-step prediction error (residual analysis, whiteness test) may still diverge catastrophically in simulation. Always perform a simulation validation: apply the identified model in open-loop simulation and compare against held-out data. This is the definitive test.

2. Input Signal Design

The quality of identification is bounded by the richness of the input signal. The Fisher information matrix $\mathcal{I}(\theta)$ is maximised by choosing $u(t)$ such that its power spectral density covers the dynamically significant frequency band $[f_{min}, f_{max}]$.

SignalPropertiesBest ForEngineering Use
PRBS Flat spectrum up to $f_{max} = 1/(2T_{clock})$; binary ±amplitude; maximal length shift register Linear systems, broadband excitation Engine ECU, industrial actuators
Multisine Sum of sinusoids at selected frequencies; exact spectral shaping; crest factor optimisable Frequency-specific excitation; nonlinear analysis (odd harmonics) Structural dynamics, impedance spectroscopy
GBN (Generalized Binary Noise) Binary signal with programmable switching probability; tunable bandwidth Systems with bandwidth constraints (rate limiters, actuator saturation) Hydraulic systems, HVAC
Chirp Linearly increasing frequency; easy to interpret in time-frequency domain Systems with known natural frequencies; resonance detection Vibration testing, structural health monitoring
Input Design Rule
  • PRBS clock period $T_{clock}$: set to 3–10× the system settling time. Too short = input changes faster than system can respond; too long = insufficient spectral richness.
  • Signal amplitude: stay within the linear operating region if a linear model is intended. If nonlinear, sweep multiple amplitude levels.
  • Multisine: optimise phases (Schroeder or random) to minimise crest factor and avoid actuator saturation.

3. Linear Model Structures

All linear SysID model structures are members of the general Box-Jenkins family:

$$y(t) = \frac{B(q)}{F(q)} u(t) + \frac{C(q)}{A(q) D(q)} e(t)$$

where $q$ is the shift operator ($q^{-1} x(t) = x(t-1)$), $A, B, C, D, F$ are polynomials in $q^{-1}$, and $e(t)$ is white noise. Different structures arise from constraints on these polynomials:

ModelStructureEstimationUse When
ARX $A(q)y = B(q)u + e$ ($F=C=D=1$) Closed-form LS; fast, global optimum One-step prediction; low-noise systems
ARMAX $A(q)y = B(q)u + C(q)e$ ($F=D=1$) Iterative PEM Correlated noise; moving average disturbances
OE $y = [B(q)/F(q)]u + e$ ($A=C=D=1$) Iterative PEM Simulation; when noise model not needed
BJ Full Box-Jenkins; separate noise model $C/D$ Iterative PEM; most flexible When noise structure is physically known

ARX: Closed-Form Solution and Its Limitation

$$\text{ARX model: } y(t) + a_1 y(t-1) + \ldots + a_{n_a} y(t-n_a) = b_1 u(t-1) + \ldots + b_{n_b} u(t-n_b) + e(t)$$ $$\text{LS solution: } \hat{\theta}_{ARX} = (\Phi^\top \Phi)^{-1} \Phi^\top Y$$
Pitfall — ARX Bias
In ARX, the noise enters as $e(t)/A(q)$ — the noise model is forced to be $1/A(q)$. If the actual noise is correlated differently, the parameter estimates are biased even with infinite data. Use ARX for one-step prediction tasks; use OE for simulation tasks where model accuracy (not noise model accuracy) matters.

Model Order Selection: BIC over AIC

$$\text{AIC} = -2\ln\hat{L} + 2p \qquad \text{BIC} = -2\ln\hat{L} + p\ln N$$

where $p$ is the number of parameters and $N$ is the number of samples. BIC penalises complexity more strongly for large $N$. In engineering, where overparameterised models cause simulation drift, prefer BIC. Fit multiple orders $(n_a, n_b) \in \{1,\ldots,6\}^2$, select by BIC, validate by residual whiteness and simulation error.

4. N4SID: Subspace Identification

For MIMO systems and state-space representations, N4SID (Numerical Subspace State-Space Identification) avoids the non-convex optimisation of PEM by working directly with oblique projections of the block Hankel matrix of input-output data.

State-space target model: $$x(t+1) = Ax(t) + Bu(t) + w(t)$$ $$y(t) = Cx(t) + Du(t) + v(t)$$

N4SID constructs the block Hankel matrices from data and uses the oblique projection $\mathcal{O}_i \hat{X}_i = \mathcal{Y}_i^{f} / \mathcal{U}_i^{f} | \mathcal{W}_i^{p}$ to extract the state sequence $\hat{X}_i$ via SVD. The system matrices $A, B, C, D$ are then recovered by least squares from the state sequence.

Lesson
N4SID provides a consistent, non-iterative estimator — no initialisation required, no local optima. Use it as the starting point for MIMO systems (≥2 inputs, ≥2 outputs). For scalar SISO problems with known polynomial structure, PEM (ARMAX/OE) is simpler and more interpretable. Refine N4SID solutions with PEM for maximum likelihood estimates.

5. Nonlinear System Identification

NARX: Nonlinear ARX

NARX extends ARX by replacing the linear regression with a nonlinear function:

$$y(t) = f(y(t-1),\ldots,y(t-n_a), u(t-1),\ldots,u(t-n_b)) + e(t)$$

The function $f$ can be a neural network, polynomial, or Gaussian process. NARX is the standard nonlinear SysID approach for moderate nonlinearity. Key properties: the regressor space is fixed by the lag orders $(n_a, n_b)$; model selection reduces to choosing $f$ and lag orders.

Hammerstein-Wiener: Block-Structured Nonlinear Models

Block-structured models assume a specific cascade of static nonlinearities and linear dynamics — physically motivated for many engineering systems:

$$\text{Hammerstein: } \quad u \xrightarrow{f_{NL}} v \xrightarrow{G(q)} y$$ $$\text{Wiener: } \quad u \xrightarrow{G(q)} v \xrightarrow{f_{NL}} y$$ $$\text{Hammerstein-Wiener: } \quad u \xrightarrow{f_{NL,1}} v \xrightarrow{G(q)} w \xrightarrow{f_{NL,2}} y$$
Example — Hammerstein Model for Throttle-to-Airflow

Physical structure: throttle position $\alpha$ drives a nonlinear discharge coefficient $C_d(\alpha)$ (static nonlinearity), followed by the linear intake manifold filling dynamics $G(q)$ (first-order lag), followed by a nonlinear air density correction $\rho(p, T)$ (static nonlinearity).

Identification: the static nonlinearity $C_d(\alpha)$ is identified from steady-state sweep data. The linear dynamics $G(q)$ are identified from PRBS excitation around operating points. The density correction uses the ideal gas law.

Advantage over black-box NARX: each block has physical meaning; the linear block is identifiable with standard methods; the nonlinear blocks need only steady-state data.

Neural State-Space Models

The data-driven limit: replace the linear state-space matrices with neural networks:

$$x(t+1) = f_\theta(x(t), u(t)) \qquad y(t) = g_\phi(x(t), u(t))$$

where $f_\theta$ and $g_\phi$ are MLPs or LSTMs. Training requires backpropagation through the state recursion — equivalent to rollout training from chapter 10. Key challenge: the state $x(t)$ is latent and must be initialised; use a separate encoder network $x(0) = \text{enc}_\psi(y_{0:T_0}, u_{0:T_0})$ trained jointly.

Nonlinear SysID Decision Tree
  • Mild nonlinearity (Hammerstein/Wiener structure physically plausible): use block-structured model
  • Moderate nonlinearity, scalar output: NARX with GP or neural network $f$
  • Strong nonlinearity, multiple states, known latent structure: neural state-space model
  • Known physics with uncertain parameters: grey-box ODE + PEM (see Ch07)
  • When in doubt: start with ARX/OE linear model and examine simulation error. If simulation RMSE >3× one-step RMSE, the residual is structured — introduce nonlinearity.

Pitfall — Overfitting in SysID
Increasing model order always improves one-step fit. Always split data: identification set (60%), model selection set (20%, BIC), and simulation validation set (20%, never touched until final model). The simulation validation set is sacred — accessing it for model selection invalidates the validation.

7. Key Takeaways

  • Design the experiment first: PRBS for broadband linear ID; multisine for frequency-specific; GBN for bandwidth-constrained systems.
  • ARX for prediction, OE for simulation: ARX bias is irrelevant for one-step prediction but fatal for closed-loop simulation.
  • BIC over AIC for model order selection in engineering — penalises overparameterisation more strongly.
  • N4SID: start here for MIMO state-space identification. Non-iterative, consistent, no local optima.
  • Hammerstein-Wiener when the physical structure implies cascaded static nonlinearity and linear dynamics (most actuator + flow systems).
  • Neural state-space: the data-driven limit. Requires rollout training and a state encoder. Use only when block-structured or NARX models are insufficient.
  • The definitive validation is simulation error on held-out data. Residual whiteness is necessary but not sufficient.
