TL;DR

  • Clustering serves six distinct purposes in engineering: operating mode detection, fault taxonomy, data-driven DOE, data compression, transfer learning, and exploratory analysis.
  • K-means: fast ($\mathcal{O}(nKdt)$), spherical clusters. Select $K$ by combined silhouette + Davies-Bouldin score, not just the elbow. Always standardise features first.
  • DBSCAN: no $K$ needed, arbitrary-shape clusters, identifies noise points. Use when clusters have irregular shapes (fault manifolds in vibration space).
  • GMM: probabilistic soft assignments; correct for elliptical clusters; provides a density score for anomaly detection: $s(\mathbf{x}) = -\log p(\mathbf{x})$.
  • Bayesian GMM (Dirichlet process) removes the need to specify $K$ — components with $\pi_k < 0.01$ are automatically pruned.
  • Always validate clusters physically: a statistically optimal $K$ that produces physically uninterpretable clusters is not a useful model.

1. Why Clustering in Engineering ML?

Six distinct purposes in engineering surrogate workflows:

  1. Operating mode detection: identify distinct dynamic regimes in a drive cycle, process, or grid to allow mode-switching or multi-model surrogates.
  2. Fault taxonomy: without labelled fault data, cluster historical anomalies to identify distinct fault types; interpret clusters physically to create maintenance procedures.
  3. Data-driven DOE: cover the operating space uniformly by clustering existing data and placing new experimental points where cluster density is lowest.
  4. Data compression: represent a large dataset by its cluster centroids for initialisation, dimensionality reduction, or codebook generation.
  5. Transfer learning: identify which source domain clusters are most similar to the target domain for domain adaptation.
  6. Exploratory analysis: discover structure in high-dimensional sensor data invisible in pairwise scatter plots.

2. K-Means Clustering

K-means partitions $n$ points into $K$ clusters by minimising within-cluster sum of squares (inertia):

$$J_K = \sum_{k=1}^K \sum_{\mathbf{x}_i \in \mathcal{C}_k} \|\mathbf{x}_i - \boldsymbol{\mu}_k\|_2^2$$

The Lloyd-Forgy algorithm alternates assignment (E-step: assign each point to the nearest centroid) and centroid update (M-step: recompute centroids) until convergence. Complexity: $\mathcal{O}(nKdt)$.

Choosing $K$: Silhouette + Davies-Bouldin

$$\text{Silhouette: } s_i = \frac{b_i - a_i}{\max(a_i, b_i)} \in [-1, 1]$$

where $a_i$ = mean intra-cluster distance, $b_i$ = mean nearest-cluster distance. $\bar{s} \to 1$ indicates well-separated clusters. The Davies-Bouldin index measures average cluster overlap — lower is better.

from sklearn.cluster import KMeans
from sklearn.metrics import silhouette_score, davies_bouldin_score
from sklearn.preprocessing import StandardScaler
import numpy as np

def select_k_and_cluster(X, k_range=(2, 15), n_init=20, random_state=42):
    """Select optimal K using combined silhouette + Davies-Bouldin score."""
    scaler = StandardScaler()
    X_s = scaler.fit_transform(X)

    K_vals, sils, dbs = [], [], []
    for K in range(*k_range):
        km = KMeans(n_clusters=K, n_init=n_init, random_state=random_state)
        labels = km.fit_predict(X_s)
        K_vals.append(K)
        sils.append(silhouette_score(X_s, labels, sample_size=min(5000, len(X))))
        dbs.append(davies_bouldin_score(X_s, labels))

    # Combined score: maximise silhouette, minimise DB
    sil_norm = (np.array(sils) - min(sils)) / (max(sils) - min(sils) + 1e-9)
    db_norm  = 1 - (np.array(dbs) - min(dbs)) / (max(dbs) - min(dbs) + 1e-9)
    combined = 0.6 * sil_norm + 0.4 * db_norm
    best_K   = K_vals[int(np.argmax(combined))]

    best_km = KMeans(n_clusters=best_K, n_init=n_init, random_state=random_state)
    labels  = best_km.fit_predict(X_s)
    print(f"Optimal K = {best_K}  |  Silhouette = {sils[best_K - k_range[0]]:.3f}")
    return best_K, best_km, labels, scaler
Pitfall — Elbow Method Alone is Insufficient
The elbow of the inertia curve is often ambiguous — the "elbow" can appear anywhere from $K=3$ to $K=8$ depending on scale. Always combine the elbow with silhouette and Davies-Bouldin. More importantly: always validate the optimal $K$ physically. A K=6 solution with silhouette=0.6 that produces physically uninterpretable clusters is worse than a K=4 solution with silhouette=0.55 where each cluster maps to a known operating regime.

3. DBSCAN: Density-Based Clustering

DBSCAN finds clusters of arbitrary shape without requiring $K$ to be specified. It classifies each point as:

  • Core point: has ≥ min_samples neighbours within radius $\varepsilon$.
  • Border point: reachable from a core point but not itself a core point.
  • Noise point: neither core nor border; labelled $-1$.
DBSCAN Parameter Selection
  1. $\varepsilon$: plot the $k$-nearest-neighbour distance (for $k = $ min_samples) sorted ascending. Choose $\varepsilon$ at the "knee" of this curve.
  2. min_samples: set to $d+1$ at minimum; $2d$ for noisy datasets. For $d > 10$: use $\max(5, d/2)$.
  3. Distance metric: for physical engineering data with mixed units, use Mahalanobis distance or pre-standardise and use Euclidean.
  4. Prefer DBSCAN over K-means when: clusters have irregular shapes (fault manifolds), noise points must be identified, or the number of clusters is unknown.

4. Gaussian Mixture Models (GMM)

GMM represents the data as a mixture of $K$ Gaussians, estimated by the Expectation-Maximisation (EM) algorithm:

$$p(\mathbf{x}) = \sum_{k=1}^K \pi_k\,\mathcal{N}(\mathbf{x};\,\boldsymbol{\mu}_k, \boldsymbol{\Sigma}_k)$$ $$\text{E-step: } \gamma_{ik} = \frac{\pi_k\,\mathcal{N}(\mathbf{x}_i;\,\boldsymbol{\mu}_k,\boldsymbol{\Sigma}_k)}{\sum_j \pi_j\,\mathcal{N}(\mathbf{x}_i;\,\boldsymbol{\mu}_j,\boldsymbol{\Sigma}_j)}$$ $$\text{M-step: } N_k = \sum_i \gamma_{ik}, \quad \boldsymbol{\mu}_k = \frac{1}{N_k}\sum_i \gamma_{ik}\mathbf{x}_i, \quad \boldsymbol{\Sigma}_k = \frac{1}{N_k}\sum_i \gamma_{ik}(\mathbf{x}_i - \boldsymbol{\mu}_k)(\mathbf{x}_i - \boldsymbol{\mu}_k)^\top$$

GMM provides:

  • Soft assignments $\gamma_{ik} = p(k|\mathbf{x}_i)$ — probabilistic cluster membership.
  • Anomaly score $s(\mathbf{x}) = -\log p(\mathbf{x})$ — log-likelihood anomaly detection.
  • Elliptical clusters — handles elongated and correlated cluster shapes that K-means (spherical) cannot.

Connection to K-means: in the limit $\boldsymbol{\Sigma}_k = \sigma^2\mathbf{I}$, $\sigma^2 \to 0$, soft assignments become one-hot and EM reduces to K-means. GMM is the probabilistic generalisation.

Bayesian GMM: Automatic Component Selection

Bayesian GMM with a Dirichlet process prior (DPGMM) removes the need to specify $K$: start with $K_{max}$ components; place a Dirichlet concentration prior $\alpha_0$ on the mixing weights. After fitting, components with $\pi_k < 0.01$ are automatically pruned. Use sklearn.mixture.BayesianGaussianMixture(n_components=20, n_init=5) — the algorithm determines the effective $K$ automatically.

6. Engineering Application: Operating Mode Detection

Example — Drive Cycle Operating Mode Detection

Setup: 1 Hz ECU data from 500 real-world drive cycles. Features: engine speed $N$, load $\text{BMEP}$, coolant temperature $T_{cool}$, gear position. Goal: identify distinct operating modes for mode-switching surrogate models.

Method: standardise features; apply K-means with silhouette selection. Result: $K = 5$ modes — cold idle, warm idle, city driving (low load), motorway (high load), deceleration fuel cut-off.

Validation: each cluster is physically interpretable; centroid $N$ and BMEP values match known ECU operating points. Train a separate XGBoost surrogate per mode — overall RMSE drops by 23% vs. a single global model.

7. Key Takeaways

  • Always standardise features before clustering — K-means and GMM are sensitive to feature scale.
  • Use combined silhouette + Davies-Bouldin for $K$ selection, not elbow alone. Validate the result physically.
  • K-means: fast, good for convex clusters. Use n_init=20 to avoid poor local optima.
  • DBSCAN: no $K$ needed, handles irregular shapes and explicitly marks noise. Choose $\varepsilon$ from the $k$-NN distance curve knee.
  • GMM: probabilistic soft assignments + anomaly score. Use Bayesian GMM (DPGMM) when $K$ is unknown.
  • Mode-switching models (one surrogate per cluster) consistently outperform global models when operating regimes have distinct dynamics.
  • Fault taxonomy via DBSCAN: label all historical anomalies, cluster them without class labels, then interpret clusters physically. This is how you build a fault dictionary from unlabelled field data.
