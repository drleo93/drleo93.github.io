Ch 3: Engineering Data Analysis & Feature Engineering

Chapter 3 of 23  ·  ~35 min read  ·  Highest leverage activity in engineering ML
EDA data audit Buckingham π feature engineering Arrhenius leakage outliers data splitting imbalance

TL;DR

  • Feature engineering based on dimensional analysis (Buckingham π) is worth more than any algorithm choice. Dimensionless inputs routinely reduce required training data by 3–10×.
  • Temporal and group leakage is the most expensive ML mistake in engineering: a model with leakage passes all standard tests and fails in deployment.
  • Data imbalance is a physics problem — fix it with targeted DOE in the underrepresented region, not synthetic augmentation.
  • Compute the Dataset Quality Index (DQI) before any modelling. DQI < 0.90: do not train, regardless of validation RMSE.
  • Use Spearman correlation (not Pearson) for engineering responses; use mutual information for non-monotone features.
  • The Arrhenius transform \(1/T\) linearises thermal activation — use it for flame speed, ignition delay, catalyst deactivation, creep.

1. The Data Audit: Before Any Modelling

Every engineering ML project begins with a structured data audit before any transformation, training, or visualisation. Six questions must be answered:

  1. Provenance: instrument, calibration certificate, sampling rate, test rig ID. A 100 Hz NOx analyser and a 5 Hz one have different response functions; mixing them introduces systematic error.
  2. Completeness and missing mechanism: MCAR/MAR/MNAR classification per column. A clipped NOx analyser returning 3000 ppm when overranged is MNAR and must be corrected before any imputation.
  3. Physical plausibility: does every signal respect physical bounds? Negative concentrations, \(\eta > 1\), and \(T < T_{amb}\) indicate faults or unit errors.
  4. Temporal ordering: rows must be in chronological order within each session. Engine test data is frequently stored out of order.
  5. Group structure: which rows belong to the same physical unit? This determines the correct cross-validation strategy.
  6. Target distribution: skewness > 1 for NOx, PM, and concentration targets requires log-transformation before Gaussian-assumption models.

Dataset Quality Index (DQI)

Four component scores, each in \([0,1]\):

$$\mathrm{DQI} = (q_c \cdot q_s \cdot q_p \cdot q_t)^{1/4}$$
Rule — DQI thresholds
DQI ≥ 0.97: proceed to modelling. DQI ∈ [0.90, 0.97): document issues and apply targeted cleaning. DQI < 0.90: do not train — the resulting model cannot be trusted regardless of validation RMSE. 
def data_quality_score(df, bounds, redundant_pairs=None, dt_expected=None):
    q_c = 1 - df.isnull().mean().mean()
    plaus = pd.Series(1.0, index=df.columns)
    for col, (lo, hi) in bounds.items():
        if col in df:
            plaus[col] = ((df[col] >= lo) & (df[col] <= hi)).mean()
    q_p = plaus.mean()
    roll_med = df.rolling(50, center=True, min_periods=5).median()
    roll_mad = (df - roll_med).abs().rolling(50, center=True, min_periods=5).median()
    q_s = ((df - roll_med).abs() < 3 * roll_mad + 1e-9).mean().mean()
    if dt_expected and 'timestamp' in df.columns:
        dts = df['timestamp'].diff().dropna()
        q_t = ((dts - dt_expected).abs() < 0.1*dt_expected).mean()
    else:
        q_t = 1.0
    dqi = (q_c * q_s * q_p * q_t) ** 0.25
    return {'DQI': dqi, 'completeness': q_c, 'consistency': q_s,
            'plausibility': q_p, 'temporal': q_t}

2. Summary Statistics and Correlation Analysis

Descriptive Statistics with Physical Interpretation

Skewness \(\gamma\) and excess kurtosis \(\kappa\):

  • \(\gamma > 1\): right-skewed — characteristic of NOx, CO, PM, concentration data. Apply \(\log(1+x)\).
  • \(\kappa > 3\): leptokurtic — impulsive vibration signals, knock events, incipient faults.
  • \(\kappa < -1\): platykurtic — bounded quantities near their limits (efficiency, normalised pressure).

Spearman vs. Pearson

Pearson \(r\) reports zero for perfectly nonlinear relationships (e.g., a U-shaped efficiency curve). Spearman rank correlation detects any monotone association and is robust to outliers:

$$\rho_{jk} = 1 - \frac{6\sum_i d_i^2}{n(n^2-1)}, \quad d_i = \mathrm{rank}(x_{ij}) - \mathrm{rank}(x_{ik})$$
Rule — Correlation method selection
For all nonlinear engineering responses (combustion, thermodynamics, kinetics): use Spearman for exploratory analysis. Use mutual information (sklearn.feature_selection.mutual_info_regression) for non-monotone features such as injection timing vs. BSFC, which has an optimum. Near-zero Pearson \(r\) with high MI is the signature of a nonlinear physical relationship.

3. Feature Engineering: Buckingham π Theorem

Feature engineering is the highest-leverage activity in engineering ML. Physical intuition and domain knowledge produce transformations that make the problem tractable with less data and better generalisation.

Theorem — Buckingham π
If a physical law involves \(n\) dimensional quantities with \(k\) independent fundamental dimensions, the relationship can be expressed as a function of \(n - k\) independent dimensionless parameters \(\Pi_1, \ldots, \Pi_{n-k}\). The surrogate function lives in an \((n-k)\)-dimensional space, not an \(n\)-dimensional one.

Example: Aerospace Compressor Map

Inputs: \(N\) [rpm], \(\dot{m}\) [kg/s], \(D\) [m], \(p_{01}\) [Pa], \(T_{01}\) [K], \(\gamma\) [-], \(R\) [J/kg/K] — 7 variables, 4 fundamental dimensions. Result: \(7 - 4 = 3\) non-trivial groups:

$$\hat{N} = \frac{ND}{\sqrt{\gamma R T_{01}}}, \qquad \hat{\dot{m}} = \frac{\dot{m}\sqrt{\gamma R T_{01}}}{p_{01}D^2}, \qquad \hat{\mathrm{Re}} = \frac{\rho_{01} N D^2}{\mu}$$

A GP trained on 2 corrected inputs instead of 7 raw ones requires \(10^5/10^2 = 1000\times\) fewer grid points to cover the same relative density in input space. This is not a heuristic — it follows from the Buckingham theorem.

Rule — Buckingham π protocol
  1. List all inputs/outputs with SI units. Write the dimensional matrix.
  2. Compute the rank \(k\) of the dimensional matrix.
  3. Select \(k\) linearly independent repeating variables (typically those spanning all dimensions).
  4. Construct \(n-k\) dimensionless \(\Pi\) groups by combining each remaining variable with the repeating variables.
  5. Verify with experiments or CFD that each \(\Pi\) group genuinely affects the output.
  6. Always train on \(\Pi\) groups when derivable; never on raw dimensional inputs.

4. Physics-Based Transformations

Arrhenius Transform

Any Arrhenius-governed quantity: \(k(T) = A\exp(-E_a/R_uT)\) becomes linear in \(1/T\) space. Use \(x_{Arr} = 1/T\) [K\(^{-1}\)] for:

  • Laminar flame speed (\(S_L \propto T_u^\alpha\), \(\alpha \approx 1.5\text{–}2.0\))
  • Ignition delay time (\(\tau_{ign} \propto \exp(E_a/R_uT)\))
  • Catalyst deactivation rate, corrosion rate, creep rate in materials

Ratio Features from Conservation Laws

  • Equivalence ratio: \(\phi = \dot{m}_f/(\dot{m}_{air}/F_{stoich})\) — encodes fuel-air balance in one dimensionless number
  • Steam-to-carbon ratio: \(S/C = \dot{n}_{H_2O}/\dot{n}_C\) — key parameter for coke deposition in steam reforming
  • Normalised temperature: \(\theta = (T - T_{ref})/T_{ref}\) — dimensionless, centred at zero, used in heat exchanger effectiveness

Product Features from Reaction Mechanisms

The Zeldovich thermal NOx rate suggests a single scalar feature encoding the dominant NOx formation mechanism:

$$\phi_{Zel} = [O_2]^{0.5} \cdot [N_2] \cdot \exp\!\left(-\frac{38{,}370}{T}\right)$$

For an ECU-based virtual sensor, \([O_2]\) and \([N_2]\) can be estimated from \(\lambda\) and the Boudouard equilibrium — computable in real time without additional sensors.

Frequency-Domain Features for Rotating Machinery

Fault frequencies are physically computable from geometry:

$$f_{BPFO} = \frac{N_b f_r}{2}\!\left(1 - \frac{d_b}{d_p}\cos\alpha\right) \quad \text{(ball pass frequency, outer race)}$$

Including spectral energy at these known physical frequencies as features provides direct diagnostic information that a convolutional network must learn from many more samples.

5. Temporal and Cycle-Based Feature Engineering

For quantities measured over a physical cycle (combustion, reactor residence time, charge-discharge):

  • Mean over cycle: removes within-cycle variability, exposes steady-state behaviour
  • Variance over cycle: quantifies cyclic dispersion; in combustion, COV\(_{IMEP} > 3\%\) indicates unstable combustion
  • Cumulative integral: \(\int_0^\tau x(t)\,dt\) — total dose (energy, reactant consumed)
  • Rate at key event: \(dx/dt\) at ignition, peak pressure, 50% MFB

Residual / Deviation Features for Condition Monitoring

$$\Delta x_j(t) = x_j(t) - x_j^{nominal}\bigl(\text{op. point}(t)\bigr)$$

This residual modelling approach: (1) reduces target variance by ~10× of the raw signal, halving required training data; (2) provides a natural fallback (zero residual = healthy); (3) is interpretable to maintenance engineers.

6. Class Imbalance in Engineering

Pitfall — Synthetic augmentation of rare operating conditions
Rare operating conditions (near-surge, near-extinction, near-failure) are rare because they are dangerous or operationally avoided. Synthetic generation of near-extinction combustion data cannot replicate the actual physics operating in these regimes — the flamelet assumption breaks down at extinction. A model trained on synthetically augmented near-extinction data will not correctly predict lean blowout.

Correct alternatives to synthetic augmentation:

  • Targeted DOE: explicitly include underrepresented operating points in the next campaign. Sobol' sensitivity identifies which regions are undersampled relative to their contribution to model variance.
  • Sample weighting: \(\mathcal{L}_{wtd} = \sum_i w_i(y_i - \hat{f}(\mathbf{x}_i))^2\) with \(w_i \propto 1/p(\mathbf{x}_i)\). Supported directly in XGBoost and PyTorch.
  • Active learning: use GP acquisition function \(\alpha_{MV} = \sigma^{*2}\) to sequentially target undersampled regions.
  • Physics-based augmentation: run the first-principles model (reduced kinetics, 0D thermodynamic model) in the underrepresented regime. This is valid because the physics model is the ground truth for that regime.

7. Outlier Detection and Data Quality

Modified Z-Score (Univariate, Robust)

$$M_i = \frac{0.6745(x_i - \tilde{x})}{\mathrm{MAD}}, \quad \mathrm{MAD} = \mathrm{median}(|x_i - \tilde{x}|)$$

Flag \(|M_i| > 3.5\). Robust to up to 50% contamination. Use before any model fitting.

Isolation Forest (Multivariate)

Anomaly score \(s(\mathbf{x},n) = 2^{-E[h(\mathbf{x})]/c(n)}\), where \(h(\mathbf{x})\) is the average path length to isolate \(\mathbf{x}\) in a random tree. \(s \to 1\) indicates anomaly. Preferred for \(d > 5\) and \(n > 1{,}000\).

Rule — Outlier handling protocol
  1. Physically impossible values (\(T < T_{amb}\), \([c] < 0\), \(\eta > 1\)): remove with documentation.
  2. Clipped sensors (\(> 2\%\) of readings at exact maximum): fix root cause; flag as MNAR.
  3. Borderline outliers (flagged but physically possible): winsorise to \([Q_{0.01}, Q_{0.99}]\).
  4. Validate: CV RMSE change < 5% after outlier removal.
  5. All removed rows logged in data audit trail with reason.

8. Correct Data Splitting for Engineering Models

Pitfall — Random shuffle for structured engineering data
The most common and most expensive error in engineering ML. Engine test data collected in back-to-back runs at similar conditions will have temporally adjacent points split into train and validation sets. The model sees "almost identical" data in both — RMSE looks great in development, fails 30–60% higher in deployment. Always ask: is there a campaign ID, test rig ID, or timestamp that should define the split?

Chapter Summary

Key takeaways
  • Audit before modelling: compute DQI, classify missing mechanisms (MCAR/MAR/MNAR), check physical bounds. A DQI < 0.90 is a hard stop.
  • Buckingham π first: always derive dimensionless groups before training. This is not optional — it is the single highest-leverage activity available. Reduces data requirements by 3–10×.
  • Arrhenius transform \(1/T\): mandatory for any thermally activated process. Linearises the dominant exponential dependence.
  • Spearman + mutual information: for exploratory correlation analysis. Pearson misses all nonlinear relationships.
  • Imbalance → targeted DOE: not SMOTE, not random oversampling. Rare engineering conditions require real data from those conditions.
  • GroupKFold: the default split for engineering data. Random KFold on batch/temporal data is a systematic error that produces optimistic RMSE by 20–60%.
