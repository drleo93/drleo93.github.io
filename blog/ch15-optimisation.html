TL;DR

  • Bayesian optimisation with Expected Improvement is the gold standard for <200 expensive evaluations in $d \leq 20$ dimensions. Beyond that, use TuRBO.
  • Gradient-based optimisation is 100× faster per iteration than gradient-free — but requires smooth, differentiable objectives. Adjoint methods provide exact gradients at the cost of one additional solve, independent of $d$.
  • Multi-objective optimisation finds a Pareto front, not a single solution. Always present the full trade-off curve — picking a single "optimal" point destroys information.
  • Constrained BO: model each constraint with a separate GP. Use $\text{cEI}(\mathbf{x}) = \text{EI}(\mathbf{x}) \times \prod_k \text{PoF}_k(\mathbf{x})$.
  • For noisy test-bench data: Knowledge Gradient outperforms EI because it measures expected improvement in the global optimum estimate, not just at the queried point.

2. Bayesian Optimisation: Expected Improvement

BO replaces expensive objective evaluations with a cheap GP surrogate and queries the next point by maximising an acquisition function. Given current best $f^* = \min_i y_i$ and GP posterior $(\mu^*, \sigma^*)$:

$$\text{EI}(\mathbf{x}) = (f^* - \mu^*)\Phi(Z) + \sigma^*\phi(Z), \quad Z = \frac{f^* - \mu^*(\mathbf{x})}{\sigma^*(\mathbf{x})}$$

where $\Phi$, $\phi$ are the standard normal CDF and PDF. EI = 0 when $\sigma^* = 0$ (already evaluated) and is maximised by balancing exploitation ($f^* > \mu^*$: region where the GP predicts improvement) and exploration (large $\sigma^*$: high uncertainty).

Knowledge Gradient for Noisy Evaluations

$$\text{KG}(\mathbf{x}) = \mathbb{E}\bigl[\min_{\mathbf{x}'}\mu_{n+1}(\mathbf{x}') - \min_{\mathbf{x}'}\mu_n(\mathbf{x}') \bigm| \mathbf{x}_{n+1}=\mathbf{x}\bigr]$$

KG measures the expected improvement in the global optimum estimate, not just at $\mathbf{x}$. More suitable for noisy test-bench data where EI is distracted by noise-induced false optima.

3. TuRBO: Trust-Region BO for High Dimensions

Global GPs become unreliable in $d > 20$: the kernel matrix is poorly conditioned and the acquisition function landscape has exponentially many local optima. TuRBO maintains a set of local trust regions $\{\mathcal{T}_k\}$, each with its own GP:

$$\mathbf{x}_{next} = \arg\min_{\mathbf{x} \in \mathcal{T}_k} f_k(\mathbf{x}), \quad f_k \sim \mathcal{GP}_k$$ $$L_{k+1} = \begin{cases} \min(2L_k, L_{max}) & \text{if } n_{succ} \geq \tau_{succ} \\ L_k/2 & \text{if } n_{fail} \geq \tau_{fail} \\ L_k & \text{otherwise}\end{cases}$$

Trust regions expand on improvement, contract on failure. Each region uses Thompson sampling (draw a sample from the GP posterior, minimise it) rather than EI — computationally cheaper for many restarts.

4. Gradient-Based Optimisation Through Surrogates

If the surrogate $\hat{f}$ is implemented in PyTorch or JAX, gradients are free via automatic differentiation:

import torch
from scipy.optimize import minimize

def gradient_opt_through_gp(model, scaler, bounds_phys, n_restarts=20):
    """L-BFGS-B minimisation of GP posterior mean using TF autodiff."""
    import tensorflow as tf
    d = bounds_phys.shape[0]
    best_val, best_x = float('inf'), None

    for _ in range(n_restarts):
        x0 = np.random.uniform(0, 1, size=d)

        def objective_and_grad(x_np):
            x_tf = tf.Variable(x_np[None, :], dtype=tf.float64)
            with tf.GradientTape() as tape:
                mu, _ = model.predict_y(x_tf)
                obj   = tf.squeeze(mu)
            grad = tape.gradient(obj, x_tf)
            return obj.numpy(), grad.numpy()[0]

        res = minimize(objective_and_grad, x0, method='L-BFGS-B',
                       jac=True, bounds=[(0., 1.)]*d,
                       options={'maxiter': 200, 'ftol': 1e-12})
        if res.fun < best_val:
            best_val, best_x = res.fun, res.x

    return scaler.inverse_transform(best_x[None, :])[0], best_val

5. Adjoint-Based Optimisation

For PDE-constrained problems (CFD, FEM), the adjoint method computes the gradient of a scalar objective $J$ with respect to $d$ design parameters at the cost of one additional linear solve — regardless of $d$:

The Adjoint Method: Let $\mathcal{R}(\mathbf{u}, \mathbf{x}) = 0$ be the state equations (Navier-Stokes, heat equation). $$\mathcal{L} = J(\mathbf{u}, \mathbf{x}) + \boldsymbol{\lambda}^\top \mathcal{R}(\mathbf{u}, \mathbf{x})$$ Setting $\partial \mathcal{L}/\partial \mathbf{u} = 0$ gives the adjoint equation: $$\left(\frac{\partial \mathcal{R}}{\partial \mathbf{u}}\right)^\top \boldsymbol{\lambda} = -\frac{\partial J}{\partial \mathbf{u}}$$ The gradient then costs only one additional linear solve: $$\frac{dJ}{d\mathbf{x}} = \frac{\partial J}{\partial \mathbf{x}} + \boldsymbol{\lambda}^\top \frac{\partial \mathcal{R}}{\partial \mathbf{x}} \quad \text{(cost: } \mathcal{O}(d^0) \text{ vs finite diff } \mathcal{O}(d)\text{)}$$
Example — Turbine Blade Aerodynamic Optimisation

State-of-the-art turbine blade optimisation uses adjoint-based gradient descent:

  1. Parameterise blade geometry with $d = 20$–100 design variables (PARSEC, CST, or B-spline control points).
  2. Run RANS CFD to compute total-to-total efficiency $\eta_{tt}$ and mass-flow-averaged loss $\omega$.
  3. Run adjoint RANS to compute $d\eta_{tt}/dx_i$ for all $i$ simultaneously.
  4. Step in the gradient direction with L-BFGS-B, subject to structural constraints.

Result: 20% drag reduction on a high-pressure turbine NGV in <50 CFD evaluations. Gradient-free NSGA-II on the same problem required >5,000 evaluations.

6. Constrained Bayesian Optimisation

Engineering problems always have constraints. For output constraints (constraint value computed by the expensive simulation), model each constraint with a separate GP and use the probability of feasibility:

$$\text{PoF}(\mathbf{x}) = P\bigl(g(\mathbf{x}) \leq 0\bigr) = \Phi\!\left(\frac{-\mu_g^*(\mathbf{x})}{\sigma_g^*(\mathbf{x})}\right)$$ $$\text{Constrained EI: } \text{cEI}(\mathbf{x}) = \text{EI}(\mathbf{x}) \times \prod_k \text{PoF}_k(\mathbf{x})$$ 
from botorch.models import SingleTaskGP
from botorch.acquisition.logei import qLogNoisyExpectedImprovement
from botorch.optim import optimize_acqf
import torch

def constrained_bo_step(train_X, train_Y, train_C, bounds, n_candidates=256):
    """One step of constrained BO with m output constraints (<=0 is feasible)."""
    obj_model = SingleTaskGP(train_X, train_Y)
    # Separate GP per constraint
    con_models = [SingleTaskGP(train_X, train_C[:, i:i+1])
                  for i in range(train_C.shape[1])]

    constraints = {i: (None, 0.0) for i in range(train_C.shape[1])}
    acqf = qLogNoisyExpectedImprovement(
        model=obj_model, X_baseline=train_X, constraints=constraints
    )
    candidate, _ = optimize_acqf(
        acqf, bounds=bounds, q=1, num_restarts=20, raw_samples=n_candidates
    )
    return candidate
Pitfall — Ignoring Constraint Uncertainty
Using hard constraint enforcement (reject infeasible points) instead of probabilistic feasibility wastes the BO budget by refusing to explore near constraint boundaries — where the best feasible solutions often lie. Always use PoF-weighted acquisition. Set PoF threshold at 0.9 (not 0.5) for engineering safety constraints.

7. Multi-Objective Optimisation: Pareto Fronts

A solution $\mathbf{x}^*$ is Pareto optimal if no other feasible $\mathbf{x}'$ is better in at least one objective without being worse in another. The Pareto front $\mathcal{P}^*$ is the set of all such solutions.

$$\mathbf{x}^* \text{ is Pareto optimal} \iff \nexists\, \mathbf{x}' : f_k(\mathbf{x}') \leq f_k(\mathbf{x}^*)\;\forall k, \quad \exists k': f_{k'}(\mathbf{x}') < f_{k'}(\mathbf{x}^*)$$

qEHVI (Expected Hypervolume Improvement): the BO acquisition function for multi-objective problems. It measures the expected increase in the dominated hypervolume — the volume of objective space dominated by the current Pareto front. Implemented in BoTorch as qExpectedHypervolumeImprovement.

Lesson
Presenting the full Pareto front to decision-makers is almost always more valuable than a single "optimal" point. A BSFC vs. NOx Pareto front for an engine calibration problem might show that a 5% BSFC penalty reduces NOx by 40% — a trade-off that cannot be captured by any single-objective formulation. The engineering decision of which Pareto point to select belongs to the system engineer, not the optimiser.
Multi-Objective Best Practices
  • Normalise all objectives to [0, 1] before computing hypervolume — EHVI is scale-sensitive.
  • Set the reference point slightly worse than the worst observed objective values (e.g., $r_k = \max_i f_k(x_i) \times 1.1$).
  • For $> 3$ objectives, hypervolume computation becomes expensive — use R2 indicator or decomposition-based methods (MOEA/D).
  • NSGA-II with surrogate assistance: use the GP surrogate to pre-screen candidates before expensive CFD evaluation. Reduces budget by 10× vs. direct NSGA-II on CFD.

8. Key Takeaways

  • BO with EI: gold standard for $d \leq 20$, $n \leq 200$ expensive evaluations. Use Knowledge Gradient for noisy test benches.
  • TuRBO: mandatory for $d > 20$. Fits local GPs in trust regions that expand/contract based on improvement history.
  • Adjoint method: compute exact gradients of PDE-constrained objectives at $\mathcal{O}(1)$ extra solve cost. 100× more efficient than finite differences for $d > 10$.
  • Constrained BO: model each constraint with its own GP; weight EI by $\text{PoF}(\mathbf{x})$. Never use hard rejection.
  • Multi-objective: always present the full Pareto front. qEHVI (BoTorch) is the correct acquisition for BO-based Pareto front generation.
  • Gradient-free methods (CMA-ES, NSGA-II) need $\mathcal{O}(10^3\text{--}10^4)$ evaluations vs. $\mathcal{O}(10\text{--}100)$ for gradient-based. Never use gradient-free when gradients are available.
