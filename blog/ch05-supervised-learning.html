Chapter 5 of 23  ·  ~40 min read  ·  Master the model family, not the problem type

TL;DR

  • Beat the baseline (OLS then default XGBoost for regression; calibrated logistic regression for classification) before considering any complex model.
  • GP (Matérn-5/2 ARD) is the default for \(n < 1{,}000\) samples and structured physical responses. Exact GP costs \(\mathcal{O}(n^3)\) — use sparse inducing points for \(n > 5{,}000\).
  • XGBoost with monotone constraints enforces physical monotonicity at zero additional data cost. A 5–15% RMSE increase on training data often yields no test penalty.
  • PCE gives Sobol' sensitivity indices for free from expansion coefficients. Use it when input distributions are known and the goal is uncertainty propagation.
  • ARD kernel length-scales are a built-in global sensitivity analysis. Large \(\ell_j\) → feature \(j\) is irrelevant.
  • Random Forest OOB error gives a free cross-validation estimate — no need for a separate validation set with RF.

1. Statistical Learning Framework

Given \(\mathcal{D} = \{(\mathbf{x}_i, y_i)\}_{i=1}^n\), find \(\hat{f}: \mathcal{X} \to \mathbb{R}\) minimising expected prediction error on unseen data:

$$R[\hat{f}] = \underbrace{\bigl[\mathbb{E}_\mathcal{D}[\hat{f}(\mathbf{x})] - f(\mathbf{x})\bigr]^2}_{\text{Bias}^2} + \underbrace{\mathbb{E}_\mathcal{D}\bigl[(\hat{f}(\mathbf{x}) - \mathbb{E}_\mathcal{D}[\hat{f}])^2\bigr]}_{\text{Variance}} + \underbrace{\sigma_\varepsilon^2}_{\text{Irreducible}}$$

In engineering practice with small \(n\), the optimum leans toward higher bias (simpler models, stronger regularisation) because variance reduction from additional data is often impractical.

2. Response Surface Methods (RSM)

The second-order RSM is the workhorse of classical DOE analysis:

$$\hat{y}(\mathbf{x}) = \beta_0 + \sum_{j=1}^d \beta_j x_j + \sum_{j=1}^d \beta_{jj} x_j^2 + \sum_{j < k} \beta_{jk} x_j x_k$$

OLS solution: \(\hat{\boldsymbol{\beta}} = (\mathbf{X}_p^\top\mathbf{X}_p)^{-1}\mathbf{X}_p^\top\mathbf{y}\). Ridge (L2): \(\hat{\boldsymbol{\beta}}_{ridge} = (\mathbf{X}_p^\top\mathbf{X}_p + \lambda\mathbf{I})^{-1}\mathbf{X}_p^\top\mathbf{y}\).

Rule — When to use RSM
  • \(d \leq 6\) inputs; \(n \geq 3p\) where \(p = 1 + 2d + d(d-1)/2\) is the number of coefficients
  • Response is smooth and approximately quadratic (verify with lack-of-fit test)
  • Regulatory deliverable requires an interpretable equation-form surrogate (e.g., EGR controller map)
  • Never use RSM for \(d > 8\): the \(\binom{d}{2}\) interaction terms explode parameter count

3. Decision Trees and Random Forests

CART Algorithm

A regression tree partitions the input space into \(M\) rectangular regions \(\{R_m\}\) and predicts the constant \(c_m = \text{mean}(y_i : \mathbf{x}_i \in R_m)\). Splits are chosen greedily:

$$(j^*, t^*) = \arg\min_{j,t} \left[\frac{|R_L|}{n}\mathrm{Var}(y|R_L) + \frac{|R_R|}{n}\mathrm{Var}(y|R_R)\right]$$

A single tree is interpretable but has high variance. The remedy is ensembling.

Random Forest

$$\hat{f}_{RF}(\mathbf{x}) = \frac{1}{B}\sum_{b=1}^B \hat{f}^{(b)}(\mathbf{x})$$

Each tree is trained on a bootstrap sample with a random subset \(m_{try} \leq d\) features per split. Averaging reduces variance by \(\approx 1/B\) while keeping bias equal to one tree.

Out-of-Bag (OOB) error: each bootstrap uses ≈63.2% of observations. The remaining 36.8% provide a free internal cross-validation estimate — no separate validation set needed.

Lesson — Feature importance: MDI vs. permutation
Mean Decrease in Impurity (MDI) is fast but biased toward high-cardinality features (e.g., continuous numerical features over binary indicators). Permutation importance (MDA) is unbiased: perturb feature \(j\), measure RMSE increase. Preferred for engineering feature selection. In scikit-learn: permutation_importance(rf, X_test, y_test, n_repeats=20).
Example — CPOX reactor conversion (RF, n=1247)
5 features: \(T\) [K], \(\phi\) [-], GHSV [h⁻¹], O₂/CH₄ [-], \(\Delta p\) [Pa]. RF with \(B=500\), \(m_{try}=2\), min_samples_leaf=3: OOB RMSE = 1.3% conversion. Permutation importance: \(T\) dominates (\(\Delta\)RMSE = +6.8%), confirming Arrhenius temperature sensitivity. GHSV has minimal individual importance (+0.3%) but strong interaction with \(T\) — detected by RF, missed by the linear model.

4. Gradient Boosting: XGBoost

XGBoost builds an additive ensemble by fitting each new tree to the negative gradient (pseudo-residuals) of the loss, using a second-order Taylor expansion:

$$r_{im} = -\left[\frac{\partial \mathcal{L}(y_i, F(\mathbf{x}_i))}{\partial F(\mathbf{x}_i)}\right]_{F=F_{m-1}}$$

The regularised objective includes a leaf-count penalty \(\gamma T_m\) and L2 leaf weight penalty \(\frac{1}{2}\lambda\|\mathbf{w}_m\|^2\).

Lesson — Monotone constraints for physical compliance
XGBoost natively enforces monotone constraints: monotone_constraints=(1, 0, -1, 0) enforces \(\partial\hat{f}/\partial x_1 \geq 0\) (efficiency increases with pressure ratio) and \(\partial\hat{f}/\partial x_3 \leq 0\) (NOx decreases with EGR). Cost: typically 5–15% RMSE increase on training; often no test penalty (constraints prevent spurious training-set patterns from being learned).

5. Gaussian Process Regression

A GP \(f \sim \mathcal{GP}(m(\mathbf{x}),\, k(\mathbf{x},\mathbf{x}'))\) is a distribution over functions. The posterior given observations \(\mathbf{y} = f(\mathbf{X}) + \boldsymbol{\varepsilon}\), \(\boldsymbol{\varepsilon} \sim \mathcal{N}(\mathbf{0},\sigma_n^2\mathbf{I})\):

$$\mu^*(\mathbf{x}^*) = \mathbf{k}_*^\top(\mathbf{K} + \sigma_n^2\mathbf{I})^{-1}\mathbf{y}, \qquad \sigma^{*2}(\mathbf{x}^*) = k_{**} - \mathbf{k}_*^\top(\mathbf{K} + \sigma_n^2\mathbf{I})^{-1}\mathbf{k}_*$$

Standard Kernels

ARD: Automatic Relevance Determination

Replace scalar \(\ell\) with per-feature vector \(\boldsymbol{\ell} = (\ell_1,\ldots,\ell_d)^\top\):

$$k_{ARD}(\mathbf{x},\mathbf{x}') = \sigma_f^2 \exp\!\left(-\sum_{j=1}^d \frac{(x_j-x'_j)^2}{2\ell_j^2}\right)$$

Small \(\ell_j\) → function varies rapidly, feature \(j\) is important. Large \(\ell_j\) → nearly constant, feature \(j\) is irrelevant. If \(\ell_j > 5\times\text{range}(x_j)\), confirm with physics that the feature is physically irrelevant. ARD is a built-in global sensitivity analysis.

Hyperparameter Optimisation via Log Marginal Likelihood

$$\log p(\mathbf{y}|\mathbf{X},\boldsymbol{\theta}) = -\frac{1}{2}\mathbf{y}^\top\mathbf{C}^{-1}\mathbf{y} - \frac{1}{2}\log|\mathbf{C}| - \frac{n}{2}\log 2\pi$$

The LML balances data fit (first term) against model complexity (second term) — automatic Occam's razor. Run ≥10 random restarts (the LML landscape is multimodal for ARD kernels).

Rule — Computational complexity
Exact GP: \(\mathcal{O}(n^3)\) training, \(\mathcal{O}(n^2)\) prediction. For \(n > 5{,}000\): use sparse approximations with inducing points, reducing training to \(\mathcal{O}(nm^2)\) where \(m = 100\text{–}500 \ll n\). Recommended toolboxes: SMT 2.0 (aerospace, gradient-enhanced Kriging), OpenTURNS (French aerospace standard, UQ toolchain), GPflow/GPyTorch (GPU, sparse, custom kernels).

6. Polynomial Chaos Expansion (PCE)

PCE expresses the model output as a series of orthogonal polynomials in the uncertain inputs:

$$Y \approx \sum_{\boldsymbol{\alpha} \in \mathcal{A}} c_{\boldsymbol{\alpha}}\,\Psi_{\boldsymbol{\alpha}}(\mathbf{Z})$$

Basis polynomials \(\psi_k^{(j)}\) are chosen orthogonal to the marginal distribution of \(Z_j\): Hermite for Gaussian, Legendre for Uniform, Laguerre for Gamma.

Sobol' indices from PCE — for orthonormal bases, free from expansion coefficients:

$$S_i = \frac{\sum_{\boldsymbol{\alpha} \in \mathcal{A}_i} c_{\boldsymbol{\alpha}}^2}{\mathrm{Var}(Y)}, \quad \mathcal{A}_i = \{\boldsymbol{\alpha} : \alpha_i > 0,\, \alpha_j = 0\,\forall j \neq i\}$$
Example — CSTR temperature UQ with sparse PCE
Uncertain inputs: \(T_f \sim \mathcal{N}(350, 5^2)\) K, \(A_0 \sim \mathrm{LogNormal}\), \(q_c \sim \mathcal{N}(0.5, 0.05^2)\) m³/s. Sparse PCE degree 4, trained on 200 LHS evaluations. Sobol' indices: \(S_{T_f}=0.62\), \(S_{A_0}=0.24\), \(S_{q_c}=0.08\). PCE Monte Carlo (\(N=10^6\), <0.1 s): \(\mathrm{Std}(T_{out}) = 3.8\) K. Speedup vs. ODE solver Monte Carlo: >\(10^6\times\).
Rule — PCE vs. GP
Use PCE when: input distributions are known; goal is uncertainty propagation or reliability analysis; Sobol' indices needed for free. Use GP when: UQ of predictions required (posterior variance); active learning; exact interpolation. For \(d > 5\): use sparse PCE (LARS algorithm) — full expansion \(|\mathcal{A}| = \binom{d+p}{d}\) grows exponentially.

7. Classification for Engineering

Rule — Classification evaluation for engineering
  • Never use accuracy alone for imbalanced classes (fault detection: 0.1% fault rate → 99.9% accuracy by predicting "no fault" always)
  • For fault detection: set Recall ≥ 0.95 (missing a fault is catastrophic), then tune threshold. AUC summarises discrimination ability independently of threshold.
  • For false-alarm-sensitive systems: maximise Precision at a given Recall threshold.
  • Calibrate predicted probabilities: a fault detector saying "80% probability of surge" must be correct 80% of the time.
  • Baseline: calibrated logistic regression. Beat it before considering any complex classifier.

Chapter Summary

Key takeaways
  • Baseline first: OLS → XGBoost → GP is the correct evaluation order for regression. Logistic regression → XGBoost is the correct order for classification.
  • GP Matérn-5/2 ARD is the default for \(n < 1{,}000\). ARD length-scales directly give feature relevance. LML gives automatic Occam's razor.
  • XGBoost monotone constraints enforce physical compliance at no data cost. Use them whenever you know the sign of a partial derivative.
  • PCE = free Sobol' indices. When input distributions are known, PCE is the surrogate choice for UQ and reliability analysis.
  • RF OOB error is free cross-validation. Use it to diagnose overfitting without a separate validation set.
  • Permutation importance, not MDI, for feature selection — MDI is biased toward high-cardinality features.
