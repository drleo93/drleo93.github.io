TL;DR

  • Test for stationarity (ADF) before any time series modelling. Non-stationarity types: trend, seasonality, regime switching, concept drift — each requires a different fix.
  • Always split at the series level before windowing. Window-level random splits create temporal leakage and produce overoptimistic CV RMSE by 30–60%.
  • The Kalman filter is the optimal estimator for linear-Gaussian state-space models. Use it for battery SoC estimation, thermal state estimation, sensor fusion.
  • ACF identifies MA order; PACF identifies AR order. Check residuals with Ljung-Box (\(p > 0.05\)) after fitting ARIMA.
  • Welch PSD with Hann window: frequency resolution \(\Delta f = f_s/L\); use \(L = 8\)–\(16 \times 1/f_{min}\).
  • STL decomposition (Loess-based) is the recommended method for trend-seasonality decomposition — robust to outliers, handles any seasonality type.

1. Engineering Time Series Characteristics

Engineering time series differ from financial or climatological ones in ways that matter for preprocessing:

  • Multi-rate: ECU at 100 Hz, thermocouples at 1 Hz, GPS at 10 Hz — synchronisation is mandatory before any analysis
  • Physically bounded: concentrations ≥ 0, efficiency ∈ [0,1], SoC ∈ [0,1] — violations indicate sensor faults
  • Operating-mode switching: drive cycles switch between idle, acceleration, cruise — piecewise stationary, not globally stationary
  • Slowly sampled outputs: PEMS emission at ~5 Hz vs. engine control at 200 Hz — requires up-/downsampling strategies

2. Stationarity Testing and Preprocessing

A process \(\{X_t\}\) is weakly stationary if: \(\mathbb{E}[X_t] = \mu\) for all \(t\), and \(\mathrm{Cov}(X_t, X_{t+h}) = \gamma(h)\) for all \(t, h\). Most classical time series models require stationarity.

Augmented Dickey-Fuller (ADF) test: \(H_0\) = unit root (non-stationary). Reject at \(p < 0.05\) to assert stationarity. In Python: statsmodels.tsa.stattools.adfuller(series).

3. STL Decomposition

$$y_t = T_t + S_t + R_t \quad \text{(additive)}, \qquad \text{or} \qquad y_t = T_t \cdot S_t \cdot R_t \quad \text{(multiplicative)}$$

STL (Seasonal-Trend decomposition using Loess) is the recommended method: handles any seasonality type, robust to outliers, works for complex changing seasonal patterns. statsmodels.tsa.seasonal.STL.

Engineering applications: catalyst activity (\(T_t\) = long-term deactivation, \(S_t\) = daily regeneration, \(R_t\) = unexpected fouling); heat exchanger temperature (\(T_t\) = gradual fouling, \(S_t\) = diurnal cooling water variation).

4. ACF, PACF, and Welch PSD

Autocorrelation and Partial Autocorrelation

$$\hat{\rho}(h) = \frac{\sum_{t=1}^{n-h}(x_t-\bar{x})(x_{t+h}-\bar{x})}{\sum_{t=1}^n(x_t-\bar{x})^2}$$

Bartlett confidence bands: \(\pm 1.96/\sqrt{n}\). ACF identifies MA order \(q\) (cuts off at lag \(q\)); PACF identifies AR order \(p\) (cuts off at lag \(p\)).

Engineering readings: in-cylinder pressure ACF shows strong autocorrelation at the firing period \(T_{fire} = 2/N_{rpm}\) and harmonics — engine healthy. Vibration signal from a healthy bearing: ACF near zero for all \(h > 0\). ACF non-zero at lag \(1/f_{BPFO}\) → early bearing fault.

Welch PSD

$$\hat{S}_{xx}(f) = \frac{1}{KL}\sum_{k=0}^{K-1}\left|\sum_{n=0}^{L-1} x[n+kD]\, w[n]\, e^{-j2\pi fn/f_s}\right|^2$$

Frequency resolution: \(\Delta f = f_s/L\). Use Hann window \(w[n]\). For a 4-stroke engine at speed \(N\) [rpm]: firing frequency \(f_{fire} = N/120\) [Hz] dominates in-cylinder pressure PSD. Broad-band energy increase at 2–5 kHz in bearing vibration PSD indicates distributed surface damage; a sharp spike at \(f_{BSF}\) indicates a ball defect.

5. Classical Time Series Models

ARIMA

$$\Phi_p(B)(1-B)^d X_t = \Theta_q(B)\varepsilon_t, \quad \varepsilon_t \sim \mathcal{N}(0,\sigma^2)$$

Box-Jenkins identification procedure:

  1. Test stationarity (ADF); apply differencing if non-stationary
  2. Use ACF to identify MA order \(q\) (cuts off at lag \(q\))
  3. Use PACF to identify AR order \(p\) (cuts off at lag \(p\))
  4. Estimate parameters by MLE
  5. Validate: residuals pass Ljung-Box test (\(p > 0.05\))

Use for univariate process control signals (reactor temperature, column pressure) where the autoregressive structure is physically motivated. For multivariate systems with known inputs: use ARIMAX or VAR.

Kalman Filter for State Estimation

Linear-Gaussian state-space model:

$$\mathbf{x}_{t+1} = \mathbf{A}\mathbf{x}_t + \mathbf{B}\mathbf{u}_t + \mathbf{w}_t, \quad \mathbf{w}_t \sim \mathcal{N}(\mathbf{0},\mathbf{Q})$$ $$\mathbf{y}_t = \mathbf{C}\mathbf{x}_t + \mathbf{v}_t, \quad \mathbf{v}_t \sim \mathcal{N}(\mathbf{0},\mathbf{R})$$

Prediction step: \(\hat{\mathbf{x}}_{t|t-1} = \mathbf{A}\hat{\mathbf{x}}_{t-1|t-1}\), \(\mathbf{P}_{t|t-1} = \mathbf{A}\mathbf{P}_{t-1|t-1}\mathbf{A}^\top + \mathbf{Q}\)

Update step (Kalman gain \(\mathbf{K}_t\)):

$$\mathbf{K}_t = \mathbf{P}_{t|t-1}\mathbf{C}^\top(\mathbf{C}\mathbf{P}_{t|t-1}\mathbf{C}^\top + \mathbf{R})^{-1}$$ $$\hat{\mathbf{x}}_{t|t} = \hat{\mathbf{x}}_{t|t-1} + \mathbf{K}_t(\mathbf{y}_t - \mathbf{C}\hat{\mathbf{x}}_{t|t-1})$$
Example — Battery SoC estimation (EKF)
For a lithium-ion cell with equivalent circuit model: state \(\mathbf{x} = [\text{SoC}, V_{RC1}, V_{RC2}]^\top\). Observation: terminal voltage \(V_{term}\). The Kalman filter fuses coulomb-counting (prediction step) with terminal voltage measurement (update step), correcting for measurement noise and current integration drift simultaneously. EKF linearises the nonlinear OCV–SoC curve at each time step.

VAR for Multivariate Systems

$$\mathbf{Y}_t = \mathbf{c} + \mathbf{A}_1\mathbf{Y}_{t-1} + \cdots + \mathbf{A}_p\mathbf{Y}_{t-p} + \boldsymbol{\varepsilon}_t$$

Each variable's forecast depends on its own lags and the lags of all other variables. Lag order selection via BIC (parsimony — preferred for engineering where overfitting to noise is costly). Use for multivariate process control signals where cross-variable dynamics matter.

7. Windowing for Supervised Learning — Temporal Leakage

Pitfall — Window-level random splits
For sliding windows of length \(W\) and hop \(H < W\), consecutive windows overlap by \(W - H\) timesteps. Random train/test split at the window level means training and test windows share timesteps — the test set is not independent. This produces CV RMSE estimates that are 30–60% lower than deployment performance. 
# CORRECT: split at the series level FIRST, then window
n_train = int(0.70 * T_total)
n_val   = int(0.85 * T_total)

X_tr, y_tr = sliding_windows(series[:n_train],     W=50, H=1, horizon=1)
X_va, y_va = sliding_windows(series[n_train:n_val], W=50, H=1, horizon=1)
X_te, y_te = sliding_windows(series[n_val:],        W=50, H=1, horizon=1)

Chapter Summary

Key takeaways
  • Test stationarity first: ADF test before any model fitting. Address trend, seasonality, and regime switching explicitly — each requires a different transformation.
  • ACF → MA order; PACF → AR order. Validate residuals with Ljung-Box after ARIMA fitting.
  • Kalman filter: optimal for linear-Gaussian state-space. Correct framework for battery SoC, thermal state, and sensor fusion. EKF for mildly nonlinear systems.
  • Split at series level before windowing: window-level splits are a systematic error that produces overoptimistic results.
  • STL decomposition: the default for trend-seasonality decomposition in engineering time series — robust, flexible, outlier-resistant.
  • For deep learning time series models (LSTM, TCN, Transformers): see Ch. 10.
