TL;DR

  • Test for stationarity (ADF) before any modelling. Engineering non-stationarity comes in four types: trend, seasonality, regime switching, and concept drift — each requires a different fix.
  • Always split at the series level before windowing. Window-level random splits create temporal leakage and produce RMSE estimates 30–60% lower than deployment performance.
  • Welch PSD for stationary broad-band analysis. STFT for non-stationary frequency-tracking. Wavelet transform for transient event detection with variable time-frequency resolution.
  • For rotating machinery: use order analysis (angle-domain resampling) — speed-normalised spectra that track harmonics regardless of RPM variation.
  • The Kalman filter is the optimal estimator for linear-Gaussian systems. Use EKF for mildly nonlinear systems (battery SoC, thermal state estimation, sensor fusion).
  • Use Granger causality to identify which signals to include in a VAR model — not mutual information alone (MI ignores temporal directionality).
  • ACF identifies MA order; PACF identifies AR order. Always validate residuals with Ljung-Box ($p > 0.05$). Use BIC for order selection, not AIC (AIC over-selects lags).

2. Stationarity: Testing, Types, and Fixes

A process $\{X_t\}$ is weakly stationary if its mean and covariance are time-invariant:

$$\mathbb{E}[X_t] = \mu \quad \forall t, \qquad \mathrm{Cov}(X_t, X_{t+h}) = \gamma(h) \quad \forall t, h$$

Classical models (ARIMA, VAR, Kalman) require stationarity. Violation leads to spurious correlations, incorrect significance tests, and non-convergent parameter estimates.

2.1 Augmented Dickey-Fuller (ADF) Test

$H_0$: unit root (non-stationary). Reject at $p < 0.05$ to assert stationarity. The ADF regression is:

$$\Delta X_t = \alpha + \beta t + \gamma X_{t-1} + \sum_{j=1}^p \delta_j \Delta X_{t-j} + \varepsilon_t$$ $$H_0: \gamma = 0 \text{ (unit root)} \qquad H_1: \gamma < 0 \text{ (stationary)}$$ 
from statsmodels.tsa.stattools import adfuller, kpss

def stationarity_report(series, name="series"):
    """Run ADF and KPSS tests and print interpretation."""
    adf_stat, adf_p, _, _, adf_crit, _ = adfuller(series, autolag='BIC')
    kpss_stat, kpss_p, _, kpss_crit = kpss(series, regression='c', nlags='auto')

    print(f"\n=== Stationarity: {name} ===")
    print(f"ADF:  stat={adf_stat:.4f}, p={adf_p:.4f}  {'→ STATIONARY' if adf_p<0.05 else '→ NON-STATIONARY'}")
    print(f"KPSS: stat={kpss_stat:.4f}, p={kpss_p:.4f}  {'→ NON-STATIONARY' if kpss_p<0.05 else '→ STATIONARY'}")
    # Agree: both consistent → strong evidence
    # Disagree: trend-stationary (ADF rejects, KPSS rejects) → check visually
Tip — Use Both ADF and KPSS Together
ADF tests for unit root (null: non-stationary); KPSS tests for stationarity (null: stationary). Use both:
  • ADF: p < 0.05, KPSS: p > 0.05 → stationary
  • ADF: p > 0.05, KPSS: p < 0.05 → unit root non-stationary — first-difference
  • Both reject → trend-stationary — detrend (HP filter or deterministic time trend)
  • Neither rejects → insufficient data, increase sample size

3. Multi-Rate Data and Resampling

Engineering data systems routinely log signals at different sampling rates. Synchronisation before any analysis is not optional — mismatched timestamps corrupt cross-correlation, Granger causality, and model training.

3.1 Upsampling (Interpolation)

Increasing sample rate of a slow signal to match a fast one. Methods by context:

  • Zero-order hold: repeat the last value until next sample. Appropriate for discrete setpoint signals (injection timing, valve position). Computationally trivial.
  • Linear interpolation: physically motivated for slowly changing continuous signals (temperature, pressure). Introduces no Gibbs artifacts.
  • Sinc interpolation: optimal (Nyquist-Shannon) for band-limited signals. Expensive but correct for vibration signals being upsampled before spectral analysis.

3.2 Downsampling (Decimation)

Reducing sample rate of a fast signal to match a slow one. Always apply anti-aliasing low-pass filter before downsampling at the Nyquist frequency of the target rate $f_s^{target}/2$:

import numpy as np
from scipy.signal import resample_poly, butter, filtfilt

def downsample_safe(signal, fs_original, fs_target):
    """Downsample with anti-aliasing Butterworth filter."""
    nyq = fs_target / 2
    b, a = butter(8, nyq / (fs_original / 2), btype='low')
    filtered = filtfilt(b, a, signal)
    ratio = int(fs_original / fs_target)
    return filtered[::ratio]

# Multi-rate synchronisation
import pandas as pd
fast = pd.Series(vib_200hz, index=pd.to_timedelta(np.arange(len(vib_200hz))/200, unit='s'))
slow = pd.Series(temp_1hz,  index=pd.to_timedelta(np.arange(len(temp_1hz)), unit='s'))

# Resample both to 10 Hz
fast_10 = fast.resample('100ms').mean()   # decimate with averaging = anti-aliasing
slow_10 = slow.resample('100ms').ffill()  # zero-order hold for slow signals
df = pd.concat([fast_10, slow_10], axis=1).dropna()

4. Trend-Seasonality Decomposition: STL and MSTL

4.1 Additive vs Multiplicative Decomposition

$$\text{Additive: } y_t = T_t + S_t + R_t \qquad \text{Multiplicative: } y_t = T_t \cdot S_t \cdot R_t$$

Use additive when seasonal fluctuation is roughly constant in amplitude regardless of trend level (e.g., temperature, pressure). Use multiplicative when seasonal amplitude scales with the level (e.g., power consumption, flow rate where percentage variation is constant).

4.2 STL (Seasonal-Trend Decomposition Using Loess)

STL iteratively applies Loess (locally weighted regression) to separate trend $T_t$, seasonality $S_t$, and remainder $R_t$. Advantages over classical decomposition and X-12-ARIMA:

  • Handles any seasonality type (daily, weekly, annual, arbitrary)
  • Robust to outliers (the IRLS weights in Loess downweight outliers)
  • Seasonal component can change slowly over time (important for long degradation studies)
from statsmodels.tsa.seasonal import STL, MSTL
import matplotlib.pyplot as plt

# STL for single seasonality
stl = STL(y, period=24, robust=True)   # e.g., hourly data, 24h seasonality
result = stl.fit()
result.plot(); plt.tight_layout()

# MSTL for multiple seasonalities (e.g., hourly data with 24h + 168h weekly)
mstl = MSTL(y, periods=[24, 24*7], stl_kwargs={'robust': True})
result = mstl.fit()
# result.seasonal contains (n, 2) array: one column per seasonality period
Engineering Example — Catalyst Deactivation with Daily Regeneration
$T_t$: long-term exponential activity decay over months (catalyst poisoning). $S_t$: daily regeneration cycle (temperature swing restores partial activity). $R_t$: unexpected fouling events, feed composition changes. After STL: model only the remainder $R_t$ with anomaly detection, and the trend $T_t$ with a degradation model (Weibull or Gamma process). Modelling $y_t$ directly without decomposition conflates these three physics — a single model trained on $y_t$ generalises poorly.

5. Frequency Domain Analysis: Welch PSD, STFT, Wavelets

5.1 Welch Power Spectral Density

For stationary signals, the Welch method estimates the power spectral density (PSD) by averaging periodograms over overlapping segments:

$$\hat{S}_{xx}(f) = \frac{1}{K L \|w\|^2} \sum_{k=0}^{K-1}\left|\sum_{n=0}^{L-1} x[n+kD]\, w[n]\, e^{-j2\pi fn/f_s}\right|^2$$

where $L$ = segment length, $D$ = hop size, $K$ = number of segments, $w[n]$ = window function. Key design choices:

ParameterEffectRecommendation
Segment length $L$Frequency resolution $\Delta f = f_s/L$$L \geq 8/f_{min}$ (resolve lowest frequency of interest)
Hop size $D$Overlap = $(L-D)/L$. More overlap → smoother PSD50% overlap ($D=L/2$): standard; 75% for noise-critical applications
Window functionSidelobe level and mainlobe width trade-offHann: default. Flat-top for accurate amplitude. Blackman-Harris for low sidelobes.
 
import numpy as np
from scipy.signal import welch, hann

fs = 20_000   # Hz
signal = vibration_signal

# Welch PSD
f, Pxx = welch(signal, fs=fs, window='hann',
               nperseg=4096,    # L = 4096 → Δf = 4.88 Hz
               noverlap=2048,   # 50% overlap
               detrend='constant', scaling='density')

# Find dominant frequency
f_dominant = f[np.argmax(Pxx)]
print(f"Dominant frequency: {f_dominant:.1f} Hz")

5.2 Short-Time Fourier Transform (STFT)

For non-stationary signals (e.g., engine run-up, transient events), the STFT tracks how frequency content changes over time:

$$STFT\{x\}(t, f) = \int_{-\infty}^\infty x(\tau)\,w(\tau - t)\,e^{-j2\pi f\tau}\,d\tau$$

The discrete STFT is the sequence of windowed FFTs. Time-frequency resolution is limited by the uncertainty principle: $\Delta t \cdot \Delta f \geq \frac{1}{4\pi}$. A long window gives fine frequency resolution but coarse time resolution; a short window gives fine time resolution but coarse frequency resolution.

from scipy.signal import stft
import matplotlib.pyplot as plt

f, t, Zxx = stft(signal, fs=fs, nperseg=512, noverlap=384, window='hann')

plt.pcolormesh(t, f, np.abs(Zxx), shading='gouraud', cmap='viridis')
plt.ylabel('Frequency [Hz]'); plt.xlabel('Time [s]')
plt.colorbar(label='Amplitude'); plt.ylim(0, 5000)
plt.title('STFT — Time-Frequency Map')
plt.tight_layout()

5.3 Continuous Wavelet Transform (CWT)

The CWT overcomes the STFT fixed-resolution limitation by using wavelets that scale their time-frequency trade-off automatically:

$$W(a, b) = \frac{1}{\sqrt{a}}\int_{-\infty}^\infty x(t)\,\psi^*\!\left(\frac{t-b}{a}\right)dt$$

where $a$ is the scale (inversely proportional to frequency) and $b$ is translation. The Morlet wavelet ($\psi(t) = e^{j\omega_0 t}e^{-t^2/2}$) is the default for vibration analysis — it resembles a sinusoid modulated by a Gaussian, providing good time-frequency localisation for transient detection.

import pywt
import numpy as np

# CWT with Morlet wavelet
scales = np.arange(1, 256)
coeffs, freqs = pywt.cwt(signal, scales, 'cmor1.5-1.0', sampling_period=1/fs)
# freqs: centre frequency at each scale
# np.abs(coeffs)**2: scalogram (power at each scale/time)
MethodBest forTime resolutionFrequency resolution

6. Order Analysis for Rotating Machinery

For machinery with variable rotational speed (engines, turbines, compressors), frequency-domain analysis in the time domain produces smeared spectral peaks because the fault-related harmonics shift with RPM. Order analysis solves this by resampling the signal from the time domain to the angle domain (equal angular increments), so that harmonics appear at fixed order numbers regardless of speed.

6.1 Key Definitions

  • Order $n$: a frequency that occurs $n$ times per revolution. Order 1 = once per revolution (fundamental), Order 2 = twice per revolution, etc.
  • Bearing fault orders (from geometry): BPFO (ball pass frequency outer race), BPFI (inner race), BSF (ball spin), FTF (fundamental train). Example for a 6-ball bearing with contact angle 15°: $BPFO = (N_{balls}/2)(1 - d_{ball}/d_{pitch}\cos\theta) \approx 2.3$ orders.
  • Gear mesh order: number of teeth × 1 order = gear mesh frequency.

6.2 Angle-Domain Resampling (Computed Order Tracking)

  1. Record a tachometer/encoder signal alongside the vibration signal.
  2. Compute instantaneous shaft angle $\phi(t)$ from the tachometer pulses.
  3. Resample the vibration signal at equal angular increments $\Delta\phi$ (instead of equal time increments).
  4. Apply FFT in the angle domain → spectrum in orders.
import numpy as np
from scipy.interpolate import interp1d
from scipy.signal import welch

def order_tracking(vibration, time, tach_times, pulses_per_rev=60,
                   orders_max=20, samples_per_rev=256):
    """
    Computed order tracking.
    tach_times: array of timestamps when tachometer pulses occur
    """
    # 1. Instantaneous angle from tach pulses (each pulse = 2pi/PPR radians)
    angle_at_pulse = np.arange(len(tach_times)) * 2 * np.pi / pulses_per_rev
    phi = interp1d(tach_times, angle_at_pulse, kind='linear', fill_value='extrapolate')

    # 2. Uniform angle grid
    phi_max = phi(time[-1])
    phi_uniform = np.arange(0, phi_max, 2*np.pi/samples_per_rev)
    t_uniform = interp1d(phi(time), time, kind='linear')(phi_uniform)

    # 3. Resample vibration at uniform angle grid
    vib_interp = interp1d(time, vibration, kind='linear')
    vib_angle  = vib_interp(t_uniform)

    # 4. Order spectrum (angular Welch PSD)
    angular_fs = samples_per_rev  # samples per revolution
    orders, Poo = welch(vib_angle, fs=angular_fs, nperseg=angular_fs*4,
                        window='hann', detrend='constant')
    mask = orders <= orders_max
    return orders[mask], Poo[mask]
Lesson — When to Use Order vs. Frequency Analysis
Use frequency (Hz) analysis when the machine runs at constant speed (fixed-speed motors, constant-pressure pumps) — Welch PSD directly shows fault frequencies. Use order analysis when speed varies during operation (engines, turbines, variable-speed drives, wind turbines) — order analysis removes speed-related smearing and reveals fault signatures at their correct order numbers. In practice, always combine both: frequency spectrum at constant-speed operating points, plus order analysis for transients and run-ups.

7. ACF, PACF, and Cross-Correlation

7.1 Autocorrelation Function (ACF)

$$\hat{\rho}(h) = \frac{\sum_{t=1}^{n-h}(x_t-\bar{x})(x_{t+h}-\bar{x})}{\sum_{t=1}^n(x_t-\bar{x})^2}$$

Bartlett 95% confidence bands: $\pm 1.96/\sqrt{n}$. ACF cuts off at lag $q$ for an MA(q) process (lags $> q$ are statistically zero). Slowly decaying ACF indicates a unit root — first-difference before ARIMA fitting.

7.2 Partial Autocorrelation Function (PACF)

PACF($h$) = correlation between $x_t$ and $x_{t+h}$ after removing the effect of all intermediate lags $1, \ldots, h-1$. PACF cuts off at lag $p$ for an AR(p) process. Combined ARIMA order identification rule:

  • ACF cuts off at $q$, PACF decays: MA(q)
  • PACF cuts off at $p$, ACF decays: AR(p)
  • Both decay: ARMA(p, q) — try multiple combinations, select by BIC

7.3 Cross-Correlation Function (CCF) and Time Delay Estimation

The CCF between two signals $x_t$ and $y_t$ identifies the lag at which they are most correlated — i.e., the time delay of a cause-effect relationship:

$$\hat{\rho}_{xy}(h) = \frac{\sum_{t=1}^{n-h}(x_t - \bar{x})(y_{t+h} - \bar{y})}{\sqrt{\hat{\sigma}_x^2 \hat{\sigma}_y^2}}$$ 
import numpy as np
from scipy.signal import correlate

def time_delay_estimate(x, y, fs):
    """
    Estimate time delay between x and y using normalised cross-correlation.
    Returns delay in seconds (positive = y lags x).
    """
    x_z = (x - x.mean()) / x.std()
    y_z = (y - y.mean()) / y.std()
    corr = correlate(y_z, x_z, mode='full')
    lags = np.arange(-len(x)+1, len(y)) / fs
    delay = lags[np.argmax(corr)]
    return delay, corr, lags

# Example: delay between fuel flow change and NOx response
delay_s, corr, lags = time_delay_estimate(fuel_flow, nox, fs=200)
print(f"NOx responds to fuel change after {delay_s*1000:.1f} ms")
Engineering Tip — CCF for Input Lag Selection
Before building a dynamic model (LSTM, ARIMA with exogenous), use CCF to identify which lag of each input to include. The lag of maximum CCF gives the physical transport delay. Include 2–3 lags around the peak for robustness. This is more principled than grid-searching over window lengths, and it has a physical interpretation: the delay is the transport time (fuel → combustion → NOx formation → exhaust measurement).

8. ARIMA: Identification, Estimation, Diagnostics

8.1 Model Specification

$$\Phi_p(B)(1-B)^d X_t = \Theta_q(B)\varepsilon_t, \quad \varepsilon_t \overset{iid}{\sim} \mathcal{N}(0,\sigma^2)$$ $$\Phi_p(B) = 1 - \phi_1 B - \cdots - \phi_p B^p, \quad \Theta_q(B) = 1 + \theta_1 B + \cdots + \theta_q B^q$$

For seasonal ARIMA (SARIMA): $\text{ARIMA}(p,d,q)(P,D,Q)_S$ adds seasonal AR, differencing, and MA terms at lag $S$.

8.2 Box-Jenkins Identification Protocol

  1. Stationarity: ADF test. Apply differencing ($d$ times) until stationary.
  2. ACF/PACF: identify candidate orders $(p, q)$.
  3. Estimation: fit all candidate models by MLE.
  4. Model selection: minimise BIC: $BIC = -2\ell + k\ln n$ (more conservative than AIC for order selection).
  5. Diagnostic checking:
from statsmodels.tsa.statespace.sarimax import SARIMAX
from statsmodels.stats.diagnostic import acorr_ljungbox
import numpy as np

# Fit ARIMA(2,1,2) with seasonal component (period=24h)
model = SARIMAX(y_train, order=(2,1,2), seasonal_order=(1,1,1,24),
                trend='n', enforce_stationarity=True)
result = model.fit(method='lbfgs', maxiter=500, disp=False)
print(result.summary())

# Residual diagnostics
resid = result.resid
lb = acorr_ljungbox(resid, lags=[10, 20], return_df=True)
print(lb)  # p-values should all be > 0.05

# Forecast with 95% CI
forecast = result.get_forecast(steps=24)
fc_mean  = forecast.predicted_mean
fc_ci    = forecast.conf_int(alpha=0.05)
ARIMA Residual Diagnostics Checklist
  • Ljung-Box test: $p > 0.05$ at lag 10 and lag 20 → no residual autocorrelation (accept)
  • Residual ACF: no spike exceeds $\pm 1.96/\sqrt{n}$ → white noise
  • Normality: Shapiro-Wilk or QQ plot. For ARIMA forecasting, non-normality widens the CIs but doesn't invalidate point forecasts
  • Heteroscedasticity: ARCH-LM test. If residual variance is non-constant over time, consider GARCH augmentation

9. Exponential Smoothing: SES, Holt-Winters, ETS

9.1 Simple Exponential Smoothing (SES)

$$\hat{y}_{t+1|t} = \alpha y_t + (1-\alpha)\hat{y}_{t|t-1}, \quad \alpha \in (0, 1)$$

Optimal for random-walk data with no trend or seasonality. Interpretable: $\alpha$ is the weight of the most recent observation. Small $\alpha$ = smooth, slow to adapt; large $\alpha$ = responsive, noisy. Optimise $\alpha$ by minimising SSE on training data.

9.2 Holt-Winters (Triple Exponential Smoothing)

Extends SES with level $\ell_t$, trend $b_t$, and seasonality $s_t$ components. For additive seasonality (period $m$):

$$\ell_t = \alpha(y_t - s_{t-m}) + (1-\alpha)(\ell_{t-1}+b_{t-1})$$ $$b_t = \beta(\ell_t - \ell_{t-1}) + (1-\beta)b_{t-1}$$ $$s_t = \gamma(y_t - \ell_{t-1} - b_{t-1}) + (1-\gamma)s_{t-m}$$ $$\hat{y}_{t+h|t} = \ell_t + h b_t + s_{t+h-m}$$

9.3 ETS (Error-Trend-Seasonality) Framework

The ETS framework (Hyndman et al.) unifies all exponential smoothing variants into a state space representation with additive (A) or multiplicative (M) error, trend, and seasonality components. Auto-select the best ETS model by minimum AIC across all 18 possible combinations:

from statsmodels.tsa.exponential_smoothing.ets import ETSModel

# Auto-select best ETS model
best_aic = np.inf
best_model = None
for error in ['add', 'mul']:
    for trend in [None, 'add', 'mul']:
        for seasonal in [None, 'add', 'mul']:
            try:
                m = ETSModel(y_train, error=error, trend=trend,
                             seasonal=seasonal, seasonal_periods=24).fit(disp=False)
                if m.aic < best_aic:
                    best_aic, best_model = m.aic, m
            except: pass

print(f"Best ETS: {best_model.model.error_type},{best_model.model.trend},"
      f"{best_model.model.seasonal_type}, AIC={best_aic:.1f}")
When to Use ETS vs ARIMA
  • ARIMA: explicit autocorrelation structure; interpretable AR/MA terms; when the lag structure is known from physics (e.g., ARIMAX with physical transport delays)
  • ETS: better for short-to-medium horizon forecasting in practice; more robust to model misspecification; preferred for production forecasting where interpretability is less critical
  • For $h > 24$ steps ahead in engineering: ETS often outperforms ARIMA because forecast uncertainty grows more slowly with the state space representation

10. VAR, VARMA, and Granger Causality

10.1 Vector Autoregression (VAR)

$$\mathbf{Y}_t = \mathbf{c} + \mathbf{A}_1\mathbf{Y}_{t-1} + \cdots + \mathbf{A}_p\mathbf{Y}_{t-p} + \boldsymbol{\varepsilon}_t, \quad \boldsymbol{\varepsilon}_t \sim \mathcal{N}(\mathbf{0}, \boldsymbol{\Sigma})$$

Each variable's forecast depends on its own lags and the lags of all other variables. VAR captures cross-variable dynamics that univariate models miss. Requirements: all variables must be stationary (test with multivariate ADF or Johansen test). Lag order $p$: select by BIC (not AIC — BIC is more parsimonious, critical for VAR which has $k^2 p$ parameters).

from statsmodels.tsa.api import VAR
from statsmodels.tsa.stattools import grangercausalitytests

# Fit VAR
var_model = VAR(Y_train)                   # Y_train: (T, k) array
ic_results = var_model.select_order(maxlags=12, ic='bic')
p_opt = ic_results.selected_orders['bic']

fitted = var_model.fit(maxlags=p_opt, ic=None)
print(fitted.summary())

# 12-step ahead forecast with 95% CI
forecast_input = Y_train[-p_opt:]
forecast = fitted.forecast_interval(y=forecast_input, steps=12, alpha=0.05)

10.2 Granger Causality

$x_t$ Granger-causes $y_t$ if past values of $x_t$ significantly improve the forecast of $y_t$ beyond what past values of $y_t$ alone can provide. Test: compare the residual sum of squares (RSS) of AR($p$) on $y_t$ vs. AR($p$) + lagged $x_t$; use F-test or LR test.

# Test if EGR_rate Granger-causes NOx
# Y matrix: [nox, egr_rate] (must be stationary)
granger_test = grangercausalitytests(
    Y_train[['nox', 'egr_rate']],
    maxlag=10, verbose=False
)
# Report p-values at each lag
for lag, result in granger_test.items():
    p_val = result[0]['ssr_ftest'][1]
    print(f"Lag {lag}: p={p_val:.4f} {'→ CAUSAL' if p_val<0.05 else ''}")
Pitfall — Granger Causality ≠ Physical Causality
Granger causality is a statistical concept: $x$ Granger-causes $y$ means past $x$ is useful for predicting $y$. This is not the same as physical causation. A confound (third variable $z$ causing both) produces Granger causality without a physical link. Use Granger causality for variable selection in VAR models, but always validate the identified dependencies against domain knowledge before interpreting as physical cause-effect.

10.3 Cointegration and VECM

If two or more non-stationary $I(1)$ series are cointegrated (share a long-run equilibrium), differencing them loses the cointegration information. Use the Vector Error Correction Model (VECM) instead:

$$\Delta\mathbf{Y}_t = \boldsymbol{\alpha\beta}^\top\mathbf{Y}_{t-1} + \sum_{j=1}^{p-1}\boldsymbol{\Gamma}_j\Delta\mathbf{Y}_{t-j} + \boldsymbol{\varepsilon}_t$$

where $\boldsymbol{\beta}^\top\mathbf{Y}_{t-1}$ is the error correction term (deviation from the long-run relationship) and $\boldsymbol{\alpha}$ is the adjustment speed. Test for cointegration with the Johansen trace test before choosing VECM over VAR.

11. Kalman Filter and Extended Kalman Filter

11.1 Linear Kalman Filter

The KF is the optimal (minimum variance unbiased) estimator for linear-Gaussian state-space models. Two-step predict-update cycle:

$$\text{State: } \mathbf{x}_{t+1} = \mathbf{A}\mathbf{x}_t + \mathbf{B}\mathbf{u}_t + \mathbf{w}_t, \quad \mathbf{w}_t \sim \mathcal{N}(\mathbf{0},\mathbf{Q})$$ $$\text{Obs: } \mathbf{y}_t = \mathbf{C}\mathbf{x}_t + \mathbf{v}_t, \quad \mathbf{v}_t \sim \mathcal{N}(\mathbf{0},\mathbf{R})$$ $$\text{Predict: } \hat{\mathbf{x}}_{t|t-1} = \mathbf{A}\hat{\mathbf{x}}_{t-1|t-1} + \mathbf{B}\mathbf{u}_{t-1}$$ $$\mathbf{P}_{t|t-1} = \mathbf{A}\mathbf{P}_{t-1|t-1}\mathbf{A}^\top + \mathbf{Q}$$ $$\text{Kalman gain: } \mathbf{K}_t = \mathbf{P}_{t|t-1}\mathbf{C}^\top(\mathbf{C}\mathbf{P}_{t|t-1}\mathbf{C}^\top + \mathbf{R})^{-1}$$ $$\text{Update: } \hat{\mathbf{x}}_{t|t} = \hat{\mathbf{x}}_{t|t-1} + \mathbf{K}_t(\mathbf{y}_t - \mathbf{C}\hat{\mathbf{x}}_{t|t-1})$$ $$\mathbf{P}_{t|t} = (\mathbf{I} - \mathbf{K}_t\mathbf{C})\mathbf{P}_{t|t-1}$$

The Kalman gain $\mathbf{K}_t$ optimally weights model prediction vs. observation: if $\mathbf{R}$ is large (noisy measurement), $\mathbf{K}_t$ is small — trust the model. If $\mathbf{Q}$ is large (uncertain model), $\mathbf{K}_t$ is large — trust the measurement.

11.2 Tuning Q and R

The most critical (and often ignored) KF design step. Poor $\mathbf{Q}$ and $\mathbf{R}$ produce suboptimal estimates regardless of model quality:

  • R (observation noise covariance): estimate from sensor data sheets or calibration records. $R = \sigma_{sensor}^2$. Can be validated by comparing KF innovation variance to $\mathbf{C}\mathbf{P}_{t|t-1}\mathbf{C}^\top + \mathbf{R}$ — they should match.
  • Q (process noise covariance): harder to specify from first principles. Inflate $\mathbf{Q}$ to allow the filter to track faster state changes (at the cost of noisier estimates). Common practice: start with $\mathbf{Q} = \epsilon \mathbf{I}$ and tune by monitoring the normalised innovation squared (NIS) — should average $\approx 1.0$ for correct $\mathbf{Q}$, $\mathbf{R}$.

11.3 Extended Kalman Filter (EKF) for Nonlinear Systems

For nonlinear dynamics $\mathbf{x}_{t+1} = f(\mathbf{x}_t, \mathbf{u}_t) + \mathbf{w}_t$ and/or nonlinear observations $\mathbf{y}_t = h(\mathbf{x}_t) + \mathbf{v}_t$, the EKF linearises around the current estimate:

$$\mathbf{A}_t \approx \left.\frac{\partial f}{\partial \mathbf{x}}\right|_{\hat{\mathbf{x}}_{t|t}}, \qquad \mathbf{C}_t \approx \left.\frac{\partial h}{\partial \mathbf{x}}\right|_{\hat{\mathbf{x}}_{t|t-1}}$$

The EKF substitutes $\mathbf{A}_t$ and $\mathbf{C}_t$ into the KF equations at each time step. It is the industry standard for battery SoC estimation (nonlinear OCV-SoC curve), thermal state estimation (nonlinear heat transfer), and navigation (nonlinear kinematics).

import numpy as np

class EKF:
    """Extended Kalman Filter for nonlinear state-space models."""
    def __init__(self, f, h, F_jac, H_jac, Q, R, x0, P0):
        self.f, self.h = f, h        # dynamics and observation functions
        self.F_jac, self.H_jac = F_jac, H_jac  # Jacobians
        self.Q, self.R = Q, R        # process and observation noise
        self.x, self.P = x0.copy(), P0.copy()

    def predict(self, u=None):
        self.x = self.f(self.x, u)
        F = self.F_jac(self.x, u)
        self.P = F @ self.P @ F.T + self.Q

    def update(self, y):
        H = self.H_jac(self.x)
        S = H @ self.P @ H.T + self.R
        K = self.P @ H.T @ np.linalg.inv(S)
        innov = y - self.h(self.x)
        self.x = self.x + K @ innov
        self.P = (np.eye(len(self.x)) - K @ H) @ self.P
        # Innovation-based consistency check:
        nis = float(innov.T @ np.linalg.inv(S) @ innov)
        return self.x.copy(), self.P.copy(), nis

12. Change Point Detection

Change point detection identifies times where the statistical properties of a time series change abruptly — critical for sensor fault detection, process modification detection, and model recalibration triggers.

12.1 PELT (Pruned Exact Linear Time)

PELT finds the optimal segmentation of a series into $K$ stationary segments, each with its own statistical model (mean, variance, or both), by minimising a penalised cost:

$$\min_{\mathbf{t}} \left[\sum_{k=1}^{K} \mathcal{C}(y_{t_{k-1}+1:t_k}) + \lambda K\right]$$

where $\mathcal{C}$ is a model-dependent cost (e.g., negative log-likelihood) and $\lambda$ is the penalty per breakpoint. PELT runs in $\mathcal{O}(n\log n)$ — exact and efficient. Use BIC penalty ($\lambda = \log n$) for engineering applications where fewer, more significant changes are preferred over many spurious detections.

import ruptures as rpt
import numpy as np

signal = np.array(...)  # 1D time series

# PELT with RBF cost (detects changes in mean AND variance)
algo = rpt.Pelt(model='rbf', min_size=50).fit(signal)
breakpoints = algo.predict(pen=np.log(len(signal)))  # BIC penalty

# Dynp: exact but O(n^2) — use for short series only
algo2 = rpt.Dynp(model='normal', min_size=100).fit(signal)
bkps  = algo2.predict(n_bkps=3)  # exactly 3 change points

print("Change point indices:", breakpoints)

12.2 CUSUM for Online Change Detection

For online (streaming) detection where you need to react quickly, CUSUM accumulates evidence of a mean shift $\delta$ in standardised residuals (see Chapter 12 and Chapter 13 for details). PELT is retrospective (best for batch analysis); CUSUM is prospective (best for real-time monitoring).

12.3 Engineering Applications of Change Point Detection

  • Compressor fouling onset: detect the change point in compressor map efficiency — marks the start of fouling and triggers a cleaning schedule.
  • Sensor calibration drift: drift in pressure or temperature sensor causes a step change in the residual of a virtual sensor — detect and flag for recalibration.
  • Process modification: a change in feedstock composition causes a change point in reactor temperature profile — the model needs retraining from that point forward.
  • Bearing defect initiation: broadband energy increase in vibration PSD marks defect onset — locate the change point to estimate remaining useful life.

13. Windowing for Supervised Learning: Temporal Leakage

Pitfall — Window-Level Random Train/Test Splits
For sliding windows of length $W$ with hop size $H < W$, consecutive windows overlap by $W-H$ timesteps. A random split at the window level places overlapping windows in both training and test sets — the test set is not independent of training data. The result: CV RMSE estimates 30–60% lower than true deployment performance. This is one of the most widespread mistakes in industrial time series ML. 
import numpy as np

def sliding_windows(series, W, H=1, horizon=1):
    """Create (X, y) windows from a 1D time series."""
    X, y = [], []
    for i in range(0, len(series) - W - horizon + 1, H):
        X.append(series[i:i+W])
        y.append(series[i+W:i+W+horizon])
    return np.array(X), np.array(y)

# CORRECT: split the raw series first, then window
T = len(raw_series)
n_train_end = int(0.70 * T)
n_val_end   = int(0.85 * T)

# Leave a gap = W between splits to avoid any leakage
gap = 100  # window length
X_tr, y_tr = sliding_windows(raw_series[:n_train_end], W=gap, H=5, horizon=1)
X_va, y_va = sliding_windows(raw_series[n_train_end+gap:n_val_end], W=gap, H=5, horizon=1)
X_te, y_te = sliding_windows(raw_series[n_val_end+gap:], W=gap, H=5, horizon=1)

# WRONG — never do this:
# all_windows = list(zip(all_X, all_y))
# random.shuffle(all_windows)  # creates temporal leakage
Cross-Validation Strategies for Time Series
  • Expanding window CV: train on [0:t], validate on [t:t+h], for increasing t. Mimics online learning; every fold uses all available history.
  • Sliding window CV: train on [t-W:t], validate on [t:t+h], for fixed W. Mimics production where old data is discarded. More conservative estimate.
  • Gap between train and validation: always add at least W timesteps gap to prevent leakage from overlapping windows.
  • Never use k-fold: destroys temporal ordering, creates severe leakage for any autocorrelated series.

14. Time Series Feature Engineering

For tree-based models (XGBoost, LightGBM) and classic ML algorithms applied to time series, explicit feature engineering is necessary. For deep learning (LSTM, TCN), the model learns features — but domain-engineered features can still improve sample efficiency.

14.1 Statistical Features (Sliding Window)

import numpy as np
import pandas as pd

def ts_features(signal, fs, W_short=200, W_long=2000):
    """
    Compute common statistical and spectral features from a time window.
    Designed for rolling feature extraction on sensor data.
    """
    features = {}

    # Time-domain statistical features
    features['mean']     = np.mean(signal)
    features['std']      = np.std(signal)
    features['rms']      = np.sqrt(np.mean(signal**2))      # critical for vibration
    features['peak']     = np.max(np.abs(signal))
    features['crest']    = features['peak'] / features['rms'] # crest factor
    features['kurtosis'] = np.mean((signal - features['mean'])**4) / features['std']**4  # >3 → impulsive
    features['skewness'] = np.mean((signal - features['mean'])**3) / features['std']**3

    # Spectral features (Welch)
    from scipy.signal import welch
    f, Pxx = welch(signal, fs=fs, nperseg=min(len(signal)//4, 512), window='hann')
    features['spectral_centroid'] = np.sum(f * Pxx) / np.sum(Pxx)
    features['spectral_std']      = np.sqrt(np.sum((f - features['spectral_centroid'])**2 * Pxx) / np.sum(Pxx))
    features['band_low']   = np.trapz(Pxx[(f>=100)  & (f<500)],  f[(f>=100)  & (f<500)])
    features['band_mid']   = np.trapz(Pxx[(f>=500)  & (f<2000)], f[(f>=500)  & (f<2000)])
    features['band_high']  = np.trapz(Pxx[(f>=2000) & (f<8000)], f[(f>=2000) & (f<8000)])

    return features

16. Key Takeaways

  • Test stationarity first: ADF + KPSS together. Address trend, seasonality, regime switching, and concept drift explicitly — each requires a different fix.
  • Multi-rate data: always apply anti-aliasing before downsampling. Use zero-order hold for discrete setpoints, linear interpolation for continuous signals.
  • STL decomposition: the default for trend-seasonality extraction. MSTL for multiple seasonality periods. Extract trend + seasonality before anomaly detection or forecasting.
  • Welch for stationary, STFT for non-stationary, CWT for transients. For rotating machinery at variable speed: order analysis in the angle domain.
  • ACF → MA order, PACF → AR order. Validate residuals with Ljung-Box after ARIMA fitting. Use BIC for order selection.
  • Granger causality for VAR variable selection — but verify against domain knowledge. Statistical causality is not physical causality.
  • Kalman filter: optimal for linear-Gaussian systems. EKF for mildly nonlinear (battery SoC, thermal state). UKF when EKF Jacobian is expensive or inaccurate.
  • Change points before model training: always check for structural breaks with PELT. Training a single model across a change point produces biased estimates.
  • Split at the series level, never at the window level. Always add a gap of ≥ W samples between train and validation partitions.
  • For deep learning time series (LSTM, TCN, Transformers, N-BEATS): see Chapter 10. This chapter covers the classical and signal processing foundations that make Chapter 10 interpretable.
