TL;DR

  • Multi-fidelity modelling: combine cheap simulations (1D, coarse CFD) with expensive measurements. The Kennedy-O'Hagan auto-regressive model needs only 15–30 HF points to correct a 200-point LF surrogate.
  • Physics constraints are non-optional in engineering ML. A surrogate that violates conservation laws is not just inaccurate — it is dangerous.
  • Three strategies to enforce physics: (1) architecture constraints (monotone networks, equivariant layers), (2) physics loss terms, (3) grey-box formulation (physics model + ML residual).
  • Multi-fidelity budget rule: if $c_{HF}/c_{LF} > 10$, invest 80% budget in LF, 20% in HF. Minimum HF points: $5(d+1)$.
  • KO scale factor $\rho \approx 1$ means the LF model is well-calibrated. $\rho \ll 0.8$ means the LF model has systematic bias — fix the physics first.

1. The Multi-Fidelity Problem

Engineering workflows produce data at multiple fidelity levels, each with a different cost-accuracy trade-off:

The goal: leverage abundant LF data to reduce the number of expensive HF evaluations required to achieve a given accuracy target.

2. Kennedy-O'Hagan Auto-Regressive Model

The KO model links two fidelity levels with a scale factor and an additive discrepancy GP:

$$y_{HF}(\mathbf{x}) = \rho\,y_{LF}(\mathbf{x}) + \delta(\mathbf{x}), \quad \delta \sim \mathcal{GP}(0, k_\delta)$$

$\rho$ is the multiplicative correction (learned by OLS) and $\delta$ is an independent GP capturing the systematic discrepancy. When $\rho \approx 1$ and $\sigma_\delta \ll \sigma_{LF}$, the LF model is already excellent. When $\rho \ll 0.8$, the LF model has large systematic bias — fix the physics first.

import gpflow, numpy as np
from sklearn.preprocessing import StandardScaler

def kennedy_ohagan_gp(X_lf, y_lf, X_hf, y_hf):
    """Two-level Kennedy-O'Hagan multi-fidelity GP."""
    scaler = StandardScaler()
    X_lf_n = scaler.fit_transform(X_lf)
    X_hf_n = scaler.transform(X_hf)

    # Step 1: Fit GP on LF data
    gp_lf = gpflow.models.GPR(
        (X_lf_n, y_lf[:, None]),
        kernel=gpflow.kernels.Matern52(lengthscales=np.ones(X_lf.shape[1])),
        noise_variance=1e-4
    )
    gpflow.optimizers.Scipy().minimize(gp_lf.training_loss, gp_lf.trainable_variables)

    # Step 2: LF predictions at HF locations
    mu_lf_at_hf, _ = gp_lf.predict_y(X_hf_n)
    mu_lf = mu_lf_at_hf.numpy().ravel()

    # Step 3: Estimate rho by OLS, compute discrepancy
    rho = float(np.linalg.lstsq(mu_lf[:, None], y_hf[:, None], rcond=None)[0][0, 0])
    delta_hf = y_hf - rho * mu_lf

    # Step 4: Fit discrepancy GP (augmented input includes LF prediction)
    X_aug = np.hstack([X_hf_n, mu_lf[:, None]])
    gp_delta = gpflow.models.GPR(
        (X_aug, delta_hf[:, None]),
        kernel=gpflow.kernels.Matern52(lengthscales=np.ones(X_aug.shape[1])),
        noise_variance=1e-4
    )
    gpflow.optimizers.Scipy().minimize(gp_delta.training_loss, gp_delta.trainable_variables)
    print(f"rho = {rho:.4f}  (1.0 = LF perfectly calibrated)")
    return gp_lf, gp_delta, rho, scaler

def predict_mf(gp_lf, gp_delta, rho, scaler, X_new):
    X_n = scaler.transform(X_new)
    mu_lf, _ = gp_lf.predict_y(X_n)
    X_aug = np.hstack([X_n, mu_lf.numpy()])
    mu_d, var_d = gp_delta.predict_y(X_aug)
    mu_hf  = rho * mu_lf.numpy().ravel() + mu_d.numpy().ravel()
    sigma   = np.sqrt(var_d.numpy().ravel())
    return mu_hf, sigma

3. Multi-Fidelity Budget Allocation

Optimal allocation minimises integrated posterior variance subject to a total cost budget $\mathcal{B}$:

$$\min_{n_{LF}, n_{HF}} \int_\mathcal{X} \sigma^{*2}_{MF}(\mathbf{x})\,d\mathbf{x} \quad \text{s.t.} \quad c_{LF}\,n_{LF} + c_{HF}\,n_{HF} \leq \mathcal{B}$$
Budget Allocation Rules of Thumb
  • If $c_{HF}/c_{LF} > 10$: invest 80% budget in LF, 20% in HF.
  • Minimum HF points: $\geq 5(d+1)$ for the discrepancy GP to be identifiable. For $d=5$: at least 30 HF points.
  • LF points: $\geq 10\times$ HF points. More LF data is almost always beneficial since it is cheap.
  • If $\rho \approx 1.0$ and $\sigma_\delta \ll \sigma_{LF}$: LF is excellent; HF adds little value. Stop collecting HF.
  • If $\rho \ll 0.8$: systematic LF bias is large. Fix the LF model physics before adding HF data.

4. Physics Constraints in Engineering ML

Why Physics Constraints Are Non-Optional

Data-driven surrogates trained without physics constraints can violate conservation laws, predict negative pressures, or produce non-monotone responses in physically monotone systems. In engineering, these are not just inaccuracies — they are dangerous if the model is used for control or certification.

Three strategies for enforcing physics, in order of increasing flexibility:

StrategyMechanismCostWhen to Use
Architecture constraintsMonotone networks, equivariant layers, softmax outputsZero (baked in)Known monotonicity or symmetry
Physics loss termsAdd residual of governing equation to training loss (PINN)Moderate (extra compute)Known PDE; continuity, energy balance
Grey-box formulationPhysics model output as input or offset; ML learns residual onlyLow (separate models)Partial physics available; most engineering problems

Architecture Constraints

  • Monotone networks: constrain all weights in one layer to be positive (or use the ICNN architecture). Guarantees output is monotonically increasing in specified inputs — e.g., compressor pressure ratio must increase with mass flow at fixed speed.
  • Input convex networks: ICNN guarantees the output is a convex function of inputs — correct for thermodynamic free energy functions.
  • Equivariant layers: build rotation/reflection symmetry into the layer structure. Critical for 3D field predictions where physics is isotropic.

Physics Loss (PINN-style)

$$\mathcal{L}_{total} = \underbrace{\mathcal{L}_{data}}_{\text{fit observations}} + \lambda\underbrace{\mathcal{L}_{physics}}_{\text{PDE residual at collocation pts}}$$ $$\mathcal{L}_{physics} = \frac{1}{N_c}\sum_{i=1}^{N_c} \left\|\mathcal{N}[\hat{u}](\mathbf{x}_i^c) - f(\mathbf{x}_i^c)\right\|^2$$

where $\mathcal{N}[\hat{u}]$ is the differential operator applied to the network prediction (via automatic differentiation) and $f$ is the source term. The collocation points $\mathbf{x}_i^c$ can be sampled anywhere in the domain — they do not require labels.

5. Transfer Learning for Engineering Surrogates

Transfer learning reuses a surrogate trained on a source domain (e.g., one engine variant, one material grade) to a target domain with limited new data:

  • Fine-tuning: freeze the feature extractor layers; retrain only the output head on target domain data. Effective when source and target domains share low-level physics (same combustion mechanism, different injection geometry).
  • Domain adaptation: align the feature distributions between source and target domains using maximum mean discrepancy (MMD) or adversarial training. Used when the input distribution shifts significantly (different test conditions).
  • KO-style transfer: treat the source domain model as LF and target domain data as HF. Apply KO auto-regressive correction. Requires only $5(d+1)$ target domain points.
Lesson
KO-style transfer (source = LF, target = HF) is almost always the right approach for engineering surrogates. It is interpretable ($\rho$ quantifies how well the source model generalises), statistically rigorous, and requires minimal target data. Fine-tuning neural networks for transfer between engine variants requires careful learning rate selection and risks catastrophic forgetting if the target dataset is small.

6. Key Takeaways

  • Kennedy-O'Hagan: $y_{HF} = \rho y_{LF} + \delta$. Just 15–30 HF points correct a 200-point LF surrogate. Check $\rho$: if $\rho \ll 0.8$, fix the LF physics first.
  • Budget allocation: $c_{HF}/c_{LF} > 10$ → 80% LF, 20% HF. Minimum HF: $5(d+1)$ points for the discrepancy GP.
  • Physics constraints are non-optional. Use architecture constraints (monotone, equivariant) when the physical relationship is known a priori. Use physics losses (PINN) when the PDE is known. Use grey-box otherwise.
  • Grey-box = physics model output + ML residual. This is the default for regulated engineering domains — it provides physical bounds on extrapolation and reduces data requirements.
  • Transfer learning: KO-style (source as LF, target as HF) is the most interpretable and data-efficient transfer strategy for engineering surrogates.
  • Two-fidelity is usually sufficient: adding a third level (e.g., handbook + 1D + HF experiment) yields diminishing returns and complicates the KO nesting.
