TL;DR

  • The Weibull distribution is the workhorse of reliability engineering. Its shape parameter $\beta$ encodes the failure mechanism directly: $\beta < 1$ (infant mortality), $\beta \approx 1$ (random), $\beta > 1$ (wear-out).
  • Censoring is the critical concept: treating right-censored observations as failures underestimates component life by 2–5×. Always use MLE with correct censoring treatment.
  • The lognormal fits fatigue and crack-growth failures better than Weibull. Use the probability plot to decide — don't default to Weibull blindly.
  • Use likelihood ratio confidence bounds, not Wald (Fisher matrix) bounds, when $n < 30$.
  • ALT compresses years of service life into days. The Arrhenius model (thermal) and Inverse Power Law (voltage, load) are the two most-used acceleration models in industry.
  • RUL prediction without a calibrated prediction interval (P10/P90 band) is not engineering output. The interval drives maintenance scheduling, not the point estimate.
  • All major algorithms in this chapter are available in the Python reliability library by Matthew Reid — use it.

1. Reliability Functions: The Four Building Blocks

Every reliability distribution is fully characterised by four equivalent functions. Understanding their relationships is more important than memorising formulas — each function is the most natural description for a different engineering question.

$$\text{PDF: } f(t) \quad \text{(failure density)} \qquad \text{CDF: } F(t) = \int_0^t f(\tau)\,d\tau \quad \text{(cumulative failure probability)}$$ $$\text{Reliability: } R(t) = P(T > t) = 1 - F(t) \quad \text{(survival probability)}$$ $$\text{Hazard rate: } h(t) = \frac{f(t)}{R(t)} = -\frac{d}{dt}\ln R(t) \quad \text{[failures/time/survivor]}$$ $$\text{Cumulative hazard: } H(t) = \int_0^t h(\tau)\,d\tau = -\ln R(t) \implies R(t) = \exp(-H(t))$$

The hazard rate $h(t)$ is the instantaneous failure rate given that the unit has survived to time $t$. It is not the same as $f(t)$. This distinction matters: $f(t)$ can decrease while $h(t)$ increases (e.g., Weibull with $\beta < 2$).

The Bathtub Curve

The famous bathtub curve describes the system-level hazard over its lifetime as the superposition of three phases:

  • Burn-in / Infant Mortality: decreasing hazard. Dominated by manufacturing defects, latent flaws, poor assembly. Weibull $\beta < 1$. Mitigated by burn-in testing, acceptance screening, proof tests.
  • Useful Life: approximately constant hazard. Random, externally caused failures. Exponential distribution, Weibull $\beta \approx 1$.
  • Wear-out: increasing hazard. Fatigue, corrosion, erosion, creep. Weibull $\beta > 1$, Lognormal for crack-driven mechanisms.
Senior Practitioner Tip
The "bathtub curve" is a system-level concept. Individual failure modes don't follow a bathtub — each has its own monotone hazard. You get the bathtub shape only when you aggregate multiple failure modes with different $\beta$ values. Always decompose field data by failure mode before fitting distributions. Mixing modes in a single Weibull fit produces a meaningless averaged result.

Key metrics derived from the four functions:

$$\text{MTTF} = \mathbb{E}[T] = \int_0^\infty R(t)\,dt \qquad \text{Median life: } t_{0.5} = F^{-1}(0.5) = \eta(\ln 2)^{1/\beta}$$ $$B_p \text{ life (p% failure probability)}: \quad t_{Bp} = F^{-1}(p/100)$$

2. Probability Distributions: The Practitioner's Guide

Selecting the right distribution is the most consequential modelling decision in reliability analysis. Getting it wrong by even one distribution family can shift $B_{10}$ life estimates by 50% or more. Always justify the choice with a probability plot and a goodness-of-fit test — not domain convention alone.

Distribution equations and fitting guidance follow the reliability Python library documentation (Matthew Reid) and Meeker & Escobar (1998), Statistical Methods for Reliability Data.

Weibull 2-Parameter (Weibull_2P)

Parameters: β (shape), η (scale) Support: t ≥ 0 Hazard: monotone power-law
$$f(t) = \frac{\beta}{\eta}\left(\frac{t}{\eta}\right)^{\beta-1}\exp\!\left[-\left(\frac{t}{\eta}\right)^\beta\right], \quad F(t) = 1-\exp\!\left[-\left(\frac{t}{\eta}\right)^\beta\right], \quad h(t) = \frac{\beta}{\eta}\left(\frac{t}{\eta}\right)^{\beta-1}$$ $$B_{10} = \eta\,(0.10536)^{1/\beta}, \quad \text{MTTF} = \eta\,\Gamma(1+1/\beta)$$
β RangeFailure ModeHazardIndustrial Examples
β < 1Infant mortalityDecreasingCasting defects, solder joints, PCB trace voids, installation errors
β = 1Random (exponential)ConstantParticle erosion, external shocks, software faults
1 < β < 2Early wear-outSlowly increasingRolling element bearings (β ≈ 1.5), adhesive bond fatigue
β ≈ 2Rayleigh wear-outLinear increaseDynamic seals, cable insulation
2 < β < 4Moderate wear-outIncreasingGear tooth fatigue (β ≈ 2.5–3.5), pump impeller erosion

When to use: Flexible monotone hazard shapes. First choice for mechanical component fatigue, wear, and corrosion when the hazard is known to be monotone. Avoid when failure time distribution is multimodal or when the WPP shows curvature (suggesting 3-parameter Weibull or a different family).

Senior Practitioner Tip — Weibull
The most common Weibull mistake in industry: fitting a single Weibull to data from mixed failure modes. If two failure modes have $\beta_1 = 0.5$ (early failures) and $\beta_2 = 3.0$ (wear-out), the mixed data produces a WPP with an S-curve — not a straight line. Fit a Competing Risks model (see Section 8) or segment the data by cause code before fitting. Another red flag: a WPP where all points follow the line except the first 2–3 (which fall below it) — this is the signature of a threshold / failure-free period, indicating a 3-parameter Weibull with location parameter $\gamma > 0$.

Weibull 3-Parameter (Weibull_3P)

Parameters: β, η, γ (location/threshold) Support: t ≥ γ ≥ 0

Adds a location parameter $\gamma$ (failure-free period): $F(t) = 1-\exp[-(t-\gamma)^{\beta}/\eta^\beta]$ for $t \geq \gamma$.

When to use: When there is a guaranteed failure-free period — e.g., a turbine blade cannot fail in the first 500 cycles before it has accumulated enough fatigue damage. The WPP will show the first points falling below the fitted line through the later points. MLE for $\gamma$ is not straightforward (the likelihood may be unbounded); use penalised MLE or the method of moments for $\gamma$ and MLE for $\beta$, $\eta$ conditional on $\hat{\gamma}$.

Pitfall — Weibull 3P Overfitting
With small samples ($n < 20$), the 3-parameter Weibull almost always fits better than 2P by construction — you're adding a free parameter. Always compare with AIC/BIC or a likelihood ratio test before adopting the 3P model. A common error: fitting 3P to censored data with $n = 10$ and reporting $\hat{\gamma}$ as a "guaranteed safe life" for certification. The confidence interval on $\gamma$ from such a fit is enormous.

Lognormal (Normal_2P on log scale)

Parameters: μ (log-mean), σ (log-std) Support: t > 0 Hazard: non-monotone (rises then falls)
$$f(t) = \frac{1}{t\sigma\sqrt{2\pi}}\exp\!\left[-\frac{(\ln t - \mu)^2}{2\sigma^2}\right], \quad F(t) = \Phi\!\left(\frac{\ln t - \mu}{\sigma}\right)$$ $$\text{Median} = e^\mu, \quad \text{Mean} = e^{\mu + \sigma^2/2}, \quad h(t) = \frac{\phi(z)}{t\sigma[1-\Phi(z)]},\quad z = \frac{\ln t - \mu}{\sigma}$$

When to use: The physical justification comes from the multiplicative failure argument: if failure occurs when damage accumulates through a product of many small random increments, the logarithm of the failure time is normally distributed (by CLT). This is the case for:

  • Fatigue crack growth governed by Paris's law ($da/dN = C(\Delta K)^m$) — crack growth per cycle is multiplicative
  • Diffusion-driven degradation (oxide growth, corrosion pitting depth)
  • Electronic device degradation (migration, dielectric breakdown, electromigration)
  • Strength distributions of composite materials and brittle ceramics
  • Repair time distributions (lognormal is the standard model for lognormal repair times in maintainability)
Senior Practitioner Tip — Lognormal vs Weibull
For fatigue failures, the lognormal often fits better in the central region (50–90% failures) while Weibull fits better in the tails. This matters enormously for B10 life estimates: the Weibull can give a 30–50% shorter B10 than lognormal on the same dataset. For safety-critical applications, always fit both and present both B10 estimates with confidence bounds to the design authority. Never hide the model uncertainty.

The lognormal hazard function peaks and then decreases at long times — this is physically realistic for some mechanisms (competing risks where survivors become "immune") but unrealistic for wear-out of a homogeneous population. Use domain knowledge to discriminate.

Exponential (Expon_1P, Expon_2P)

Parameters: λ (rate), γ (location, optional) Constant hazard: h(t) = λ Memoryless property
$$f(t) = \lambda e^{-\lambda t}, \quad F(t) = 1 - e^{-\lambda t}, \quad h(t) = \lambda \text{ (constant)}$$ $$\text{MTTF} = 1/\lambda, \quad \text{Median} = \ln 2/\lambda \approx 0.693/\lambda$$

When to use: When failure is driven by random external events independent of component age — particle impacts, random load spikes, software fault activation. The memoryless property means a unit that has survived $t_0$ hours is statistically identical to a new unit. This is the only distribution for which age-based preventive maintenance provides zero reliability improvement — a critical result for maintenance planning.

Pitfall — Exponential as Default
Many military standards (MIL-HDBK-217) and early FMEA tools defaulted to exponential distributions for all components. This is statistically wrong for mechanical and structural components, which have clearly increasing hazard rates. The exponential model is appropriate only for electronic components in their useful-life phase and for systems where many independent failure modes compete. Always test the constant-hazard assumption (Crow-AMSAA test, or simply plot the WPP and check if β is statistically distinguishable from 1.0).

Normal (Normal_2P)

Parameters: μ (mean), σ (std dev) Support: (−∞, ∞) — truncate to t ≥ 0 in practice Hazard: monotone increasing

When to use: Strength and material property distributions (yield strength, ultimate tensile strength, hardness). Time-to-failure for well-controlled manufacturing processes where the central limit theorem applies directly to failure accumulation. Wear-out governed by additive damage (not multiplicative). The normal distribution is rarely the best fit to failure time data but is very common in strength modelling for stress-strength interference calculations.

Tip — Normal for Strength, Lognormal for Life
A frequently validated empirical rule: material strengths are well-modelled by Normal; component lives (time to failure) are better modelled by Lognormal (multiplicative accumulation) or Weibull (weakest-link / size effect). The distinction matters for stress-strength analysis where you combine the two distributions.

Gamma (Gamma_2P)

Parameters: α (shape), β (scale) Special cases: α=1 → Exponential; α=n/2 → Chi-squared Hazard: monotone for α≠1
$$f(t) = \frac{t^{\alpha-1}e^{-t/\beta}}{\beta^\alpha\Gamma(\alpha)}, \quad \text{MTTF} = \alpha\beta, \quad \text{Var} = \alpha\beta^2$$

When to use: When time to failure is the sum of $\alpha$ exponentially distributed events (e.g., cumulative damage from $\alpha$ random shocks). Common in degradation modelling where degradation increments are Gamma-distributed (Gamma process). Also the conjugate prior for the Poisson rate, making it the natural prior for inter-failure interval rates in Bayesian reliability.

Tip — Gamma for Degradation
The Gamma process is the preferred degradation model when: (1) degradation is monotonically increasing, (2) independent increments, (3) increments follow a Gamma distribution. It's analytically tractable with conjugate Bayesian updates. For turbomachinery tip clearance growth, creep strain accumulation, and corrosion depth, the Gamma process outperforms the Weibull degradation model because it directly models the degradation trajectory rather than the failure time marginal.

Gumbel / Smallest Extreme Value (Gumbel_2P)

Parameters: μ (location), σ (scale) Log(Weibull) ≡ SEV distribution
$$f(t) = \frac{1}{\sigma}\exp\!\left(\frac{t-\mu}{\sigma}\right)\exp\!\left[-\exp\!\left(\frac{t-\mu}{\sigma}\right)\right]$$

When to use: The log-scale equivalent of Weibull. If $T \sim \text{Weibull}(\eta,\beta)$, then $\ln T \sim \text{SEV}(\ln\eta, 1/\beta)$. Used internally in reliability software because regression on the SEV model is computationally identical to Weibull fitting on the log scale. Also directly applicable for extreme value problems (minimum strength in a large structure, maximum stress in a random load process).

Loglogistic (Loglogistic_2P)

Parameters: α (scale), β (shape) Non-monotone hazard Heavier right tail than lognormal
$$F(t) = \frac{1}{1+(t/\alpha)^{-\beta}}, \quad h(t) = \frac{\beta/\alpha \cdot (t/\alpha)^{\beta-1}}{1+(t/\alpha)^\beta} \quad \text{(peaks at } t=\alpha[(\beta-1)/(\beta+1)]^{1/\beta}\text{)}$$

When to use: When the hazard rate first increases then decreases — a pattern seen in some biological populations (therapy-induced failures that eventually subside) and in software reliability (discovery rate of latent bugs). Also useful as a robust alternative to Lognormal when the right tail is heavier than lognormal predicts. In financial reliability contexts (time-to-default modelling), the Loglogistic is closely related to the Proportional Odds model.

Distribution Selection Protocol (Senior Practitioner)
  1. Identify the failure mechanism from failure analysis / FMEA, not from the data alone.
  2. Select 2–3 candidate distributions based on the physics (table above).
  3. Fit all candidates using MLE. Generate probability plots for each — the best-fitting distribution produces the straightest probability plot.
  4. Perform a goodness-of-fit test: Anderson-Darling is preferred over KS for reliability data (more sensitive to tail behaviour). Compare AIC/BIC for model selection.
  5. Validate the choice by comparing $B_{10}$ life confidence bounds between candidate models. If bounds are nearly identical, model choice matters little. If they diverge, report the range as epistemic uncertainty.
  6. Document the justification. "Weibull because everyone uses Weibull" is not a justification.

3. Parameter Estimation: MLE, Rank Regression, Plotting Positions

3.1 Maximum Likelihood Estimation (MLE)

MLE finds the parameter values that maximise the likelihood of observing the actual dataset. It is the gold standard because it is asymptotically efficient (minimum variance), handles censoring correctly, and provides a natural framework for confidence bounds.

$$\hat{\boldsymbol{\theta}} = \arg\max_{\boldsymbol{\theta}}\, \ell(\boldsymbol{\theta}) = \arg\max_{\boldsymbol{\theta}} \left[\sum_{i \in \mathcal{F}} \ln f(t_i;\boldsymbol{\theta}) + \sum_{i \in \mathcal{C_R}} \ln R(c_i;\boldsymbol{\theta}) + \sum_{i \in \mathcal{C_L}} \ln F(c_i;\boldsymbol{\theta}) + \sum_{i \in \mathcal{C_I}} \ln[F(u_i;\boldsymbol{\theta})-F(l_i;\boldsymbol{\theta})]\right]$$

where $\mathcal{F}$ are failures, $\mathcal{C_R}$ right-censored, $\mathcal{C_L}$ left-censored, and $\mathcal{C_I}$ interval-censored observations. Numerically, minimise $-\ell$ using L-BFGS-B or trust-region methods. The reliability library handles all censoring types automatically.

3.2 Rank Regression (Least Squares on Probability Plot)

Rank Regression on Y (RRY) minimises squared vertical distances on the probability plot. It is simpler than MLE but less efficient with censored data — MLE is always preferred when censoring is present. RRY is useful for graphical diagnostics and for cross-checking MLE results on small complete datasets.

3.3 Plotting Positions (Median Ranks)

To construct a probability plot, you need an estimate $\hat{F}_i$ for each ordered failure time $t_{(i)}$. Several estimators exist:

4. Confidence Bounds: Fisher Matrix vs Likelihood Ratio

A reliability analysis without confidence bounds is not a reliability analysis — it is a point estimate. Confidence bounds quantify the uncertainty in the fitted parameters due to finite sample size. Two main methods:

4.1 Fisher Matrix (Wald) Bounds

Based on the asymptotic normality of MLE. The information matrix $\mathbf{I}(\hat{\boldsymbol{\theta}}) = -\nabla^2\ell|_{\hat{\boldsymbol{\theta}}}$ gives the covariance of parameter estimates:

$$\text{Var}(\hat{\theta}_j) \approx [\mathbf{I}(\hat{\boldsymbol{\theta}})^{-1}]_{jj}$$ $$\text{Wald bound: } \hat{\theta}_j \pm z_{\alpha/2}\,\sqrt{\text{Var}(\hat{\theta}_j)}$$

Problem: The normal approximation for log-likelihood contours is only valid asymptotically. For $n < 30$, or when parameters are on the boundary of their support, Wald bounds are unreliable — they can produce negative lower bounds on a positive-only parameter like $\beta$, or grossly asymmetric intervals.

4.2 Likelihood Ratio (LR) Bounds

Based on the profile likelihood. The 90% confidence region for $\theta_j$ is the set of values for which the likelihood ratio statistic is below the chi-squared critical value:

$$\left\{\theta_j : 2[\ell(\hat{\boldsymbol{\theta}}) - \ell(\theta_j, \hat{\boldsymbol{\theta}}_{-j}(\theta_j))] \leq \chi^2_{1,0.90}\right\}$$

where $\hat{\boldsymbol{\theta}}_{-j}(\theta_j)$ is the profile likelihood: the MLE of all other parameters at fixed $\theta_j$. LR bounds are:

  • Invariant under reparameterisation (e.g., bounds on $B_{10}$ are consistent with bounds on $\eta$ and $\beta$ separately)
  • Inherently bounded to the parameter support (cannot produce negative $\beta$)
  • Preferred for $n < 30$ and recommended by the reliability library as the default
CriterionFisher (Wald)Likelihood Ratio
Computation speedFast (matrix inversion)Slower (profile optimisation)

Respects parameter supportNo (can give negative $\beta$)Yes

Industry Rule
Use LR bounds by default. Use Wald bounds only when $n > 50$ and you need fast computation for Monte Carlo reliability growth assessments. Always report both bounds alongside the point estimate. A reported $B_{10}$ life of "1200 hrs" is nearly meaningless without "90% LR bounds: [800, 1900] hrs."

5. Probability Plotting: Diagnostics and Pattern Recognition

The probability plot is not just a fitting tool — it is a diagnostic instrument. Every pattern deviation from a straight line tells you something about your failure physics.

5.1 How to Read a Weibull Probability Plot (WPP)

The WPP linearises the Weibull CDF by plotting $y = \ln[-\ln(1-\hat{F}_i)]$ vs. $x = \ln(t_i)$. For a perfect Weibull, all points fall on a straight line. The slope equals $\hat{\beta}$; the x-intercept at $y=0$ (i.e., $F = 63.2\%$) gives $\ln\hat{\eta}$.

WPP PatternInterpretationAction
Straight line2-parameter Weibull fits wellProceed with 2P Weibull
S-curve (concave then convex)Mixed failure modes or heterogeneous populationFit Competing Risks or Mixture model; segment by failure mode
Concave (curves upward)3-parameter Weibull with threshold γ > 0Fit Weibull_3P; verify physics of failure-free period
Convex (curves downward)Lognormal may fit better; or mixture with early infant failuresTry Lognormal_2P; check early failures separately
First 1–2 points far below lineOutliers or early failures from a different failure modeInvestigate first failures; separate infant-mortality events
Last points above lineHeavy right tail; possible extreme-value behaviourTry Loglogistic or 3P Weibull; check for a plateau
Very steep slope (β >> 4)Tight failure distribution, well-controlled processVerify — tight Weibull on a small sample can be misleading

5.2 Goodness-of-Fit Tests for Reliability Data

Visual assessment of the probability plot is necessary but not sufficient. Always supplement with a quantitative test:

  • Anderson-Darling (A-D) test: preferred for reliability data because it weights the tails more heavily, where reliability decisions are most sensitive. Modified A-D statistics are available for each distribution family.
  • Kolmogorov-Smirnov (K-S) test: based on the maximum CDF deviation. Less sensitive to tail behaviour, but widely implemented. Conservative with censored data.
  • AIC / BIC: for comparing distributions with different numbers of parameters. BIC penalises extra parameters more strongly and is generally preferred for model selection with $n < 100$.

6. Censored Data: The Most Underestimated Problem

Pitfall — Treating Censored Observations as Failures
Right-censored units (not yet failed at end of study) contribute information that $T_i > c_i$ — they should NOT be treated as failures at time $c_i$. Doing so introduces catastrophic negative bias: component life is underestimated by factors of 2–5×. This is the single most common error in reliability data analysis. It leads to over-maintained systems, unnecessary part replacements, and false conclusions about fleet health.

6.1 Types of Censoring

TypeWhat you knowEngineering Context
Right censoring$T_i > c_i$Unit still running at test/study end; removed before failure (competing reason). Most common in field data.
Left censoring$T_i < c_i$Unit found failed before first inspection. Common in periodic maintenance campaigns.
Interval censoring$L_i < T_i \leq R_i$Failure known to have occurred between two inspections (borescope, NDT). Very common in gas turbine disc inspection data.
Type I censoringAll units monitored to fixed time $c$Test-to-time studies. Variable number of failures.
Type II censoringTest run until $r$ failures occurAccelerated test design. Fixed number of failures, variable test time.

6.2 MLE with Censored Data

$$\ell(\boldsymbol{\theta}) = \sum_{i \in \mathcal{F}} \ln f(t_i;\boldsymbol{\theta}) + \sum_{i \in \mathcal{C_R}} \ln[1-F(c_i;\boldsymbol{\theta})] + \sum_{i \in \mathcal{C_L}} \ln F(c_i;\boldsymbol{\theta}) + \sum_{i \in \mathcal{C_I}} \ln[F(u_i;\boldsymbol{\theta})-F(l_i;\boldsymbol{\theta})]$$ 
from reliability.Fitters import Fit_Weibull_2P

# Mixed censoring types
t_fail    = [850, 1200, 1340, 1650, 1980]
t_right   = [2000, 2000, 2000, 2000, 2000]   # still running at 2000 hrs
t_left    = [300, 400]                         # found failed before inspection
t_int_low = [500, 700]                         # interval: failed between
t_int_up  = [600, 850]                         #   these two inspection times

fit = Fit_Weibull_2P(
    failures=t_fail,
    right_censored=t_right,
    left_censored=t_left,
    interval_censored=list(zip(t_int_low, t_int_up)),
    CI=0.90,
    show_probability_plot=False
)
print(f"β = {fit.beta:.3f}  η = {fit.alpha:.1f} hrs")
print(f"B10 = {fit.quantiles(0.1):.0f} hrs  [90% LR CI]")

6.3 Effective Sample Size with Censored Data

A critical practical question: how much information does a censored observation contribute? A right-censored observation at time $c$ contributes the same amount of information as approximately $F(c;\hat{\boldsymbol{\theta}})$ failures. A unit censored at the median of its distribution contributes about 50% as much information as a failure. This means:

  • A fleet of 100 units with 80% still running (80 censored, 20 failed) has an effective sample size of roughly 20 + 80×F(c) ≈ 60 for a typical mid-life censoring time.
  • You need approximately 20 actual failures for reasonable precision on $\beta$ and $\eta$ — not 20 units total.
  • With fewer than 5 failures, all confidence bounds are very wide. Report them anyway; do not suppress wide bounds.

7. Non-Parametric Methods: Kaplan-Meier, Nelson-Aalen, Rank Adjustment

Non-parametric methods estimate the reliability function without assuming a distribution family. They are essential for exploratory analysis, for checking parametric model assumptions, and for data where no distribution fits well.

7.1 Kaplan-Meier (Product-Limit) Estimator

$$\hat{R}(t) = \prod_{t_{(i)} \leq t} \left(1 - \frac{d_i}{n_i}\right)$$

where $d_i$ is the number of failures at time $t_{(i)}$ and $n_i$ is the number of units at risk just before $t_{(i)}$ (risk set). Right-censored observations reduce the risk set at their censoring times without contributing to the product.

from reliability.NonParametricEstimators import KaplanMeier

failures       = [100, 200, 300, 500, 700, 900, 1200]
right_censored = [150, 400, 600, 800, 1100]

km = KaplanMeier(failures=failures, right_censored=right_censored,
                 print_results=True, show_plot=True,
                 plot_CI=True, CI=0.95)
# km.results: DataFrame with t, d, n, KM estimate, CI bounds

7.2 Nelson-Aalen Estimator

Estimates the cumulative hazard function $H(t)$ non-parametrically:

$$\hat{H}(t) = \sum_{t_{(i)} \leq t} \frac{d_i}{n_i}, \qquad \hat{R}(t) = \exp(-\hat{H}(t))$$

The Nelson-Aalen estimator is smoother than Kaplan-Meier for the survival function and is preferred when the hazard function itself is of interest (e.g., for ROCOF estimation in repairable systems).

7.3 Rank Adjustment

A non-parametric estimator that adjusts the ordering of observations to account for censoring. It computes the "mean order number" (effective rank) for each failure, then converts to failure probability using the median rank formula. The reliability library implements this for plotting on probability paper when censoring prevents the direct Kaplan-Meier application.

8. Competing Risks and Mixture Models

8.1 Competing Risks (Series Failure Modes)

When a component can fail from multiple independent causes, and the failure time is the minimum across all modes, the failure processes compete. The system reliability under competing risks is:

$$R_{system}(t) = \prod_{k=1}^K R_k(t) = \prod_{k=1}^K [1-F_k(t)]$$ $$h_{system}(t) = \sum_{k=1}^K h_k(t)$$

This is a series system in the failure-mode sense. Each failure mode $k$ has its own distribution (e.g., Mode 1: bearing fatigue, Weibull $\beta_1 = 2.1$; Mode 2: seal leakage, Weibull $\beta_2 = 0.7$). The mixed data without cause code produces a WPP that curves — an S-shape or inflection that does not fit any single distribution.

from reliability.Fitters import Fit_Weibull_2P
from reliability.CompetingRisks import CompetingRisks

# Data with cause codes (0 = mode A, 1 = mode B)
t_modeA = [850, 1200, 1500, 1900]          # failures from mode A
t_modeB = [200, 400, 550]                   # failures from mode B
cens_A  = [2000, 2000, 2000]               # mode A right-censored (survived)
cens_B  = [2000, 2000, 2000, 2000]         # mode B right-censored

# Fit each mode independently
fit_A = Fit_Weibull_2P(failures=t_modeA, right_censored=t_modeB + cens_A,
                        CI=0.9, show_probability_plot=False)
fit_B = Fit_Weibull_2P(failures=t_modeB, right_censored=t_modeA + cens_B,
                        CI=0.9, show_probability_plot=False)
# Note: units that don't fail from mode A are right-censored for mode A analysis
Important — Independent Censoring Assumption
Competing risks analysis assumes that the cause of failure from one mode is independent of the failure time from another mode. This is the "independent competing risks" assumption and is often violated in practice: a bearing that is heating up may simultaneously accelerate seal degradation. In such cases, the competing risks decomposition still gives correct marginal estimates for each mode's distribution, but the cause-specific hazards cannot be simply added to predict system reliability under modified conditions.

8.2 Mixture Models (Heterogeneous Populations)

A mixture model is appropriate when the population consists of distinct subpopulations with different failure characteristics — for example, a production batch with a fraction $p$ of "weak" units (infant mortality, small $\beta$) and fraction $1-p$ of "strong" units (wear-out, large $\beta$):

$$F_{mix}(t) = p\,F_1(t) + (1-p)\,F_2(t)$$ $$h_{mix}(t) = \frac{p\,f_1(t) + (1-p)\,f_2(t)}{p\,R_1(t) + (1-p)\,R_2(t)}$$ 
from reliability.Fitters import Fit_Weibull_Mixture

# Mixture of two Weibull populations
failures = [50, 100, 150, 200, 800, 1200, 1500, 1800, 2100, 2400]

mix = Fit_Weibull_Mixture(failures=failures, right_censored=[2500, 2500, 2500],
                           CI=0.9, show_probability_plot=True)
print(f"Subpop 1: beta={mix.beta_1:.2f}, alpha={mix.alpha_1:.1f}, p={mix.proportion_1:.3f}")
print(f"Subpop 2: beta={mix.beta_2:.2f}, alpha={mix.alpha_2:.1f}")

The mixture fraction $p$ can be interpreted as the fraction of "defective" units in the population. It is a key quality control metric: reducing $p$ through process improvement is often more cost-effective than redesigning the component.

9. Accelerated Life Testing (ALT)

ALT deliberately stresses components beyond their normal operating conditions (higher temperature, voltage, vibration, load) to compress years of service life into days of testing. The objective is to extrapolate the failure time distribution back to normal conditions using a physically motivated acceleration model.

ALT theory and Python implementation follow reliability.readthedocs.io — "Accelerated Life Testing" section.

9.1 The Three Core Assumptions of ALT

  1. Same failure mode: the failure mode at stress level $S$ must be the same as at use stress $S_0$. If accelerated stress causes a different failure mechanism, the extrapolation is invalid. Verify with failure analysis at each stress level.
  2. Known acceleration model: the relationship between failure time and stress follows a specific functional form (Arrhenius, Inverse Power Law, etc.).
  3. Same distribution family: the life distribution family (Weibull, Lognormal) is the same at all stress levels. Only the scale parameter changes with stress; the shape parameter $\beta$ is constant.

9.2 Arrhenius Model (Thermal Acceleration)

The Arrhenius model describes thermally-activated degradation processes (corrosion, diffusion, electromigration, chemical degradation):

$$\text{Life}(T) = A\,\exp\!\left(\frac{E_a}{k_B T}\right)$$ $$\text{Acceleration Factor: } AF = \frac{\text{Life}(T_0)}{\text{Life}(T_s)} = \exp\!\left[\frac{E_a}{k_B}\left(\frac{1}{T_0} - \frac{1}{T_s}\right)\right]$$

where $T$ is absolute temperature (Kelvin), $k_B = 8.617 \times 10^{-5}$ eV/K (Boltzmann constant), $E_a$ is the activation energy (eV, material/mechanism property). Typical values:

Failure MechanismActivation Energy $E_a$ (eV)
Electromigration (Al)0.5 – 0.7
Corrosion (Al in humid air)0.3 – 0.5
Oxide growth (SiO₂)1.0 – 1.3
Interdiffusion (solder)0.3 – 0.5
Polymer degradation0.5 – 1.0
CMOS hot carrier injection−0.1 to −0.3 (inverse)

9.3 Inverse Power Law (Voltage, Stress, Vibration Acceleration)

$$\text{Life}(S) = \frac{A}{S^n}, \qquad AF = \left(\frac{S_s}{S_0}\right)^n$$

where $S$ is the stress level (voltage, load amplitude, vibration g-level) and $n$ is the power law exponent. Common values: $n \approx 2$–4 for voltage (capacitor dielectric), $n \approx 4$–12 for fatigue (stress amplitude, S-N curves), $n \approx 2$ for vibration (Miner's rule with random loading).

9.4 Eyring and Multi-Stress Models

$$\ln(\text{Life}) = a_0 + \frac{a_1}{T} + a_2 V + \frac{a_3 V}{T} \qquad \text{(Eyring with thermal and voltage)}$$

The dual-stress Eyring model is the workhorse for electronic components subjected to both temperature and humidity (or temperature and voltage). All three models are available in the reliability library's ALT fitters.

from reliability.ALT_fitters import Fit_Weibull_Arrhenius

# ALT data: failures and censored observations at two stress levels (Kelvin)
failures_hot  = [200, 320, 450, 600, 700]   # at 398K (125°C)
failures_warm = [1500, 2100, 2700, 3200]    # at 358K (85°C)
cens_hot   = [800, 800]
cens_warm  = [4000, 4000, 4000]

fit = Fit_Weibull_Arrhenius(
    failures      = failures_hot + failures_warm,
    failure_stress= [398]*len(failures_hot) + [358]*len(failures_warm),
    right_censored      = cens_hot + cens_warm,
    right_censored_stress=[398]*len(cens_hot) + [358]*len(cens_warm),
    use_level_stress=293,  # extrapolate to 20°C (293K)
    CI=0.9
)
print(f"Ea = {fit.Ea:.3f} eV")
print(f"Use-level B10: {fit.distribution_at_use_level.quantiles(0.1):.0f} hrs")
Pitfall — ALT Extrapolation Range
The Arrhenius acceleration factor grows exponentially with temperature difference. At $E_a = 0.7$ eV, going from 398K to 293K gives AF ≈ 3000×. A test producing 200 hours of failure data is extrapolated to 600,000 hours of use life. The confidence bounds at that extrapolation distance are enormous — often spanning an order of magnitude. Always plot and report the extrapolated life CDF with confidence bounds, not just the point estimate. And always verify that the failure mode at the accelerated stress matches the use-condition failure mode.

10. Repairable Systems: Crow-AMSAA, MCF, ROCOF

A repairable system is one that is restored to operation after failure rather than replaced. The statistical framework is fundamentally different from non-repairable analysis: instead of a single failure time, you observe a sequence of inter-failure times (or equivalently, cumulative failure counts). The question is no longer "what is the time to first failure?" but "how does the failure rate evolve with system age?"

Theory follows reliability.readthedocs.io — "Repairable Systems" section.

10.1 Non-Homogeneous Poisson Process (NHPP) and the Power Law (Crow-AMSAA)

The Power Law NHPP is the standard model for repairable system reliability growth/degradation:

$$\text{ROCOF: } \rho(t) = \frac{\lambda\beta}{\theta}\left(\frac{t}{\theta}\right)^{\beta-1} = \lambda\beta t^{\beta-1}$$ $$\text{Expected cumulative failures: } \mathbb{E}[N(t)] = \left(\frac{t}{\theta}\right)^\beta = \lambda t^\beta$$

The Crow-AMSAA model (MIL-HDBK-189C) is the Power Law NHPP applied to reliability growth testing:

  • $\beta < 1$: ROCOF is decreasing — the system is improving (reliability growth, early-life failure elimination through design fixes)
  • $\beta = 1$: Constant ROCOF — Homogeneous Poisson Process, random independent failures
  • $\beta > 1$: ROCOF is increasing — the system is degrading (wear-out, accumulated damage)
from reliability.Repairable_systems import reliability_growth

# Cumulative failure times for a single system
failure_times = [5, 12, 21, 35, 42, 58, 67, 82, 93, 108]

rg = reliability_growth(times=failure_times, target_MTBF=15,
                         show_plot=True, print_results=True)
print(f"Crow-AMSAA: beta={rg.beta:.3f}, lambda={rg.Lambda:.4f}")
print(f"Current MTBF = {rg.current_MTBF:.1f} hrs")
print(f"Projected failures at 150 hrs: {rg.Lambda * 150**rg.beta:.1f}")

10.2 Mean Cumulative Function (MCF)

When you have failure data from multiple repairable systems (a fleet), the Mean Cumulative Function is the non-parametric estimate of the expected cumulative number of failures per system as a function of age:

$$\hat{M}(t) = \sum_{t_{(i)} \leq t} \frac{d_i}{n_i}$$

Unlike the Kaplan-Meier (which estimates survival from the first failure), the MCF counts all recurrent events. A concave MCF (decelerating failures) suggests reliability growth; a convex MCF (accelerating failures) indicates system degradation.

from reliability.Repairable_systems import MCF_nonparametric, MCF_parametric

# Multiple systems, each with their own failure history
system1 = [20, 45, 80, 130]   # failure ages of system 1
system2 = [10, 55, 100]        # failure ages of system 2
system3 = [30, 70]             # failure ages of system 3
censor  = [180, 150, 120]      # end of observation for each system

MCF_nonparametric(data=[system1, system2, system3], times_to_censoring=censor,
                  print_results=True, show_plot=True)

# Parametric Power Law MCF
MCF_parametric(data=[system1, system2, system3], times_to_censoring=censor,
               print_results=True, show_plot=True)

10.3 Optimal Replacement Time

For a repairable system with increasing ROCOF ($\beta > 1$), the optimal preventive replacement time $t^*$ minimises the long-run cost per unit time:

$$t^* = \arg\min_{t}\,\frac{C_p + C_f \cdot \hat{M}(t)}{t}$$ $$\text{where } C_p = \text{cost of planned replacement}, \quad C_f = \text{cost of failure (unplanned)}$$ 
from reliability.Repairable_systems import optimal_replacement_time

optimal_replacement_time(cost_PM=500, cost_CM=2000,
                          weibull_alpha=1000, weibull_beta=2.4,
                          show_plot=True, print_results=True)
# Returns t* (optimal interval) and the minimum cost rate
Key Distinction — Repairable vs Non-Repairable
  • Non-repairable: unit is discarded after failure. Analysis tool: Kaplan-Meier, Weibull. Question: "What is the time to first failure?"
  • Repairable: unit is restored after failure. Analysis tool: NHPP/Power Law, MCF. Question: "How does the failure rate evolve over system life?"
Mixing the two frameworks is a critical error. Fitting a Weibull to the inter-failure times of a repairable system assumes each repair restores the system to "as good as new" (Renewal Process assumption). The Power Law NHPP makes no such assumption — it models the actual event history without requiring perfect repair.

11. Remaining Useful Life and Prognostics

RUL estimation is a regression problem where the target is time-to-failure from the current observed system state. It differs from standard regression in two critical ways: the target is inherently censored (running systems have $T > t_{now}$), and the uncertainty in the prediction is as operationally important as the point estimate.

Pitfall — RUL Without Uncertainty
Reporting "RUL = 47 hours" without a confidence interval is insufficient for maintenance scheduling. The optimal maintenance decision depends on the cost ratio of false alarm (unnecessary replacement) vs. false miss (failure in service). Only a calibrated P10/P90 interval enables the correct cost-optimal decision. Point estimate RUL reports are unsuitable for operational use.

11.1 LSTM-Weibull Hybrid for Prognostics

  1. Health Index (HI): train an LSTM autoencoder on healthy operating data (vibration, temperature, pressure time series). The reconstruction error is the HI. HI ≈ 0 for new components; HI increases monotonically toward a threshold at failure.
  2. Monotonicity enforcement: apply PAVA isotonic regression post-processing, or add a monotone output layer (softplus cumulative sum) to enforce physically correct non-decreasing HI.
  3. Weibull trend fit: at each time step, fit the Weibull CDF to the observed (time, HI) trajectory. As more observations arrive, the fit stabilises.
  4. RUL estimation: the RUL is the extrapolated time for HI to reach the failure threshold $h^*$, given the fitted Weibull parameters. Propagate parameter uncertainty via Monte Carlo or likelihood ratio bounds to obtain P10/P90 RUL intervals.
  5. Online update: re-fit the Weibull parameters (sliding window MLE or Bayesian update) as new HI observations arrive.
import numpy as np
from scipy.optimize import minimize
from scipy.stats import weibull_min

def fit_weibull_to_hi(times, hi_values, threshold=1.0):
    """
    Fit Weibull CDF to health index trajectory.
    Returns beta, eta, and P10/P90 RUL.
    """
    def neg_loglik(params):
        beta, eta = np.exp(params)
        f_pred = 1 - np.exp(-(times/eta)**beta)
        return np.sum((hi_values/threshold - f_pred)**2)  # simplified WLS

    res = minimize(neg_loglik, x0=[np.log(2), np.log(max(times)*1.5)],
                   method='Nelder-Mead')
    beta, eta = np.exp(res.x)

    # RUL = time for CDF to reach 1.0 (full HI = threshold)
    # P10 (optimistic): 10th percentile of remaining life
    t_now = times[-1]
    t_failure_med = eta * (-np.log(1 - 0.5))**(1/beta)  # median life
    rul_p50 = max(0, t_failure_med - t_now)

    # Monte Carlo uncertainty propagation (simplified)
    return {'beta': beta, 'eta': eta, 'rul_p50': rul_p50}


# Example: monitor bearing health index
t = np.array([0, 50, 100, 150, 200, 250])
hi = np.array([0.02, 0.05, 0.12, 0.25, 0.45, 0.68])  # normalised

result = fit_weibull_to_hi(t, hi, threshold=1.0)
print(f"β={result['beta']:.2f}, η={result['eta']:.0f}, RUL≈{result['rul_p50']:.0f} hrs")

12. Physics of Failure: Fatigue, Fracture, Creep

Physics-of-failure (PoF) models provide the mechanistic foundation for reliability predictions when field failure data is scarce — as is typical for new designs, long-life components, and novel materials. PoF models are also essential for establishing the physical justification of ALT acceleration models.

12.1 High-Cycle Fatigue: S-N (Wöhler) Curves

For metals subjected to cyclic stress, the S-N curve relates applied stress amplitude $S$ to the number of cycles to failure $N_f$:

$$S = \sigma_f'(2N_f)^b \qquad \text{(Basquin's law, log-log linear)}$$ $$\ln N_f = \ln A - n\ln S \implies \text{life is Lognormal at fixed } S$$

The statistical S-N curve includes scatter in $N_f$ at each $S$ level. For design, use the P10 or P1 S-N curve (10% or 1% failure probability), not the mean. The Weibull distribution is often fitted to $N_f$ at each stress level, with $\beta$ typically in the range 2.5–4.5 for metallic fatigue.

12.2 Fracture Mechanics (Paris's Law)

For cracks growing under cyclic loading (damage-tolerant design):

$$\frac{da}{dN} = C(\Delta K)^m, \quad \Delta K = Y\Delta\sigma\sqrt{\pi a}$$ $$N_f = \int_{a_0}^{a_c} \frac{da}{C(Y\Delta\sigma\sqrt{\pi a})^m}$$

where $a$ is crack length, $a_0$ initial crack size, $a_c$ critical crack size, $Y$ geometry factor, $C$ and $m$ are material constants (Paris parameters). This integral gives the number of cycles from initial flaw to failure. The scatter in $C$, $m$, $a_0$ translates into a Lognormal failure time distribution, justifying the Lognormal distribution choice for crack-growth-dominated failures.

12.3 Miner's Rule for Cumulative Fatigue Damage

$$D = \sum_{i} \frac{n_i}{N_{f,i}} \qquad \text{Failure when } D \geq 1$$

where $n_i$ is the number of cycles at stress level $S_i$ and $N_{f,i}$ is the fatigue life at $S_i$ (from the S-N curve). Miner's rule is linear damage accumulation — it ignores sequence effects (high-to-low stress is unconservative; low-to-high is conservative). Modified rules (Corten-Dolan, Marco-Starkey) account for sequence effects but add parameters.

12.4 Creep Damage (Larson-Miller Parameter)

$$P_{LM} = T(C + \log t_r) \quad [\text{°R or K}] \qquad \text{(material constant } C \approx 20\text{ for Ni superalloys)}$$

The Larson-Miller parameter allows scaling creep rupture time across temperatures: at fixed stress, if $P_{LM}$ is constant, the rupture life scales accordingly. This provides the thermal acceleration model for high-temperature turbomachinery components — the PoF equivalent of the Arrhenius ALT model.

Connecting PoF to the Reliability Distribution
Physics-of-failure analysis gives you the acceleration model (which variable drives life) and the distribution family (multiplicative damage → Lognormal; weakest-link → Weibull). Don't fit distributions blindly to field data: use the PoF model to narrow your distribution choice to 1–2 candidates, then confirm with the probability plot and goodness-of-fit test. This combination of physical reasoning and statistical validation is the hallmark of mature reliability practice.

13. Stress-Strength Interference

Stress-strength interference (SSI) computes the probability that a random load (stress $S$) exceeds the random capacity (strength $X$) of a component. It is the fundamental tool for probabilistic structural design:

$$R = P(X > S) = \int_0^\infty F_S(x)\,f_X(x)\,dx = \int_0^\infty R_S(x)\,f_X(x)\,dx$$

For the Normal-Normal case: if $X \sim \mathcal{N}(\mu_X, \sigma_X^2)$ and $S \sim \mathcal{N}(\mu_S, \sigma_S^2)$, then:

$$R = \Phi\!\left(\frac{\mu_X - \mu_S}{\sqrt{\sigma_X^2 + \sigma_S^2}}\right) = \Phi(z), \quad z = \frac{\mu_X - \mu_S}{\sqrt{\sigma_X^2 + \sigma_S^2}}$$

For other distribution combinations, numerical integration or Monte Carlo is required:

from reliability.Other_functions import stress_strength

stress_strength(stress=dist_S, strength=dist_X, print_results=True, show_plot=True)

# Or manually via Monte Carlo:
import numpy as np
from scipy.stats import weibull_min, norm

n = 1_000_000
strength = norm.rvs(loc=500, scale=40, size=n)   # MPa
stress   = weibull_min.rvs(c=2.5, scale=200, loc=0, size=n)  # MPa

R = np.mean(strength > stress)
print(f"P(strength > stress) = {R:.6f}  →  PF = {1-R:.2e}")
Senior Practitioner Tip — SSI and Safety Factors
Traditional deterministic safety factors (SF = mean strength / mean stress) ignore distribution shapes. A safety factor of 2.0 can correspond to a failure probability anywhere from 10⁻² to 10⁻⁸ depending on the scatter in both distributions. Probabilistic SSI replaces the single SF number with a failure probability, which is directly comparable to reliability targets. When transitioning from deterministic to probabilistic design, the first step is always to characterise the scatter (CoV) of both stress and strength from test data.

14. System Reliability: Series, Parallel, RBD

14.1 Series Systems

All components must function for the system to function. System reliability decreases as components are added:

$$R_s(t) = \prod_{i=1}^n R_i(t), \qquad h_s(t) = \sum_{i=1}^n h_i(t)$$

The weakest component dominates. For a series system of $n$ identical components with Weibull life $(\eta, \beta)$:

$$R_s(t) = \exp\!\left[-n\left(\frac{t}{\eta}\right)^\beta\right] \implies \text{System Weibull: } \eta_s = \eta/n^{1/\beta}, \quad \beta_s = \beta$$

14.2 Parallel Systems (Redundancy)

System fails only when all components fail. Adding redundancy improves reliability but also changes the shape of the system failure time distribution:

$$F_s(t) = \prod_{i=1}^n F_i(t), \qquad R_s(t) = 1 - \prod_{i=1}^n [1-R_i(t)]$$
Redundancy Increases MTTF but not Indefinitely
A key result: for exponential components ($h = \lambda$), adding $n$ parallel units gives system MTTF = $\frac{1}{\lambda}(1 + \frac{1}{2} + \cdots + \frac{1}{n})$. This harmonic series grows slowly — doubling from 1 to 2 units adds 50% MTTF; going from 2 to 3 adds only 17%. The marginal reliability improvement from each additional redundant unit diminishes rapidly.

14.3 k-of-n Systems

The system functions if at least $k$ of $n$ components function (e.g., 2-of-3 voting, 3-of-4 engines):

$$R_s(t) = \sum_{j=k}^n \binom{n}{j} R(t)^j [1-R(t)]^{n-j}$$

14.4 Reliability Block Diagrams and Minimal Cut Sets

For complex systems, Reliability Block Diagrams (RBD) or Fault Trees are used to model the system structure. The key computational algorithm is the minimal cut set method: a cut set is a set of components whose failure causes system failure; a minimal cut set has no redundant elements. The system unreliability is approximated by:

$$Q_s(t) \approx \sum_j \prod_{i \in C_j} Q_i(t) \qquad \text{(rare event approximation for small } Q_i\text{)}$$

For dynamic systems with load sharing, standby redundancy, or common-cause failures, Monte Carlo system simulation is the most practical approach.

15. Reliability Testing and Sample Sizing

15.1 Demonstration Test Plans

A reliability demonstration test answers: "How many units must I test for how long, with zero failures, to demonstrate a given reliability target with a given confidence?"

$$\text{Zero-failure test: } n \geq \frac{\ln(1-C)}{\ln R_0} \qquad \text{at time } t_{test}$$ $$\text{or equivalently: } n \cdot t_{test}^{\beta} \geq \frac{-\ln(1-C)}{\eta^{-\beta}} \quad \text{for Weibull with known } \beta$$

For a Weibull with known $\beta$ and target $R_0 = 0.90$ at time $t_0$ with 90% confidence ($C = 0.90$):

from reliability.Reliability_testing import reliability_test_planner

# Weibull-based test plan: demonstrates B10 life >= 1000 hrs at 90% confidence
reliability_test_planner(
    test_duration=1500,    # hrs of testing per unit
    CI=0.9,                # confidence level
    lifetimes_by_units=True,
    target_reliability=0.9,  # demonstrate R >= 90%
    weibull_shape=2.5,     # known beta from prior data
    show_plot=False
)

15.2 Sequential Sampling and Wald SPRT

The Sequential Probability Ratio Test (SPRT) allows continuous monitoring of a test: after each observation, you decide to accept ($H_0$: reliability $\geq R_1$), reject ($H_1$: reliability $\leq R_0 < R_1$), or continue testing. This is dramatically more efficient than fixed-sample tests — average sample number is typically 40–60% of the equivalent fixed-sample test.

15.3 Proportion Confidence Intervals

For pass/fail test results (e.g., $k$ failures in $n$ units), use the Wilson or Clopper-Pearson interval — not the Wald (normal approximation) interval which fails catastrophically when $k$ is near 0 or $n$:

from reliability.Reliability_testing import one_sided_proportion_test

# Observed 0 failures in 30 units. Upper 95% CI on failure probability?
one_sided_proportion_test(trials=30, failures=0, CI=0.95)
# Returns: the demonstrated reliability with 95% confidence

16. Reliability-Based Design Optimisation (RBDO)

RBDO replaces deterministic design constraints $g(\mathbf{x}) \leq 0$ with probabilistic feasibility constraints, explicitly accounting for uncertainty in loads, material properties, and geometry:

$$\min_{\mathbf{x}} f(\mathbf{x}) \quad \text{s.t.} \quad P\bigl(g_k(\mathbf{x}, \mathbf{Z}) \leq 0\bigr) \geq \Phi(\beta_{target,k})$$

where $\mathbf{Z}$ are uncertain parameters (material properties, loads), $\beta_{target}$ is the target reliability index (e.g., $\beta = 3 \Rightarrow P_f \leq 1.35 \times 10^{-3}$; $\beta = 4.75$ for most aerospace structural components under FAR 25.571).

16.1 FORM and SORM (First/Second Order Reliability Methods)

FORM approximates the limit-state surface $g(\mathbf{u}) = 0$ in standard normal space by a hyperplane at the Most Probable Point (MPP), giving:

$$P_f \approx \Phi(-\beta_{HL}), \quad \beta_{HL} = \min_{\mathbf{u}: g(\mathbf{u})=0} \|\mathbf{u}\|_2 \quad \text{(Hasofer-Lind index)}$$

SORM improves on FORM by fitting a paraboloid at the MPP, correcting for limit-state curvature. Both methods require $\partial g/\partial \mathbf{u}$ (gradient) information — numerically via finite differences or analytically.

16.2 Surrogate-Assisted RBDO

For expensive simulations (FEM structural analysis, CFD), the inner reliability loop (FORM/SORM or Monte Carlo) is too expensive to embed in an outer optimisation loop. Surrogate-based alternatives:

  • GP-based PoF: fit a Gaussian process to predict $P_f(\mathbf{x})$ directly. Efficient acquisition functions (e.g., Expected Improvement on Reliability, EIR) guide the sampling. See Chapter 15.
  • Polynomial Chaos Expansion (PCE): project the response onto polynomial basis functions of the random inputs. PCE coefficients give moments and failure probabilities analytically.
  • Importance Sampling with GP: use the GP surrogate to identify the dominant failure region, then sample efficiently.

from scipy.stats import norm
from scipy.optimize import minimize

def form_reliability(g_func, mu, sigma, n_iter=100):
    """
    Simple FORM implementation for reliability index.
    g_func: limit state function g(x) < 0 is failure.
    mu, sigma: mean and std dev of random variables.
    """
    # Transform to standard normal space: u = (x - mu)/sigma
    u0 = np.zeros(len(mu))  # start at origin

    def g_in_u(u):
        x = mu + sigma * u
        return g_func(x)

    def objective(u):
        return np.sum(u**2)  # minimise distance from origin

    constraints = {'type': 'eq', 'fun': lambda u: g_in_u(u)}
    res = minimize(objective, u0, method='SLSQP', constraints=constraints)

    beta_HL = np.sqrt(res.fun)
    pf = norm.cdf(-beta_HL)
    return {'beta': beta_HL, 'pf': pf, 'mpp': res.x}

# Example: shaft bending, g(x) = x[0] - x[1]  (strength - load)
result = form_reliability(
    g_func=lambda x: x[0] - x[1],
    mu=np.array([500.0, 350.0]),    # mean strength, mean load [MPa]
    sigma=np.array([50.0, 40.0])    # std dev
)
print(f"Hasofer-Lind β = {result['beta']:.3f}  →  Pf = {result['pf']:.2e}")

17.1 Minimal Working Example: Full Reliability Analysis Pipeline

"""
Full reliability analysis pipeline using the reliability library.
Source: reliability.readthedocs.io
"""
from reliability.Fitters import Fit_Everything, Fit_Weibull_2P
from reliability.NonParametricEstimators import KaplanMeier
from reliability.Other_functions import stress_strength
from reliability.Distributions import Weibull_Distribution, Normal_Distribution
import matplotlib.pyplot as plt

# ── 1. Data ──────────────────────────────────────────────────────────────────
failures       = [482, 637, 712, 830, 955, 1040, 1190, 1380, 1600, 1820]
right_censored = [2000, 2000, 2000]

# ── 2. Non-parametric (no model assumption) ──────────────────────────────────
km = KaplanMeier(failures=failures, right_censored=right_censored,
                 CI=0.9, print_results=False, show_plot=True)

# ── 3. Fit all distributions, rank by BIC ────────────────────────────────────
best = Fit_Everything(failures=failures, right_censored=right_censored,
                      sort_by='BIC', show_probability_plot=False,
                      show_PP_plot=False, show_histogram_plot=False)
print(best.results[['Distribution','BIC','AD']].head(4))

# ── 4. Detailed Weibull fit with confidence bounds ───────────────────────────
w = Fit_Weibull_2P(failures=failures, right_censored=right_censored,
                    CI=0.9, CI_type='time')
print(f"\nWeibull: β={w.beta:.3f}, η={w.alpha:.0f} hrs")
print(f"B10 = {w.quantiles(0.1):.0f} hrs  [90% LR: {w.quantiles_lower_CI:.0f} – {w.quantiles_upper_CI:.0f}]")

# ── 5. Stress-strength interference ─────────────────────────────────────────
strength_dist = Weibull_Distribution(alpha=w.alpha, beta=w.beta)
stress_dist   = Normal_Distribution(mu=400, sigma=60)   # random load

result = stress_strength(stress=stress_dist, strength=strength_dist,
                         print_results=True, show_plot=True)

plt.show()

18. Key Takeaways

  • Know your failure mode before picking a distribution. Weibull for monotone hazard (wear, fatigue with known slope); Lognormal for multiplicative/diffusion damage (crack growth, electromigration); Exponential for random external events. Never default to Weibull blindly.
  • Never treat censored observations as failures. Right-censored units contribute $\ln R(c_i)$ to the log-likelihood, not $\ln f(c_i)$. The resulting underestimate of component life can be 2–5× — a catastrophic error in maintenance planning.
  • Use MLE with LR confidence bounds. LS on the probability plot is for graphical diagnostics only. Use Wald (Fisher matrix) bounds only when $n > 50$; use LR bounds otherwise.
  • Report $B_{10}$ life with confidence bounds, not just a point estimate. "B10 = 1200 hrs" is not an engineering output. "B10 = 1200 hrs, 90% LR: [820, 1750] hrs" is.
  • Separate failure modes before fitting. Mixed failure modes produce curved WPPs and meaningless single-distribution fits. Use failure cause codes from CMMS/FRACAS to segment data before analysis.
  • ALT extrapolation requires the same failure mode at accelerated and use stress. Always verify with failure analysis at each stress level. The acceleration factor uncertainty often dominates the total life prediction uncertainty.
  • Repairable vs non-repairable is a fundamental distinction. Crow-AMSAA / Power Law NHPP for recurrent failure histories on maintained systems. Weibull for time-to-first-failure on components.
  • RUL prediction requires a P10/P90 interval. The maintenance decision depends on the cost ratio of false alarm vs. false miss — only a calibrated interval enables the optimal decision.
  • Stress-strength interference connects load and strength distributions. Probabilistic SSI replaces safety factors with failure probabilities, enabling direct comparison to reliability targets.
  • Use the Python reliability library. It is the most comprehensive open-source toolkit for this domain and implements everything in this chapter correctly: reliability.readthedocs.io.
