TL;DR

  • For causal virtual sensors on ECU hardware (<1 ms budget): LSTM with 1–2 layers, hidden size 32–64.
  • For cloud analytics with long sequences (>200 steps): TCN or Transformer (PatchTST / TFT).
  • Always train on simulation error (multi-step rollout) when the model runs in closed-loop. One-step RMSE and simulation RMSE can differ by 5×.
  • Gradient clipping (max norm = 1.0) and AdamW are non-negotiable for recurrent models. Without clipping, LSTM training on engineering transients diverges within 50 epochs.
  • Grey-box (physics + ML residual) consistently outperforms pure black-box with half the training data.

1. LSTM: Architecture and Engineering Implementation

The LSTM cell maintains two state vectors: the cell state $\mathbf{c}_t \in \mathbb{R}^h$ (long-term memory) and the hidden state $\mathbf{h}_t \in \mathbb{R}^h$ (short-term context).

LSTM Forward Pass — let $[\mathbf{h}_{t-1}, \mathbf{x}_t]$ denote their concatenation: $$\mathbf{f}_t = \sigma(\mathbf{W}_f[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_f) \quad \text{(forget gate: what to erase from } \mathbf{c}_{t-1}\text{)}$$ $$\mathbf{i}_t = \sigma(\mathbf{W}_i[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_i) \quad \text{(input gate: what new info to write)}$$ $$\tilde{\mathbf{c}}_t = \tanh(\mathbf{W}_c[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_c) \quad \text{(candidate cell)}$$ $$\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \mathbf{i}_t \odot \tilde{\mathbf{c}}_t \quad \text{(cell state update)}$$ $$\mathbf{o}_t = \sigma(\mathbf{W}_o[\mathbf{h}_{t-1},\mathbf{x}_t] + \mathbf{b}_o) \quad \text{(output gate)}$$ $$\mathbf{h}_t = \mathbf{o}_t \odot \tanh(\mathbf{c}_t) \quad \text{(hidden state output)}$$

Why LSTM solves the vanishing gradient problem. Vanilla RNN backpropagation through time (BPTT) computes:

$$\frac{\partial \mathcal{L}}{\partial \mathbf{h}_0} = \prod_{t=1}^T \frac{\partial \mathbf{h}_t}{\partial \mathbf{h}_{t-1}} \cdot \frac{\partial \mathcal{L}}{\partial \mathbf{h}_T}$$

For $T > 50$ and $\|\partial \mathbf{h}_t/\partial \mathbf{h}_{t-1}\| < 1$, the product vanishes exponentially. The LSTM cell state path $\mathbf{c}_t = \mathbf{f}_t \odot \mathbf{c}_{t-1} + \ldots$ has gradient $\partial \mathbf{c}_t / \partial \mathbf{c}_{t-1} \approx \mathbf{f}_t$, which stays close to 1 for many timesteps when the forget gate saturates high.

2. Simulation Error vs. One-Step Prediction Error

Pitfall
Training on one-step MSE for dynamic models.

A model trained on $\mathcal{L}_{1step} = \sum_t (y_t - \hat{y}(y_{t-1},\ldots,x_{t-1},\ldots))^2$ performs well at forecasting $t+1$ but diverges rapidly in closed-loop simulation. Accumulated error grows $\mathcal{O}(T^2)$ for mismatched dynamics. Always test simulation RMSE (100+ step rollout with no teacher forcing). If simulation RMSE >5× one-step RMSE, the model is not suitable for dynamic simulation.

The fix: train on simulation (rollout) error directly, backpropagating through the entire window:

$$\mathcal{L}_{rollout} = \sum_{t=1}^{T_{rollout}} \bigl\| y_t - \hat{y}_t^{(rollout)} \bigr\|^2, \quad \hat{y}_{t+1}^{(rollout)} = f_\theta(\hat{y}_t^{(rollout)}, x_t)$$ 
import torch

def rollout_loss(model, X_seq, Y_seq, T_rollout=20):
    batch, T, d_in = X_seq.shape
    loss = torch.tensor(0.0)
    h = None

    # Teacher forcing for first (T - T_rollout) steps to stabilise training
    for t in range(T - T_rollout):
        y_hat, h = model(X_seq[:, t:t+1, :], h)
        h = h.detach() if isinstance(h, torch.Tensor) else \
            tuple(hi.detach() for hi in h)

    # Rollout window: no teacher forcing, backprop through all T_rollout steps
    y_prev = Y_seq[:, T - T_rollout - 1:T - T_rollout, :]
    for t in range(T - T_rollout, T):
        x_t = torch.cat([X_seq[:, t:t+1, :], y_prev], dim=-1)
        y_hat, h = model(x_t, h)
        loss += torch.nn.functional.mse_loss(y_hat.squeeze(1), Y_seq[:, t, :])
        y_prev = y_hat

    return loss / T_rollout

# Curriculum: start T_rollout=1, increase by 1 every 20 epochs
for epoch in range(n_epochs):
    optimizer.zero_grad()
    T_roll = min(1 + epoch // 20, T_max_rollout)
    loss = rollout_loss(model, X_batch, Y_batch, T_rollout=T_roll)
    loss.backward()
    torch.nn.utils.clip_grad_norm_(model.parameters(), 1.0)
    optimizer.step()
Rollout Training Protocol
  1. Curriculum: start $T_{rollout}=1$ and increase by 1 every 20 epochs. Direct training with large $T_{rollout}$ from epoch 0 causes vanishing gradients.
  2. Gradient clipping: always clip at norm 1.0 when backpropagating through long rollout windows.
  3. Validation diagnostic: report both one-step RMSE and 100-step simulation RMSE. Ratio >3× indicates rollout window too short during training.
  4. TCN advantage: TCN gradients through rollout avoid LSTM vanishing-gradient issues since convolutions are fully differentiable without recurrence.

3. Temporal Convolutional Networks (TCN)

A TCN uses dilated causal convolutions — causal ensures no future leakage, dilation exponentially expands the receptive field:

$$\text{Dilated causal convolution: } (f *_d g)(t) = \sum_{j=0}^{k-1} w_j \cdot x_{t - d \cdot j}$$ $$\text{Receptive field with } L \text{ layers, dilation } d_\ell = 2^\ell: \quad \text{RF} = 1 + 2(k-1)(2^L - 1)$$ $$\text{Example: } k=3, L=8 \Rightarrow \text{RF} = 1021 \text{ timesteps at } \mathcal{O}(1) \text{ depth}$$

Key TCN advantages over LSTM: fully parallelisable during training, no hidden state to initialise at inference, gradients flow cleanly through dilated conv layers without vanishing.

4. Transformers and Self-Attention

The self-attention mechanism allows the model to attend to the entire input sequence simultaneously, solving the long-range dependency problem that limits LSTMs to ~200 steps.

Scaled dot-product attention: $$\mathbf{Q} = \mathbf{X}\mathbf{W}^Q, \quad \mathbf{K} = \mathbf{X}\mathbf{W}^K, \quad \mathbf{V} = \mathbf{X}\mathbf{W}^V$$ $$\text{Attention}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \text{softmax}\!\left(\frac{\mathbf{Q}\mathbf{K}^\top}{\sqrt{d_k}}\right)\!\mathbf{V}$$

Physical interpretation (NOx prediction from $(T_{peak}, p_{cyl}, \lambda, \dot{m})$):

  • Query at timestep $t$: "which past states are relevant for predicting NOx now?"
  • Key at $t'$: "what information is available here to answer queries?"
  • Value at $t'$: the actual content to retrieve.
  • The softmax produces attention weights $a_{tt'} \geq 0$, $\sum_{t'} a_{tt'} = 1$. High $a_{tt'}$ means $t'$ strongly drives the prediction at $t$.
  • Scaling by $1/\sqrt{d_k}$ prevents dot products from saturating the softmax.

Causal Masking for Autoregressive Prediction

Pitfall — Data Leakage
Without causal masking, the model attends to future timesteps during training. For forecasting applications, set the upper triangle of the attention matrix to $-\infty$ before softmax: $$\tilde{a}_{tt'} = \begin{cases} \mathbf{q}_t^\top \mathbf{k}_{t'} / \sqrt{d_k} & \text{if } t' \leq t \\ -\infty & \text{if } t' > t \end{cases}$$ After softmax, $a_{tt'} = 0$ for all $t' > t$. Default for all virtual sensor and forecasting applications.

Multi-Head Attention

$$\text{MultiHead}(\mathbf{Q},\mathbf{K},\mathbf{V}) = \text{Concat}(\text{head}_1, \ldots, \text{head}_H)\mathbf{W}^O$$ $$\text{where } \text{head}_h = \text{Attention}(\mathbf{Q}\mathbf{W}^Q_h, \mathbf{K}\mathbf{W}^K_h, \mathbf{V}\mathbf{W}^V_h)$$

Different heads specialise: one may capture short-range temporal correlations (adjacent timesteps), another long-range dependencies (periodic injection events), another cross-variable correlations ($T_{peak}$–$\lambda$ interaction affecting NOx). Standard config: $H=8$ heads, $d_{model}=256$, $d_k = d_v = 32$.

Positional Encoding

Attention is permutation-invariant — positional encoding injects temporal position:

$$\text{Sinusoidal: } PE(t, 2i) = \sin\!\left(\frac{t}{10000^{2i/d_{model}}}\right), \quad PE(t, 2i+1) = \cos\!\left(\frac{t}{10000^{2i/d_{model}}}\right)$$

Alternatives: Learnable embeddings (preferred for fixed-length windows); RoPE (rotary, encodes relative positions — preferred when generalising to sequence lengths not seen during training).

Architecture Selection Rule
  • Sequence length $T > 200$: Transformer (PatchTST or Informer)
  • Multi-step forecasting with mixed static/dynamic covariates: Temporal Fusion Transformer (TFT)
  • Real-time ECU deployment, strict memory (<256 KB): LSTM or TCN
  • Small dataset ($n < 5{,}000$ windows): LSTM; Transformers underfit
  • Need interpretability (which timesteps drive prediction?): Transformer with attention weight visualisation
  • Grey-box hybrid with physics residual: LSTM (simpler concatenation)

6. Engineering-Specific Transformer Variants

Informer (Zhou et al., 2021)

Sparse self-attention: selects only top-$u$ queries based on a statistical importance measure. Reduces complexity from $\mathcal{O}(T^2)$ to $\mathcal{O}(T \log T)$. Effective for long-horizon forecasting (SCADA, energy demand, degradation trajectories).

PatchTST (Nie et al., 2023)

Divides the time series into non-overlapping patches of size $P$ and treats each patch as a token. Reduces sequence length from $T$ to $T/P$, dramatically reducing attention cost. State-of-the-art on many multivariate forecasting benchmarks.

Temporal Fusion Transformer (TFT, Lim et al., 2021)

Designed for multi-horizon forecasting with static and dynamic covariates. Combines LSTM-based processing for local patterns with self-attention for long-range dependencies. Includes variable selection networks (attention-based feature importance), gating mechanisms, and quantile forecasting heads. Recommended for SCADA-based energy forecasting and multi-step process control.

Example — Transformer for Combustion Engine NOx Forecasting

Problem: predict NOx 500 ms ahead (50-step at 10 Hz) from engine variables $(n, \text{MAP}, T_{cyl}, \lambda, m_f)$ ($d=5$). Long-horizon requirement makes LSTM insufficient (forgets injection events 400 ms ago).

Architecture: PatchTST, patch size $P=5$, $H=4$ attention heads, $d_{model}=128$, 3 encoder blocks, dropout 0.1. With patches: 10 tokens from 50 timesteps.

Training: 8,000 drive cycle windows, GroupKFold by test bench session. AdamW, $\eta_0=5\times10^{-4}$, cosine LR, 100 epochs.

Result: PatchTST RMSE = 5.8 mg/km vs. LSTM RMSE = 9.2 mg/km. Attention weights correctly focus on injection events 300–400 ms before the NOx measurement.

Deployment: 10-token attention is small enough for inference on an automotive domain controller (Aurix TC397); not deployable on a basic MCU.

7. The Grey-Box Approach for Engineering Dynamics

The most reliable engineering time series model combines known physics with a data-driven correction:

$$\hat{y}(t) = \underbrace{\hat{y}_{phys}(t;\,\boldsymbol{\theta}_p)}_{\text{physics model}} + \underbrace{\hat{\delta}_{ML}(\mathbf{x}_{t-W:t};\,\boldsymbol{\theta}_{ML})}_{\text{ML residual}}$$
Example — Grey-Box NOx Virtual Sensor: Zeldovich + LSTM

Physics component: simplified Zeldovich thermal NOx rate $\dot{n}_{NO} = A\exp(-E_a/RT_{peak})[O_2]^{0.5}[N_2]$, integrated over one engine cycle. Captures 70% of NOx variance with 0 training data.

ML component: LSTM (2 layers, hidden=64) trained on the residual $y_{NOx,meas} - y_{NOx,Zeldovich}$ from 800 drive cycles. Corrects for transient effects (charge cooling, wall wetting, combustion phasing dynamics) not captured by the steady-state model.

Result: Zeldovich alone: MAPE = 18.3%. Pure LSTM: MAPE = 12.1%. Grey-box: MAPE = 6.8% — below the 10% regulatory threshold with half the training data of the pure LSTM approach.

Lesson
The grey-box pattern is not optional for regulated engineering domains — it provides physical bounds on extrapolation behaviour, interpretable failure modes ("the physics is right but transient dynamics are wrong"), and dramatically reduced data requirements. The LSTM corrects; the physics model anchors.

8. Key Takeaways

  • Train on simulation error whenever the model will run in closed-loop — one-step MSE is a trap.
  • Curriculum rollout: start $T_{rollout}=1$, increase by 1 every 20 epochs.
  • LSTM default: hidden=64, 2 layers, AdamW, grad clip=1.0, cosine LR. This covers 80% of ECU virtual sensor problems.
  • TCN: use when parallelism matters and sequences are long; receptive field = $1 + 2(k-1)(2^L-1)$.
  • Transformers: prefer for $T > 200$ and $n > 5{,}000$ windows; always apply causal masking for forecasting.
  • PatchTST: reduces attention cost from $\mathcal{O}(T^2)$ to $\mathcal{O}((T/P)^2)$; state-of-the-art for long sequences.
  • Grey-box: physics model anchors extrapolation, ML corrects transient dynamics. Almost always beats pure black-box on engineering data.
  • Bidirectional LSTM is forbidden for causal (real-time) applications — it violates causality.
