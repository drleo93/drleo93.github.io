Ch 6: Deep Learning for Engineering Regression

Chapter 6 of 23  ·  ~35 min read  ·  Deep learning is a hammer — make sure your problem is a nail
MLP ResNet PINN AdamW LayerNorm residual blocks regularisation CNN Optuna

TL;DR

  • For static 0D tabular regression below 10,000 samples, deep learning is almost never the best first choice. GP or XGBoost will win.
  • DL crossover point: MLP becomes competitive above \(n \approx 5{,}000\) for tabular data; always competitive for structured inputs (time series, fields, spectra).
  • AdamW + cosine LR + gradient clipping + early stopping is the universal starting recipe. Non-negotiable.
  • LayerNorm over BatchNorm for engineering: works with batch size = 1 (ECU inference), stateless, robust to distribution shift.
  • Pre-Norm residual blocks (\(\mathbf{h}_{out} = \mathbf{h}_{in} + f(\mathrm{LN}(\mathbf{h}_{in}))\)) for \(L > 4\) layers. Post-Norm creates vanishing gradient in deep networks.
  • PINNs can learn accurate field predictions from very sparse measurements — but convergence is fragile. Always cross-validate against held-out physical measurements.

Deep learning is justified when at least one of the following holds:

  1. \(n > 10{,}000\) samples and XGBoost performance has plateaued
  2. Input is a high-dimensional structured object: time series, 2D/3D field, vibration spectrum, mesh
  3. Physics structure can be encoded in the architecture (PINNs, monotone NNs, port-Hamiltonian NNs)
  4. Transfer learning from a large pre-trained model is possible

2. MLP Architecture for Engineering Regression

Anatomy

$$\mathbf{h}_l = \sigma\!\left(\mathbf{W}_l \mathbf{h}_{l-1} + \mathbf{b}_l\right), \quad l = 1,\ldots,L, \qquad \hat{y} = \mathbf{W}_{L+1} \mathbf{h}_L + \mathbf{b}_{L+1}$$

Output layer is linear (no activation) for regression. For bounded outputs: sigmoid for \([0,1]\) targets (SoC, efficiency); softplus for non-negative outputs (rates, distances).

Depth and Width

  • Default start: 3 hidden layers, 128 units each
  • \(n > 10{,}000\): try 4–5 layers, 256–512 units
  • \(n < 5{,}000\): 2 layers, 64–128 units, heavy dropout
  • Prefer constant width (all layers same) over tapering

Activation Functions

FunctionFormulaBest for
ReLU\(\max(0,x)\)Shallow networks; ECU deployment
GELU\(x\Phi(x)\)Deep networks; tabular data; Transformers
SiLU/Swish\(x\sigma(x)\)Modern MLPs; time series
Sigmoid\(1/(1+e^{-x})\)Output only for \([0,1]\) targets

Pitfall — ReLU in PINNs
For physics-informed networks (PINNs), GELU or SiLU are mandatory. Their second derivatives \(\partial^2 f/\partial x^2\) are non-zero and smooth everywhere — required for PDE residuals involving second-order spatial derivatives (diffusion terms, Laplacians). ReLU has zero second derivative almost everywhere and a Dirac delta at \(x=0\), making it unsuitable for computing \(\nabla^2 u\) in a PDE loss.

Lesson — LayerNorm for engineering deployment
LayerNorm does not depend on batch statistics at inference time. It works correctly with batch size = 1 (single-sample ECU inference), and is stable under distribution shift between training and deployment conditions. BatchNorm at inference uses running statistics from training — if the deployment distribution shifts (new sensor, different operating regime), BatchNorm statistics become incorrect while LayerNorm adapts automatically.

4. Residual Connections (Pre-Norm)

The Pre-Norm residual block — the standard in all modern production models:

$$\mathbf{h}_{out} = \mathbf{h}_{in} + f(\mathrm{LN}(\mathbf{h}_{in}))$$

Post-Norm (\(\mathrm{LN}(\mathbf{h}_{in} + f(\mathbf{h}_{in}))\)) normalises the output including the residual, creating a vanishing gradient problem in deep networks. Pre-Norm leaves the skip connection gradient path clean.

class ResidualBlock(nn.Module):
    def __init__(self, width, dropout):
        self.norm = nn.LayerNorm(width)
        self.fc1  = nn.Linear(width, 4 * width)   # 4x expansion (MLP block)
        self.fc2  = nn.Linear(4 * width, width)
        self.act  = nn.GELU()
        self.drop = nn.Dropout(dropout)

    def forward(self, x):
        residual = x
        x = self.norm(x)           # Pre-Norm: normalise before branch
        x = self.fc2(self.drop(self.act(self.fc1(x))))
        return x + residual        # skip connection: clean gradient path

In PyTorch Transformer: norm_first=True in nn.TransformerEncoderLayer. Use pre-norm for all engineering regression networks with \(L > 4\) layers.

5. Training Recipe

Rule — Universal training starting configuration
  1. Optimiser: AdamW with \(\eta_0 = 10^{-3}\), weight decay \(\lambda = 10^{-4}\). Never Adam + L2 penalty (AdamW decouples weight decay from gradient adaptation).
  2. LR schedule: cosine annealing from \(\eta_0\) to \(\eta_{min} = 10^{-5}\).
  3. Gradient clipping: clip_grad_norm_(params, max_norm=1.0) before each optimizer step.
  4. Early stopping: patience = 20–50 epochs on validation loss. Non-negotiable.
  5. Batch size: 256–512 for \(n > 5{,}000\); 32–64 for smaller datasets.
For PINNs: use L-BFGS with full-batch mode for final fine-tuning (Adam for initial convergence).

Regularisation Hierarchy

  1. Weight decay (L2) in AdamW: \(\lambda \in [10^{-5}, 10^{-2}]\). Include in every model. Always ON.
  2. Early stopping: always ON. The training checkpoint at minimum validation loss is the deployed model.
  3. Dropout: rate \(p \in [0.1, 0.5]\). Higher for smaller datasets. Apply after LayerNorm. Disable at inference.
  4. Data augmentation: for time series, add Gaussian noise (\(\sigma \approx 0.01\sigma_{signal}\)) + random time warping. For tabular, Mixup interpolation.
  5. Label smoothing: for classification, replace hard targets with \(y = 1 - \varepsilon + \varepsilon/K\), \(\varepsilon \in [0.05, 0.15]\).
Pitfall — Adding too many regularisers simultaneously
Never apply more than 2 regularisers simultaneously. Debugging becomes impossible when weight decay, dropout, label smoothing, and data augmentation all interact. Add regularisers one at a time, measure the effect of each.

6. CNN for Structured Inputs

When the input has spatial or spectral structure, a CNN exploits translation invariance to reduce parameters and improve generalisation.

  • 1D CNN for vibration and process signals: stacked dilated convolutions (TCN-style) give receptive field \(\mathcal{O}(2^L \cdot K)\) with \(\mathcal{O}(L \cdot K)\) parameters — correct for vibration spectra, combustion pressure traces, mass flow transients
  • 2D CNN for structured field surrogates: U-Net combining encoder (downsampling + max-pool) and decoder (upsampling + skip connections) for 2D flow field prediction on uniform grids
  • When NOT to use CNNs: tabular data with no spatial structure (use MLP/XGBoost); irregular sensors/unstructured meshes (use GNN, Ch. 18); long sequences with non-local dependencies (use Transformer, Ch. 10)

7. Physics-Informed Neural Networks (PINNs)

PINNs embed the governing PDE as a residual loss term:

$$L_{PIML}(\boldsymbol{\theta}) = w_d\,L_{data} + w_r\,L_{phys} + w_b\,L_{bc}$$
$$L_{data} = \frac{1}{n_d}\sum_i (u_{\boldsymbol{\theta}}(\mathbf{x}_i) - u_i)^2, \qquad L_{phys} = \frac{1}{n_r}\sum_j \mathcal{N}[u_{\boldsymbol{\theta}};\mathbf{x}_j]^2$$
Example — Packed-bed reactor temperature profile (PINN vs. MLP)
8 thermocouple measurements along a 2 m packed-bed methanol synthesis reactor. Governing equation known: \(\rho c_p u \partial T/\partial z = \lambda_{eff}\partial^2T/\partial z^2 + (-\Delta H_r)r(T,p)\).

PINN trained on 8 measurements + 500 collocation points: RMSE = 4 K, hot-spot location correct, energy balance to ±0.5%.
Pure data-driven MLP: RMSE = 19 K, hot-spot location wrong.

The PINN extrapolates physically because the PDE constrains the solution everywhere; the MLP can only interpolate between the 8 measurement points.

PINN Training Challenges

  • Loss weight imbalance: \(L_{data} \gg L_{PDE}\) in early training. Use adaptive weighting: \(w_r = L_{data} / L_{PDE}\).
  • Spectral bias: standard MLPs learn low-frequency components first. Use Fourier feature embedding \(\gamma(\mathbf{x}) = [\cos(2\pi\mathbf{B}\mathbf{x}), \sin(2\pi\mathbf{B}\mathbf{x})]\) to resolve high-frequency components.
  • Stiff equations: use L-BFGS instead of Adam for final fine-tuning; curriculum training (increase collocation points progressively).
  • Convergence not guaranteed: always cross-validate against a few held-out physical measurements.

8. Hyperparameter Optimisation with Optuna

def objective(trial):
    depth    = trial.suggest_int("depth", 2, 6)
    width    = trial.suggest_categorical("width", [64,128,256,512])
    dropout  = trial.suggest_float("dropout", 0.0, 0.4)
    lr       = trial.suggest_float("lr", 1e-4, 1e-2, log=True)
    wd       = trial.suggest_float("wd", 1e-5, 1e-2, log=True)
    act      = trial.suggest_categorical("act", ["gelu","silu"])

    model = MLP(depth, width, dropout, act)
    train(model, AdamW(lr=lr, weight_decay=wd),
          scheduler=CosineAnnealingLR, early_stopping=20)
    return validation_rmse

study = optuna.create_study(
    direction="minimize",
    sampler=optuna.samplers.TPESampler(seed=42),
    pruner=optuna.pruners.MedianPruner()  # 30% compute savings
)
study.optimize(objective, n_trials=100)
Rule — Optuna search budget
  • XGBoost: 50–100 trials (fast evaluation)
  • MLP with \(n < 5000\): 30–50 trials; each trial <30 s
  • LSTM with \(n \approx 100k\): 20–30 trials; use GPU
  • MedianPruner: prunes trials whose validation loss exceeds the median of completed trials at the same epoch — ~30% compute savings
  • Log all trials to a database (storage=RDB URL): enables resuming interrupted studies

Chapter Summary

Key takeaways
  • DL crossover at \(n \approx 5{,}000\) for tabular data. Below this: GP or XGBoost wins. Above: evaluate both.
  • AdamW + cosine LR + gradient clipping + early stopping: mandatory starting configuration. Deviating without reason is not justified.
  • LayerNorm is the default normalisation layer for engineering surrogates — stateless, batch-size-agnostic, robust to distribution shift.
  • Pre-Norm residual blocks for \(L > 4\) layers. Post-Norm creates vanishing gradient. This is established engineering practice, not a nuance.
  • PINNs: use when physics is known and data is sparse. They outperform data-driven models in these regimes, but convergence requires careful loss balancing and curriculum training.
  • ReLU must not be used in PINNs — zero second derivative breaks PDE residuals. Use GELU or SiLU.
