TL;DR

  • Six architectures compete for engineering field prediction: POD-ROM, PINN, U-Net, FNO, DeepONet, GNN/Transolver. None is universally best — selection depends on data volume, mesh type, and PDE knowledge.
  • Selection rule: $n_s < 200$ simulations → POD-GP; structured mesh → FNO; unstructured + varying geometry → GNN or Transolver; sparse data + known PDE → PINN.
  • Neural operators (FNO, DeepONet) are discretisation-invariant: trained once, evaluated at any mesh resolution without retraining.
  • Transformer-based operators (Transolver, OFormer) apply attention over $K$ compressed tokens instead of $N$ mesh points — $\mathcal{O}(NK)$ vs $\mathcal{O}(N^2)$.
  • POD: choose $K$ to retain ≥99.9% cumulative energy ($\sum_{k=1}^K \sigma_k^2 / \sum \sigma_k^2 \geq 0.999$). For smooth aerodynamics: $K \approx 10$–30 modes.

1. 3D Engineering Data: Types and Challenges

Three fundamental challenges distinguish 3D field surrogates from 0D scalar surrogates:

  1. Dimensionality: $n_{outputs} = n_{mesh\,cells} \times n_{fields} \sim \mathcal{O}(10^6)$. Direct regression is infeasible.
  2. Mesh sensitivity: outputs are defined on a mesh. A different mesh (refined near a boundary) produces an incomparable output vector.
  3. Physical constraints: divergence-free velocity $\nabla\cdot\mathbf{u} = 0$, non-negative temperature, continuity of stress across interfaces.

3. Proper Orthogonal Decomposition (POD-ROM)

Given a snapshot matrix $\mathbf{S} \in \mathbb{R}^{N_h \times n_s}$ ($N_h$ DOFs, $n_s$ snapshots), the POD basis is found via SVD:

$$\mathbf{S} = \mathbf{U}\boldsymbol{\Sigma}\mathbf{V}^\top, \quad \boldsymbol{\phi}_k = \mathbf{u}_k \in \mathbb{R}^{N_h}$$ $$\text{POD coefficients: } \boldsymbol{\alpha}(\boldsymbol{\mu}) = \mathbf{U}_{:,1:K}^\top \mathbf{s}(\boldsymbol{\mu}) \in \mathbb{R}^K \quad (K \ll N_h)$$ $$\text{Non-intrusive ROM: } \hat{\mathbf{s}}(\boldsymbol{\mu}) = \mathbf{U}_{:,1:K}\hat{f}(\boldsymbol{\mu})$$

$\hat{f}$ is a GP or MLP trained on pairs $\{(\boldsymbol{\mu}_i, \boldsymbol{\alpha}_i)\}$. Choose $K$ to retain ≥99.9% cumulative energy. For smooth aerodynamics: $K \approx 10$–30 modes. For combustion with sharp gradients: $K \approx 100$–500.

Handling Varying Meshes Across CFD Snapshots

The POD snapshot matrix requires all snapshots on the same mesh. When blade morphing or AMR changes the mesh between CFD runs, project all snapshots onto a canonical reference grid using RBF interpolation:

import numpy as np
from scipy.interpolate import RBFInterpolator

def project_to_reference(snapshots_list, coords_list, ref_coords):
    """
    Project CFD snapshots from varying meshes to a canonical reference grid.
    snapshots_list: list of (n_nodes_i, n_fields) arrays
    coords_list:    list of (n_nodes_i, 3) coordinate arrays
    ref_coords:     (N_ref, 3) reference grid coordinates
    """
    n_snap = len(snapshots_list)
    n_ref, n_fields = ref_coords.shape[0], snapshots_list[0].shape[1]
    S_ref = np.zeros((n_ref, n_fields, n_snap))
    for i, (snap, coords) in enumerate(zip(snapshots_list, coords_list)):
        rbf = RBFInterpolator(coords, snap,
                              kernel='thin_plate_spline', smoothing=0.0)
        S_ref[:, :, i] = rbf(ref_coords)
    return S_ref   # shape: (N_ref, n_fields, n_snap)
Lesson
For Transolver and FNO, the projection step is unnecessary — they natively handle unstructured and varying meshes (FNO: spectral domain; Transolver: point cloud attention). Use the native mesh coordinates as positional inputs directly.

4. Physics-Informed Neural Networks (PINN)

PINNs minimise a composite loss combining observed data and the PDE residual at unlabelled collocation points:

$$\mathcal{L} = \underbrace{\lambda_d \sum_{i\in\mathcal{D}}\|u_\theta(\mathbf{x}_i) - u_i\|^2}_{\text{data loss}} + \underbrace{\lambda_r \sum_{j\in\mathcal{R}}\|\mathcal{N}[u_\theta](\mathbf{x}_j)\|^2}_{\text{PDE residual loss}}$$

The PDE operator $\mathcal{N}[u_\theta]$ is evaluated via automatic differentiation — no mesh required. Collocation points $\mathbf{x}_j^c$ are sampled anywhere in the domain without labels.

Pitfall — PINN Loss Weighting
The balance between $\lambda_d$ and $\lambda_r$ is critical and problem-dependent. If $\lambda_r \gg \lambda_d$, the network satisfies the PDE but ignores the data (underfitting). If $\lambda_d \gg \lambda_r$, physics is not enforced. Start with $\lambda_d = \lambda_r = 1$; use self-adaptive weights (SA-PINN) or gradient normalisation (GradNorm) for automatic tuning.

5. Neural Operators: Learning Mappings Between Function Spaces

Classical ML maps finite vectors $\mathbb{R}^n \to \mathbb{R}^m$. Neural operators learn mappings between function spaces $\mathcal{G}^\dagger: \mathcal{A} \to \mathcal{U}$ — making them discretisation-invariant: once trained, evaluate at any mesh resolution without retraining.

Fourier Neural Operator (FNO)

FNO applies its integral kernel in the Fourier domain, truncating to the $K_{max}$ lowest-frequency modes:

$$v_{t+1}(\mathbf{x}) = \sigma\!\left(W v_t(\mathbf{x}) + \mathcal{F}^{-1}\!\left(R_\phi \cdot \mathcal{F}(v_t)\right)(\mathbf{x})\right)$$

where $R_\phi$ are learnable complex weights (applied to the $K_{max}$ Fourier modes) and $W$ is a local linear transform. Requires structured uniform grids. Spectral truncation provides implicit regularisation and $\mathcal{O}(N \log N)$ complexity via FFT.

DeepONet: Deep Operator Network

DeepONet separates the operator learning into two sub-networks:

  • Branch network: encodes the input function $a$ (evaluated at $m$ sensor locations) → trunk-compatible embedding.
  • Trunk network: encodes the query location $\mathbf{y}$ where the output $u(\mathbf{y})$ is evaluated.
  • Output: $u_\theta(\mathbf{y}) = \sum_{k=1}^p b_k(a) \cdot t_k(\mathbf{y})$ (dot product of branch and trunk outputs).

Works with irregular sensor locations — ideal for sparse experimental data combined with a physics-informed constraint.

Graph Neural Networks for Unstructured Meshes

GNN/MeshGraphNet represents the CFD mesh as a graph: nodes = mesh points, edges = mesh connectivity. Message passing aggregates information from geometric neighbours, respecting the mesh topology. Each message-passing step propagates information by one edge hop — stack $L$ layers to cover a neighbourhood of radius $L$ edges.

Transolver: Token-Based Transformer for Physics

Transolver compresses the $N$-point mesh into $K \ll N$ physics tokens (learned via attention-weighted pooling), then applies full self-attention over the $K$ tokens:

$$\text{Complexity: } \mathcal{O}(NK) \text{ vs } \mathcal{O}(N^2) \text{ for full attention}$$

Each token captures a coherent physical region (boundary layer, wake, separation bubble) rather than a fixed spatial patch. State-of-the-art on Navier-Stokes and elasticity benchmarks as of 2024.

6. Key Takeaways

  • POD-ROM: the production-ready baseline. SVD on the snapshot matrix; GP or MLP maps parameters to POD coefficients. Choose $K$ for 99.9% energy retention.
  • PINN: mesh-free, data-efficient, physically consistent. Requires the PDE to be known. Slow training; tuning $\lambda_d/\lambda_r$ is critical.
  • FNO: discretisation-invariant, $\mathcal{O}(N \log N)$, requires structured grids. Best for parametric PDE families (Navier-Stokes, heat equation).
  • DeepONet: branch + trunk network architecture; works with irregular sensor locations. Good for operator learning from sparse experimental data.
  • GNN/MeshGraphNet: topology-respecting message passing on unstructured meshes. Handles arbitrary geometry without projection.
  • Transolver: $\mathcal{O}(NK)$ attention over physics tokens. SOTA accuracy; requires $n_s \geq 1{,}000$ simulations.
  • All architectures can be wrapped with conformal prediction for field-level uncertainty quantification (prediction intervals on the full field output).
