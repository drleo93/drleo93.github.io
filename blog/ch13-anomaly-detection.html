TL;DR

  • Any ML anomaly detector must beat SPC baseline first — CUSUM/EWMA are extremely powerful for scalar statistics and often all you need.
  • Three anomaly types require different detectors: point (CUSUM, Z-score), collective (LSTM autoencoder), contextual (conditional autoencoder).
  • Isolation Forest for multivariate point anomalies: $\mathcal{O}(n \log n)$, no distance metric, works in high dimensions. Preferred over One-Class SVM for $d > 5$.
  • LSTM autoencoder: train on normal sequences only; anomaly score = reconstruction MSE. Set threshold at 99.5th percentile of normal training scores (FAR = 0.5%).
  • Bearing faults: use spectral kurtosis to find the resonance band, then envelope analysis at $f_{BPFO/BPFI/BSF}$ harmonics.
  • VAE regularises the latent space — prevents memorising outliers. Better than deterministic AE for anomaly detection.

1. Anomaly Types and Detection Strategy

Engineering anomalies are classified along two dimensions:

  • By temporal structure: Point — single timestep. Collective — a subsequence (e.g., compressor surge onset). Contextual — normal globally, anomalous given context (high coolant temperature at full load is normal; at idle it is an anomaly).
  • By mechanism: Sensor fault — drift, bias, spike, stuck. Component degradation — bearing wear, fouling, seal leakage. Process upset — external disturbance, wrong setpoint, feedstock change.

2. Statistical Process Control: The Baseline

Pitfall
Never skip SPC as a baseline. If your LSTM autoencoder cannot outperform a well-tuned CUSUM chart, your model is not adding value. SPC charts are auditable, have formal statistical guarantees, and are accepted by regulators.

Shewhart, CUSUM, and EWMA

For individual observations, Shewhart uses moving range to estimate $\sigma$: $\hat{\sigma} = \overline{MR}/d_2$ where $d_2 \approx 1.128$ and $MR_t = |x_t - x_{t-1}|$. Control limits at $\pm 3\hat{\sigma}$ give FAR = 0.27% under normality.

CUSUM: $C_t^+ = \max(0, C_{t-1}^+ + e_t - k)$, $C_t^- = \max(0, C_{t-1}^- - e_t - k)$, alarm when $C_t^\pm > h$

EWMA: $Z_t = \lambda x_t + (1-\lambda)Z_{t-1}$, limits: $\mu_0 \pm L\sigma\sqrt{\lambda/(2-\lambda)}$

3. Multivariate Anomaly Detection

Isolation Forest

Isolation Forest scores anomalies by how quickly they can be isolated in random splits. Anomalies require fewer splits because they occupy sparse regions. Score is inversely proportional to average path length across the ensemble of trees.

One-Class SVM

$$\min_{\mathbf{w}, \xi, \rho} \frac{1}{2}\|\mathbf{w}\|^2 - \rho + \frac{1}{\nu n}\sum_i \xi_i, \quad \text{s.t. } \langle\mathbf{w}, \phi(x_i)\rangle \geq \rho - \xi_i, \xi_i \geq 0$$ $$\text{Anomaly score: } s(\mathbf{x}) = -[\langle\mathbf{w},\phi(\mathbf{x})\rangle - \rho]$$

4. Temporal Anomaly Detection with Autoencoders

LSTM Autoencoder

The LSTM autoencoder learns to reconstruct normal sequences. Anomaly score:

$$s_t = \frac{1}{W}\sum_{\tau=t-W}^{t} \|\mathbf{x}_\tau - \hat{\mathbf{x}}_\tau\|_2^2$$ 
class LSTMAutoEncoder(nn.Module):
    """LSTM Autoencoder for temporal anomaly detection."""
    def __init__(self, n_features, seq_len, hidden_size=64, latent_size=16, n_layers=2):
        super().__init__()
        self.seq_len = seq_len
        self.encoder = nn.LSTM(n_features, hidden_size, n_layers,
                               batch_first=True, dropout=0.1)
        self.enc_fc  = nn.Linear(hidden_size, latent_size)
        self.dec_fc  = nn.Linear(latent_size, hidden_size)
        self.decoder = nn.LSTM(hidden_size, hidden_size, n_layers,
                               batch_first=True, dropout=0.1)
        self.out_fc  = nn.Linear(hidden_size, n_features)

    def encode(self, x):
        _, (h_n, _) = self.encoder(x)
        return self.enc_fc(h_n[-1])   # (batch, latent_size)

    def decode(self, z):
        h0 = self.dec_fc(z).unsqueeze(0).repeat(2, 1, 1)
        dec_in = self.dec_fc(z.unsqueeze(1).repeat(1, self.seq_len, 1))
        out, _ = self.decoder(dec_in, (h0, torch.zeros_like(h0)))
        return self.out_fc(out)

    @torch.no_grad()
    def anomaly_score(self, x):
        x_hat, _ = self(x)
        return ((x - x_hat) ** 2).mean(dim=(1, 2)).cpu().numpy()

def set_threshold(model, X_normal, percentile=99.5):
    """Set threshold at percentile of reconstruction errors on normal data."""
    scores = model.anomaly_score(torch.FloatTensor(X_normal))
    return float(np.percentile(scores, percentile))

Variational Autoencoder (VAE) for Anomaly Detection

The VAE learns a generative model of normal data. The latent space is regularised (Gaussian), preventing the encoder from memorising outliers:

$$\text{Anomaly score: } s(\mathbf{x}) = -\log p_\theta(\mathbf{x}|z), \quad z \sim q_\phi(z|\mathbf{x})$$ $$\mathcal{L}_{VAE} = \underbrace{\mathbb{E}_{q_\phi}[\log p_\theta(\mathbf{x}|z)]}_{\text{reconstruction}} - \underbrace{D_{KL}[q_\phi(z|\mathbf{x}) \| \mathcal{N}(0,\mathbf{I})]}_{\text{regularisation}}$$
Lesson — VAE vs. AE for Anomaly Detection
A deterministic AE can memorise anomalies seen during training (if any contaminate the normal dataset) — it learns to reconstruct them well. The VAE KL term forces the latent space to be compact and Gaussian, so rare patterns cannot be embedded without cost. Use VAE when the normal training set may contain occasional anomalies (contamination fraction <5%).

5. Rotating Machinery Diagnostics

Bearing Fault Frequencies

Bearing defects produce characteristic frequency components determined by geometry:

$$f_{BPFO} = \frac{N_b f_r}{2}\left(1 - \frac{d_b}{d_p}\cos\alpha\right) \quad \text{(Ball Pass Frequency, Outer race)}$$ $$f_{BPFI} = \frac{N_b f_r}{2}\left(1 + \frac{d_b}{d_p}\cos\alpha\right) \quad \text{(Ball Pass Frequency, Inner race)}$$ $$f_{BSF} = \frac{d_p f_r}{2 d_b}\left(1 - \left(\frac{d_b}{d_p}\cos\alpha\right)^2\right) \quad \text{(Ball Spin Frequency)}$$

where $N_b$ = number of balls, $f_r$ = shaft frequency, $d_b$ = ball diameter, $d_p$ = pitch diameter, $\alpha$ = contact angle.

Envelope Analysis Procedure

  1. Compute Spectral Kurtosis (SK) via STFT to find the resonance band with highest impulsiveness.
  2. Bandpass filter around the peak SK frequency.
  3. Extract the envelope via Hilbert transform: $envelope = |\text{analytic signal}|$.
  4. Compute PSD of the envelope signal.
  5. Identify peaks at $f_{BPFO}$, $f_{BPFI}$, $f_{BSF}$ and harmonics. Alarm if SNR > 10 dB.
import numpy as np
from scipy import signal
from scipy.signal import hilbert

def bearing_fault_detection(x, fs, f_fault, n_harmonics=5):
    """Envelope analysis + spectral kurtosis for bearing fault detection."""
    # 1. Spectral Kurtosis via STFT
    f_stft, _, Zxx = signal.stft(x, fs=fs, nperseg=256, noverlap=192)
    power = np.abs(Zxx)**2
    sk    = (power**2).mean(axis=1) / (power.mean(axis=1)**2 + 1e-12) - 2

    # Find peak SK band
    peak_idx = np.argmax(sk)
    f_center = f_stft[peak_idx]
    bw       = max(50.0, f_center * 0.1)

    # 2. Bandpass filter
    b, a = signal.butter(4, [max(10, f_center - bw), min(fs/2 - 10, f_center + bw)],
                         btype='bandpass', fs=fs)
    x_filt = signal.filtfilt(b, a, x)

    # 3. Envelope via Hilbert transform
    envelope = np.abs(hilbert(x_filt))

    # 4. PSD of envelope
    f_env, S_env = signal.welch(envelope - envelope.mean(), fs=fs, nperseg=1024)

    # 5. SNR at fault harmonics
    noise_floor = np.median(S_env)
    snr_list = []
    for k in range(1, n_harmonics + 1):
        idx = np.argmin(np.abs(f_env - f_fault * k))
        band_energy = S_env[max(0, idx-3):idx+4].max()
        snr_list.append(10 * np.log10(band_energy / (noise_floor + 1e-12)))

    return {
        'fault_score_dB':  max(snr_list),
        'fault_detected':  max(snr_list) > 10,   # 10 dB threshold
        'sk_peak_freq_Hz': float(f_center),
        'harmonic_snr_dB': snr_list,
    }

6. Remaining Useful Life (RUL) Prediction

RUL estimation is a regression problem where the target is time-to-failure. Key requirements that distinguish it from standard regression:

  • Monotonic degradation constraint: physically, RUL should not increase (use monotone constraints in XGBoost or penalise non-monotone predictions).
  • Censored data: most run-to-failure datasets have components that were removed before failure. Use survival analysis (Cox proportional hazards) or treat as right-censored regression.
  • Uncertainty quantification: always report a prediction interval, not just a point estimate. RUL at 80% confidence (conservative) is more actionable than the median RUL.
  • Health Index (HI): construct a scalar HI from multivariate sensor data (using PCA or autoencoder bottleneck) and monitor HI degradation trend with EWMA. RUL threshold: when HI drops below a defined failure threshold.
Pitfall — RUL Prediction Without Uncertainty
Reporting RUL = 47 hours without a confidence interval is useless for maintenance scheduling. The decision to pull a component depends on the cost of false alarm (unnecessary replacement) vs. false miss (failure in service). Only a calibrated prediction interval enables the correct cost-optimal decision.

7. Key Takeaways

  • SPC first: CUSUM ($k=0.5$, $h=5$, ARL$_0=370$) and EWMA ($\lambda=0.1$, $L=2.7$) are your baselines. Beat them before deploying ML.
  • Anomaly type determines method: point → CUSUM/Isolation Forest; collective → LSTM AE; contextual → conditional AE.
  • Isolation Forest > One-Class SVM for $d > 5$: $\mathcal{O}(n \log n)$ complexity, no kernel, interpretable path length score.
  • LSTM autoencoder: train on normal sequences only; set threshold at 99.5th percentile of training reconstruction errors.
  • VAE regularises latent space — use when training data may contain a small fraction of anomalies (<5%).
  • Bearing faults: spectral kurtosis → resonance band → Hilbert envelope → SNR at $f_{BPFO/BPFI/BSF}$. This pipeline is provably optimal for detecting cyclostationary impulses.
  • RUL prediction without calibrated uncertainty intervals is incomplete — always report confidence bounds for maintenance decisions.
